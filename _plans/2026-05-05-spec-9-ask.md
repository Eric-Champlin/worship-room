**Reactive store consumption (BB-45 anti-pattern).** Per `06-testing.md` § "Reactive Store Consumer Pattern" and `11b-local-storage-keys-bible.md` § "Reactive Store Consumption", the memorization store is a Pattern A reactive store — consumed via the standalone `useMemorizationStore()` hook from `@/hooks/bible/useMemorizationStore` which uses `useSyncExternalStore`. The hook returns the array of cards. **Components MUST consume via the hook — local `useState(getAllCards())` mirrors without subscription are the BB-45 anti-pattern and ship as silent correctness bugs.** The spec's memorize-button wire-up uses the hook, not a snapshot.

**Memorization store API (`@/lib/memorize`, re-exported from `@/lib/memorize/store.ts`).** `getAllCards()` returns sorted snapshot. `addCard({ book, bookName, chapter, startVerse, endVerse, verseText, reference })` deduplicates by verse range and returns the existing or new `MemorizationCard`. `removeCard(id)` removes by id, no-ops if missing. `isCardForVerse(book, chapter, startVerse, endVerse?)` returns boolean. `getCardForVerse(book, chapter, startVerse, endVerse?)` returns the card or undefined. `subscribe(listener)` returns unsubscribe function. `_resetForTesting()` clears cache + listeners (used in `beforeEach` for test isolation).

**ParsedVerseReference shape (`lib/parse-verse-references.ts`).** `{ raw, book, bookSlug, chapter, verseStart, verseEnd?, startIndex, endIndex }`. The Ask verse card receives `parsedRef: ParsedVerseReference | null` from the parent. Note that `bookName` is NOT a field on `ParsedVerseReference` — only `book` (full name) and `bookSlug` exist. The memorization store's `addCard` requires `bookName` (display name, e.g., "Romans"); for AskPage, the parsed `book` field IS the display name (e.g., "Romans" — see `parse-verse-references.ts:7`). So `bookName: parsedRef.book` is the correct mapping. The `book` field on the store should be the slug (e.g., "romans") for cross-store lookup parity with BibleReader's stores; `parsedRef.bookSlug` is the source. To match the existing BibleReader memorize handler shape (`verseActionRegistry.ts:412-420` uses `sel.book` for the slug AND `sel.bookName` for display — see `VerseSelection` type), the Ask wire-up uses `book: parsedRef.bookSlug, bookName: parsedRef.book`.

/**

 * with a question pre-filled and auto-submitted on arrival.

 * Pre-fill template: `Help me understand <reference>: "<verse text>"`

 * Mirrors the buildDailyHubVerseUrl pattern but uses the natural-language
 * reference + verse text to seed the AskPage textarea, rather than passing
 * structured params. AskPage's existing ?q= handler reads the q param and

export function buildAskUrl(selection: VerseSelection): string {

  return `/ask?${params.toString()}`
}
