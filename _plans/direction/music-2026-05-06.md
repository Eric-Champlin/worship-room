# Direction — Specs 11A and 11B: Music + Bedtime Routines Round 3 Visual Migration

**Date:** 2026-05-06
**Recon:** `_plans/recon/music-2026-05-06.md`
**Scope:** All Music surfaces — `/music` (single tabbed route, 3 tabs: Worship Playlists / Ambient Sounds / Sleep & Rest), `/music/routines` (separate page with RoutineBuilder, RoutineCard, 4 dialogs), the AudioProvider + AudioDrawer + AudioPill flyout architecture chrome (engine internals OUT OF SCOPE), and BibleSleepSection (BB-26 audio Bible cross-cutting integration).

## Premise

Music is one of the 5 canonical navbar items (Daily Hub, Bible, Grow, Prayer Wall, Music) and one of the few remaining surfaces not yet migrated to Round 3. The recon recommends a 2-spec split given ~10,580 production LOC + ~12,900 test LOC across 73 production files.

The cluster's central work shape is **applying Spec 10A canonical patterns to a new surface**: active-state opacity unification, font-script removal, font-serif italic migration, white-pill primary CTA migrations, severity color system migration, decorative tint preservation per Decision 11. Plus Music-specific patterns: AudioPill / AudioDrawer / SharedMixHero / scene + sound + scripture + story card chrome.

11A establishes cluster patterns. 11B applies them mechanically to RoutinesPage + adds a small behavioral fix mirroring Spec 10A's Bible-data sweep precedent.

## Locked decisions

### 1. Atmospheric layer

**Keep uniform `bg-dashboard-dark` + `<PageHero>` (which inlines `ATMOSPHERIC_HERO_BG`).
No per-tab atmospheric variants in Spec 11A.**

Per-tab atmospheric is a real Round 3 enhancement opportunity (Sleep darker than Worship, ambient-active subtle scene tint) but introducing a new atmospheric primitive on a visual-migration spec is the same anti-pattern we rejected on Settings/Insights (Spec 10 Direction Decision 1). Defer to Spec 11c.

Music continues to use the canonical inner-page atmospheric pattern shared with Friends, Grow, Settings, MyPrayers, Insights.

### 2. Active-state opacity unification

**Migrate `bg-white/[0.12]` → `bg-white/15` per Spec 10A canonical.**

Site:

- `pages/MusicPage.tsx:219` — active tab indicator

Trivial cluster-pattern alignment.

### 3. RoutinesPage Spec 10A canonical pattern migrations

**Apply all three Spec 10A canonical patterns to RoutinesPage in Spec 11B:**

- `font-script` span removal on "Routines" wordmark heading (line 125)
- `font-serif italic` subtitle migration to canonical body chrome (line 127) — `font-sans`, no italic, `text-white/60`
- `bg-primary` solid Create Routine CTA migration to canonical white-pill primary CTA Pattern 2 (line 187)

### 4. FrostedCard tier system on cards

**Unify border opacity to `[0.12]` only. NO blanket FrostedCard adoption.**

Music's cards have intentional chrome variance:

- Scene cards (FeaturedSceneCard, SceneCard) carry visual weight via gradient backgrounds — no border by design
- Sound buttons (SoundCard) are transparent buttons — no border by design
- Scripture/story cards (ScriptureSessionCard, BedtimeStoryCard) use `border-white/10` rolls-own — migrate to `border-white/[0.12]`
- SavedMixCard uses `border-white/10` — migrate to `border-white/[0.12]`
- "Build Your Own Mix" panel border `border-white/10` — migrate to `border-white/[0.12]`
- "Build a Bedtime Routine" card border `border-white/10` — migrate to `border-white/[0.12]`
- AudioDrawer border `border-white/10` — migrate to `border-white/[0.12]`
- RoutineCard already uses `border-white/[0.12]` — preserve

Same precedent as Spec 10B Direction Decision 21 (FrostedCard adoption deferral on Insights). Border opacity unification only; rolls-own chrome preserved.

### 5. Scene preset count

**Keep 11 scenes (3 featured + 8 all). No reduction.**

Variety is part of Worship Room's "more than competitors give you" positioning. 11 scenes spread across 2 visual tiers (Featured row + All Scenes grid) does not overwhelm.

### 6. Featured scenes carousel cinematic upgrade

**Defer to Spec 11c.**

Cinematic upgrade (desktop hero-feature pattern, larger artwork) is net-new design work, not migration. Spec 11A keeps current treatment (mobile carousel + desktop grid).

### 7. "Build Your Own Mix" panel chrome elevation

### 12. MeditateTabContent scripture-card visual drift

**Leave Meditate untouched. Migrate only Music scripture cards in Spec 11A.**

Cross-surface chrome unification doesn't fix anything visible; the surfaces don't render side-by-side. Cluster scoped narrowly. Defer cross-surface unification to Spec 11c if drift becomes a complaint.

### 13. RoutinesPage empty state

**Preserve 4 templates + add subtle warm one-liner hint between templates and Create Routine CTA.**

Templates are real content, not placeholders. Replacing with `<FeatureEmptyState>` would destroy content.

Hint copy: "Tap a template to start, or create your own."

Light-touch warmth, anti-pressure, matches Spec 10B narrative one-liner pattern. Single line, `text-white/60`, between templates section and Create Routine button.

### 14. Routine delete behavioral fix

**Fold into Spec 11B. ~10 LOC + 1 test.**

When a user deletes the actively-playing routine, `wr_session_state.activeRoutine` orphans. Fix: after `storageService.deleteRoutine(...)`, check if `state.activeRoutine?.id === deletingRoutine.id` and dispatch `STOP_ROUTINE` to AudioProvider.

Mirrors Spec 10A Bible-data sweep precedent. Surface behavioral correctness during execution, fold trivial fix into spec.

Add test: deleting an actively-playing routine clears the AudioPill / Drawer "Currently in routine: <name>" indicator and the underlying state.

### 15. AudioDrawer `rgba(15, 10, 30, 0.95)` background
