# Grow Migration — Direction Document

**Created:** 2026-05-04
**Status:** Locked decisions for Specs 6A, 6B, 6C
**Branch:** forums-wave-continued
**Save location:** `_plans/direction/grow-2026-05-04.md`

This is a decision document, not a spec. It captures the locked design
decisions for the Grow visual migration. All three Grow sub-specs
reference this document. Decisions here are LOCKED.

## Context

Grow is a feature cluster, not a single page. It spans:

- `/grow` — shell with Plans + Challenges tabs (~700 LOC)
- `/reading-plans/:planId` — detail page (329 LOC)
- `/challenges/:challengeId` — detail page (566 LOC)
- CreatePlanFlow — full-screen create surface (320 LOC)
- 4 overlay/celebration components

Total ~7,000 LOC across 31 production files + 23 test files. The
deprecated pattern density is the highest of any surface migrated so
far: `font-script` Caveat in 9 places, cyan textarea glow,
multiple `bg-primary` buttons, rolls-own FrostedCard chrome on every
card primitive.

Recon at `_plans/recon/grow-2026-05-04.md` (full audit).

User-emotional context: Grow is where users invest in themselves —
choosing a reading plan, joining a challenge, completing milestones.
The visual treatment must support both the calm browsing state
("which plan should I pick?") AND the emotional peaks (completing
a 21-day plan, finishing a challenge). Restraint on the browsing
surfaces; intentional celebration on completion moments.

## Locked decisions

### Structural

**1. Three-sub-spec split.**

- **Spec 6A: Grow shell + Plans tab + Challenge cards** — the
  listing surfaces. ~10 files, ~700 LOC, ~40 test updates.
- **Spec 6B: Detail pages + day content** — `/reading-plans/:planId`,
  `/challenges/:challengeId`, day content components. ~7 files,
  ~700 LOC.
- **Spec 6C: CreatePlanFlow + completion overlays** — the create
  surface and all 4 celebration overlays. ~6 files, ~1,300 LOC.

Rationale: ~7,000 LOC + ~40 test updates is too much to review in
one PR. Each sub-spec covers 1-2 pattern types, not 5. Mirrors the
Dashboard 4A/4B/4C split that worked well. Each is independently
shippable; failures in one don't block the others.

**2. BackgroundCanvas added to all Grow surfaces.**

- `/grow` — wrap main content (below hero) in BackgroundCanvas
- `/reading-plans/:planId` — wrap below-hero content
- `/challenges/:challengeId` — wrap below-hero content
- CreatePlanFlow — replace `CREATION_BG_STYLE` inline gradient with
  BackgroundCanvas

Hero keeps `ATMOSPHERIC_HERO_BG` overlay (radial purple ellipse at
top center). The hero's stronger atmospheric effect is appropriate
for hero zones; BackgroundCanvas fills the area below with subtler
blooms.

Rationale: atmospheric continuity with Dashboard, BibleLanding, Local
Support. Users navigating between feature clusters shouldn't
experience visual context switching.

**3. Hero subtitle migration: italic → plain prose.**

All three Grow surfaces have `font-serif italic text-white/60`
subtitles below the gradient h1. Migrate to `text-white/70
leading-relaxed` (no italic). Per design system rule: italic on body
subtitles is deprecated.

**4. Scripture text preserved as Lora italic.**

Same exception as Dashboard direction doc Decision 7. Scripture in
`DayContent`, `ChallengeDayContent`, `PlanCompletionOverlay`
blockquote, and `CreatePlanFlow` Jeremiah quote stays
`font-serif italic`. Scripture is the canonical italic-callout
exception across the entire app.

**5. Sticky tab bar gets background tint.**

Current state: `sticky top-0 z-40 backdrop-blur-md` with no `bg-*`
token, so blur sits over transparent. Reads as "floating
disconnected" once user scrolls past hero.

Add: `bg-hero-bg/70` (or equivalent muted dark tint) so the bar
anchors visually at scroll.

### Pattern decisions

**6. ALL `font-script` Caveat usages migrate to `GRADIENT_TEXT_STYLE`.**

The 9 sites:

- `ReadingPlanDetail.tsx:200` — h1 last word → entire h1 uses
  GRADIENT_TEXT_STYLE
- `ChallengeDetail.tsx:278` — same treatment
- `MilestoneCard.tsx:109` — celebration heading
- `ChallengeCompletionOverlay.tsx:191` — celebration heading
- `PlanCompletionOverlay.tsx:192` — celebration heading
- `CreatePlanFlow.tsx:182` (StepOne h1) — "What's on your heart?"
- `CreatePlanFlow.tsx:251` (StepTwo h1) — "How long of a journey?"
- `PlanNotFound.tsx:20` — fallback link
- `ChallengeNotFound.tsx:20` — fallback link

No exceptions. Consistency across the app outweighs individual
"emotional intimacy" arguments at any one site. Caveat is fully
deprecated; this completes its removal.

For PlanNotFound + ChallengeNotFound link styling: use plain text-
white/80 underlined link (these are utility nav, not celebration).

**7. ALL rolls-own FrostedCard chrome migrates to FrostedCard
primitive.**

Sites and target variants:

| Component                          | Current chrome                                     | Migrate to                                                                        |
| ---------------------------------- | -------------------------------------------------- | --------------------------------------------------------------------------------- |
| PlanCard                           | rolls-own `rounded-2xl bg-white/[0.06]`            | `<FrostedCard variant="default">`                                                 |
| UpcomingChallengeCard              | rolls-own `rounded-xl bg-white/[0.06]`             | `<FrostedCard variant="default">`                                                 |
| NextChallengeCountdown             | rolls-own `rounded-2xl bg-white/[0.06]`            | `<FrostedCard variant="default">`                                                 |
| HallOfFame items                   | rolls-own `rounded-xl bg-white/[0.06]`             | `<FrostedCard variant="subdued">`                                                 |
| PastChallengeCard                  | rolls-own `rounded-xl bg-white/[0.04]`             | `<FrostedCard variant="subdued">`                                                 |
| ActiveChallengeCard                | rolls-own `rounded-2xl border-2 border-primary/30` | `<FrostedCard variant="default">` (theme-color buttons preserved as inline-style) |
| Create-Your-Own-Plan card          | rolls-own `rounded-2xl bg-white/[0.06]`            | `<FrostedCard variant="default">`                                                 |
| DayContent action callout          | rolls-own `rounded-2xl bg-white/5`                 | `<FrostedCard variant="subdued">`                                                 |
| ChallengeDayContent action callout | rolls-own `rounded-2xl bg-white/5`                 | `<FrostedCard variant="subdued">`                                                 |

Side effect: rounded radius bumps from `rounded-2xl`/`rounded-xl` to
`rounded-3xl`/`rounded-2xl` per FrostedCard primitive defaults.

**8. Theme-color CTAs preserved as rolls-own with inline style.**

Per recon: ActiveChallengeCard's Resume/Continue/Join buttons,
ChallengeDetail Mark Complete, ChallengeShareButton, MilestoneCard
share button — all use `style={{ backgroundColor: themeColor }}`
with `getContrastSafeColor()` for accessibility.

This is the seasonal-palette brand expression and stays. Migration
only changes surrounding card chrome, not the buttons themselves.

The buttons' surrounding chrome (border, padding, layout) stays as
rolls-own since they're embedded in cards that consume FrostedCard.
No `<Button>` primitive wrapping for theme-color CTAs.

**9. `bg-primary` solid buttons migrate per emotional weight.**

| Site                                | Current      | Migrate to                    | Rationale                                       |
| ----------------------------------- | ------------ | ----------------------------- | ----------------------------------------------- |
| ConfirmDialog "Pause & Start New"   | `bg-primary` | `<Button variant="subtle">`   | Switching plans is utility, not climax          |
| DayCompletionCelebration "Continue" | `bg-primary` | `<Button variant="subtle">`   | Per-day completion is incremental, not climax   |
| CreatePlanFlow Step 1 "Next"        | `bg-primary` | `<Button variant="subtle">`   | Just navigation between steps                   |
| CreatePlanFlow Step 2 "Generate"    | `bg-primary` | `<Button variant="gradient">` | THIS is the emotional climax — "create my plan" |

Rationale: gradient showstopper reserved for genuine emotional
peaks. CreatePlanFlow's Generate button IS the peak (the user has
been building up to this moment for 2 steps). Other migrations are
utility navigation.

**10. Cyan textarea glow migrates to violet textarea glow.**

`CreatePlanFlow.tsx:195` StepOne textarea currently uses:
`border border-glow-cyan/30 ... shadow-[0_0_12px_2px_rgba(0,212,255,0.35),0_0_27px_5px_rgba(139,92,246,0.26)]`

Migrates to canonical violet textarea glow from DailyHub
Pray/Journal:
`border-violet-400/30 bg-white/[0.04] shadow-[0_0_20px_rgba(167,139,250,0.18),0_0_40px_rgba(167,139,250,0.10)] focus:border-violet-400/60 focus:ring-2 focus:ring-violet-400/30`

Reasoning: same emotional register (intimate authoring), same
canonical pattern. Cyan was a one-off experiment that never spread.

**11. Tonal Icon Pattern application.**

Per direction doc Decision 11 from Dashboard's direction doc, applied
to Grow's iconography:

| Surface                          | Icon                           | Tonal color                   | Rationale               |
| -------------------------------- | ------------------------------ | ----------------------------- | ----------------------- |
| GrowPage hero                    | (no icon — gradient text only) | n/a                           | Hero owns visual energy |
| Plans tab                        | BookOpen (in Tabs primitive)   | text-sky-300                  | Study/scripture         |
| Challenges tab                   | Flame (in Tabs primitive)      | text-amber-300                | Challenge/effort        |
| Create-Your-Own-Plan card        | Sparkles                       | text-violet-300               | Discovery/creation      |
| PlanCard cover                   | (emoji from data)              | preserve emoji                | Plan-specific identity  |
| ChallengeIcon                    | (component-determined)         | preserve themeColor           | Theme-color brand       |
| HallOfFame Trophy                | Trophy                         | text-amber-300                | Celebration/achievement |
| NextChallengeCountdown countdown | Clock or Calendar              | text-violet-300               | Anticipation/time       |
| Reminder bell (set)              | Bell                           | text-emerald-300              | Active state            |
| Reminder bell (unset)            | Bell                           | text-white/50                 | Affordance              |
| ChallengeShareButton             | Share2                         | text-white/80 (within button) | Action affordance       |
| Pulse dot on Challenges tab      | (small dot)                    | preserve themeColor           | Active challenge brand  |

The pulse dot on the Challenges tab keeps its inline-themed color —
this is functioning categorical signal, not a Tonal Icon Pattern
application.

### Cleanup

**12. Delete dead `FilterBar.tsx`.**

78 LOC, 0 imports. Pre-merge legacy. Removed in Spec 6A.

**13. CreatePlanFlow CREATION_BG_STYLE removal.**

Replace inline `radial-gradient(...) + linear-gradient(...)` with
`<BackgroundCanvas>` wrapper. Aligns with the rest of the rollout.

**14. PastChallengeCard `role="button"` semantic — out of scope.**

Pre-existing accessibility deviation (div with role=button instead
of real button or Link). NOT addressed in Spec 6. Logged for a
future a11y-focused spec.

**15. AudioPill behavior on Grow surfaces — out of scope.**

The global AudioPill renders on Grow routes for authenticated users
when no audio plays. Currently working as designed. Same future
"UX tuning on crisis-adjacent surfaces" follow-up as Local Support
captures this concern, but Grow likely isn't included since it's
growth-oriented, not crisis-oriented.
