**Critical observations:**

- **No atmospheric BackgroundCanvas.** Hero has its own inline `ATMOSPHERIC_HERO_BG` style (radial purple ellipse at top center on `#0f0a1e`). Below the hero, the page is just `bg-dashboard-dark` (`#0f0a1e`) — no glow blobs, no canvas.
- **Sticky tab bar** with shadow-on-scroll using IntersectionObserver sentinel. `backdrop-blur-md` only — there is no `bg-*` token on the wrapper, which means the blur sits over the page background unmodified. The recon JSON snapshot shows depth-1 ancestor with `sticky top-0 z-40 bg-white/[0.08] backdrop-blur-xl` — that's likely a stale capture; the live source is `backdrop-blur-md` with no background tint.
- **Both tab panels mount simultaneously** (`hidden` toggle, not conditional render). State preservation across tab switches.
- **Tabs primitive** (`components/ui/Tabs.tsx`, 88 LOC) is shared with other pages and already uses Round 3 tokens (`rounded-full border-white/[0.12] bg-white/[0.06] p-1`, active = `bg-white/[0.12] border-white/[0.15]` with violet shadow). This is the canonical violet tab pattern.
- **Hero subtitle uses `font-serif italic`** — flagged for migration (italic on prose body deprecated per design system).

### Authentication behavior

- Browsing the page (any tab, list of plans/challenges) is public.
- `useAuth()` + `useAuthModal()` gate every interactive write:
  - "Create Plan" CTA → `Sign in to create a personalized reading plan`
  - "Start Plan" / "Continue" / "Resume" on PlanCard → `Sign in to start this reading plan`
  - "Join Challenge" → `Sign in to join this challenge`
  - "Resume" on paused challenge → `Sign in to resume this challenge`
  - "Remind me" toggle → `Sign in to set a reminder`
  - "Mark Complete" only renders when `isAuthenticated && !isPastChallenge && isCurrentDay`
- Custom plan IDs (`getCustomPlanIds()`) gated behind `isAuthenticated` so logged-out users don't see "Created for you" tags from a previous session.
- `Sign in to ...` shell modal does NOT perform real auth (still UI shell — Phase 3 backend pending). Forums Wave Phase 1.5 will wire actual JWT auth.
