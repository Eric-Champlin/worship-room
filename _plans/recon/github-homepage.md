## Responsive Summary

**Strategy:** Desktop-first with max-width breakpoints, content section-scoped
**Key breakpoints:** 480px, 768px, 1012px, 1280px
**Component visibility swaps:** Nav global menu hidden on mobile, hamburger visible
**CSS order changes:** None detected
**Custom breakpoints needed:** max-width: 1012px is the primary layout shift (not standard Tailwind)

**Key architectural observations:**
- Dark theme throughout — no light/dark mode toggle
- Full-width sections with content constrained by max-width per section
- Heavy use of variable font weights (425, 440, 460, 480) requiring @font-face variable font support
- Purple/violet radial glow backgrounds used as decorative ambient lighting across sections
- Customer logo marquee uses continuous CSS animation with pause control
- Feature and story tabs use pill-style (rounded-full) design
- Accordion items change heading color (white ↔ gray) to indicate expanded/collapsed state
- Hero form stacks to column on mobile, inline row on desktop
- Stats section uses large 40px/500-weight text for impact numbers
