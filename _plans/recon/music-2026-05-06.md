# Worship Room — Music Recon Report

**Date:** 2026-05-06
**Branch:** forums-wave-continued
**Scope:** Every Music surface — `/music` (single tabbed route, 3 tabs: Worship Playlists / Ambient Sounds / Sleep & Rest), `/music/routines` (separate page with RoutineBuilder, RoutineCard, 4 dialogs), plus the `AudioProvider` + `AudioDrawer` + `AudioPill` flyout architecture that backs the audio runtime. Plus `BibleSleepSection` (BB-26 audio Bible cross-cutting integration) and the saved-mix / shared-mix / sound-favorites / listening-history substrate.
**Source-of-truth files inspected:** 73 production .tsx/.ts files — `pages/MusicPage.tsx` (293), `pages/RoutinesPage.tsx` (206), `components/music/*` (12 files / 1,617 LOC), `components/audio/*` (42 files / 4,438 LOC), 10 music-related hooks (1,886 LOC), `data/music/*` (4 files / 661 LOC), `data/{sound-catalog,scenes,scene-backgrounds}.ts` (694 LOC), `lib/audio-engine.ts` (476 LOC), `services/storage-service.ts` (music CRUD methods), 2 types files, 3 constants files, plus 49 test files spanning music + audio + hooks + data. Skeleton: `components/skeletons/MusicSkeleton.tsx` (30 LOC).
**Auth posture:** `/music` and all 3 tabs are **fully unauthenticated-browse** — no `<Navigate>` redirect, no `<RequireAuth>` wrapper. Browse, play, mix, share-mix-URL all work logged-out. Auth gate fires only on: (a) Saving a mix (`SaveMixButton` calls `useAuthModal`), (b) Bookmarking a sound or scene (`FavoriteButton` calls `useAuthModal`), (c) Bedtime Routines page (`RoutinesPage` Create / Clone via `useAuthModal`). The Saved Mixes section in AmbientBrowser is gated by `{isAuthenticated && mixes.length > 0}` so it disappears entirely when logged out OR when the auth'd user has zero mixes. Confirmed by Playwright captures at `playwright-screenshots/music-11A-ambient-laptop-1280.png` (logged-out — no Saved Mixes section) and `music-11A-ambient-authed-with-saved-mixes-laptop.png` (logged in + 3 seeded mixes — section visible).
**Recon recommends a 2-spec split:** **Spec 11A** for `/music` page shell + 3 tabs + AudioDrawer chrome + AmbientBrowser cards + scripture/story cards + SharedMixHero; **Spec 11B** for `/music/routines` + RoutineBuilder + RoutineCard + ContentPicker + 4 dialogs + RoutineCard delete-handler behavioral fix. Combined LOC ~10,580 production + ~12,900 test, two surfaces with related-but-distinct work shapes (audio cluster card chrome vs. routine builder form chrome). 11A is sequenced first because every cluster pattern it establishes (active-tab opacity unification, scene/sound/scripture card chrome, AudioDrawer / AudioPill / SharedMixHero canonical chrome) is reused by 11B.

**Critical observations:**

- **Tab pattern is already 95% canonical.** Same shape as Daily Hub, Settings, AskPage. Single drift point: active-state opacity `bg-white/[0.12]` vs. Spec 10A's `bg-white/15`. Trivial unification.
- **`<PageHero>` wrapper used here (Music + MyPrayers).** Other inner pages (Friends, Grow, Settings, Insights, RoutinesPage) inline `ATMOSPHERIC_HERO_BG` directly. Two patterns. Spec 8C's Decision 19 recommended consolidation but bounded scope to Bible files; Music's PageHero usage is documented intentional drift. Worth confirming: does 11A migrate to inline `ATMOSPHERIC_HERO_BG` for consistency, or keep `<PageHero>`?
- **Hero copy uses `<PageHero title="Music" subtitle="...">` —** the wrapper renders title + subtitle in a fixed shape; the title is rendered with `GRADIENT_TEXT_STYLE`. NO `font-script` span on Music hero (good — this drift is RoutinesPage-only).
- **Tab state derived from URL on every render** via `useSearchParams` — no `useState`, no race condition. Deep linking works; the verified pattern from DailyHub.
- **SharedMixHero is mounted INSIDE the ambient tabpanel** — when `?mix=<base64url>` is in URL, the hero is forced into ambient tab regardless of `?tab=` param.
- **`<TooltipCallout>` for `music-ambient-tab` tooltip** mounted at root (not within the tab bar div) — points at `tabBarRef`. Consumes `wr_tooltips_seen` localStorage key (canonical tooltip pattern).
- **Skip-to-main-content link** is provided by the canonical `Navbar` mount; MusicPage does NOT mount its own. `<main id="main-content">` is the target. Confirmed canonical pattern.
- **No `<DevAuthToggle />` mount** on Music shell (unlike Settings/Insights). Logged-out browse is the dominant use case so the dev toggle isn't surfaced here.

## Part 13 — Cross-Cutting Concerns

### Routine delete behavioral audit (Direction Decision 6 ask)

`RoutinesPage.tsx:80-86`:
```tsx
const handleDelete = () => {
  if (!deletingRoutine) return
  storageService.deleteRoutine(deletingRoutine.id)
  refreshRoutines()
  showToast(`Deleted "${deletingRoutine.name}"`)
  setDeletingRoutine(null)
}
```

`storageService.deleteRoutine` (line 281-287):
```ts
deleteRoutine(id: string): void {
  this.requireAuth('delete routine')
  const routines = readJSON<RoutineDefinition[]>(KEYS.routines, []).filter(
    (r) => r.id !== id,
  )
  writeJSON(KEYS.routines, routines)
}
```

**Behavioral findings (analogous to Spec 10A's Bible-data leak):**

1. **Active routine state is NOT cleared.** If the user is currently playing the routine they're about to delete, `wr_session_state.activeRoutine` will continue to reference the deleted id. Result: the AudioPill/Drawer continues to show "Currently in routine: <name>" but the underlying routine no longer exists in `wr_routines`. Subsequent step transitions will fail silently (the next step lookup will return undefined). **MINOR severity — only triggered if user deletes the actively-playing routine, which is unusual but possible.**
2. **Listening history entries with `contentType: 'routine'` are NOT cleared.** `wr_listening_history` may contain `{ contentType: 'routine', contentId: '<deleted-routine-id>', ... }` rows after deletion. `useGardenElements` and `badge-engine` consume these — neither dereferences `contentId` against `wr_routines` (they only look at `contentType` + `durationSeconds`), so the orphaned references are **harmless**. Could remain orphaned indefinitely; that's acceptable as an immutable historical record.
3. **No reference cleanup needed for mixes / favorites** — neither references routines.

**Recommendation for Spec 11B:**
- **Fold a small behavioral fix into 11B**: after `deleteRoutine`, dispatch `STOP_ROUTINE` to AudioProvider IF the deleted routine matches `state.activeRoutine?.id`. Add the check in `RoutinesPage.handleDelete`. ~10 lines, mirrors the Spec 10A Bible-data sweep precedent.
- Alternative: defer to a future Spec 11c if scope concern. Document as Open Question 6.

  contentType: 'ambient' | 'scene' | 'scripture' | 'story' | 'routine'

## Round 3 Enhancement Lenses

Each lens applied to Music's 3 tabs + RoutinesPage, then a single block of competitor analysis.

### Speed-to-peace

- **Worship Playlists:** Fast — first embed visible above fold. But Spotify Premium gate locks full playback.
- **Ambient Sounds:** Mixed — AmbientBrowser is dense but well-structured. First Featured scene visible above fold on desktop.
- **Sleep & Rest:** Slow — long vertical stack (24 readings + 12 stories + Bible Sleep section + Tonight's Scripture + Build a Routine). Decision fatigue at the bottom.
- **Routines:** Fast on revisit (4 templates + user routines). Slow first-time (no onboarding).

**Enhancement candidates:**
- Sleep tab: collapse-by-default sub-collections; show only Tonight's Scripture + first collection unprompted, "Show more" for the rest
- Routines: add a short "How routines work" hint above the templates on first visit

### Time-of-day awareness

- **Music as a whole:** Time-agnostic. Same default tab regardless of time.
- **Tab content:** Sleep & Rest's TonightScripture rotates daily, but doesn't differentiate morning vs evening.

**Enhancement candidates:**
- Default tab: morning → Worship Playlists, midday → Ambient, evening → Sleep & Rest
- Sleep tab heading: "Tonight's Scripture" works for evening; "Today's Restful Reading" for daytime
- Routines hero: "Bedtime Routines" reads slightly off if a user opens it at 9 AM. Could be "Restful Routines" with a smaller "best at night" hint

### Decision fatigue reduction

- **Worship Playlists:** Low — 1 hero + 7 explore embeds.
- **Ambient Sounds:** High — 11 scenes + 24 sounds + 3 mixes (logged-in) = 38 buttons in scope at once.
- **Sleep & Rest:** Very high — 36 audio sessions + Bible link + Routine link.
- **Routines:** Low — 4 templates + create button.

**Enhancement candidates:**
- Sleep tab: show only "Tonight's Scripture" + the most-recently-played collection by default; collapse other collections behind an accordion
- Ambient tab: hide "Build Your Own Mix" behind a tap-to-expand, surface featured + all scenes only for the casual user

### Ritual building

- **Music:** **Strongest ritual potential** of any feature. Bedtime routine = pre-sleep ritual; ambient mix = focus ritual; worship playlist = morning prayer ritual.
- **Routines page:** Already ritual-aligned by name. Just needs onboarding to communicate the value.

**Enhancement candidates:**
- Sleep tab: "Your nightly routine" callout if the user has a saved routine
- Ambient tab: "Your usual mix" callout when active sounds match a saved mix
- Routines: first-run onboarding hint

### Growth narrative

- **Music:** Light. The Worship Listener badge + garden integration via `wr_listening_history` are the only narrative threads.
- **Routines:** None. No tracking of how many times a routine has played.

**Enhancement candidates:**
- Routines: small "Last played 3 days ago" timestamp on each user routine (currently absent)
- Music: a "Your most-played scene" callout in Settings → App or Insights

### Emotional moments

- **Worship Playlists:** Mild emotional peak when hero plays.
- **Ambient Sounds:** Mild celebration when saving a mix (confetti? toast?). Currently just a toast.
- **Sleep & Rest:** Quiet emotional moment when "Tonight's Scripture" lands on a meaningful verse. No celebration on completion.
- **Routines:** Mild peak when a routine completes successfully (currently silent).

**Enhancement candidates:**
- Routines: gentle completion celebration when a routine finishes its last step (matches the gentle gamification rule)
- Sleep tab: "Sweet dreams" toast when sleep timer expires

### Surprise / delight

- **Music:** None today.
- **Tonight's Scripture:** Rotates daily, so there's a small surprise each visit, but no signaling of "it's different today."

**Enhancement candidates:**
- Sleep tab: subtle "New tonight" pill on Tonight's Scripture if it's different from yesterday
- Ambient tab: surprise scene-of-the-day featured at top once per week

### Guilt-free re-engagement

- **Music:** Low risk — no streak shaming on music; no "you haven't listened in N days" messaging.
- **Routines:** Low risk — same.

**Enhancement candidates:**
- None needed — Music is already anti-pressure-aligned.

### Shareable moments

- **Music:** SharedMixHero is the canonical share-receive surface. Saving a mix shareably is via SaveMixButton → share-mix-URL pattern.
- **Routines:** No share affordance.

**Enhancement candidates:**
- Routines: "Share this routine" affordance — generate a deep link
- Saved mixes: promote share-mix-URL more prominently (currently hidden in the SaveMixButton flow)

### Beauty as sanctuary

- **Music:** **Strongest beauty surface** — scene gradients, hero artwork on Spotify embeds, Tonight's Scripture card. But the gradients on All Scenes feel utility (small thumbnails) compared to Featured.
- **Routines:** Functional. Cards are utility-tier.

**Enhancement candidates:**
- Promote scene artwork in All Scenes grid to taller cards
- Featured scenes carousel on desktop (Direction Decision 6)
- Routines cards: subtle scene-art preview if the routine starts with a scene

### Sound as atmosphere

- **Music:** **THE sound-as-atmosphere lens.** Currently visual chrome doesn't reflect what's playing — when a scene is active, the page chrome stays the same. Endel's pattern of generative visuals tied to active sound is the gold standard.

**Enhancement candidates:**
- Ambient-active subtle background tint that picks up the active scene's gradient color (Direction Decision 1 (b))
- AudioPill / AudioDrawer breathing-pulse animation tied to active scene tempo
- Sleep tab: dimmer atmospheric overlay during evening hours (Direction Decision 1 (b))

### Freshness

- **TonightScripture:** Daily rotation — good.
- **Featured scenes:** Static. No rotation.
- **Worship Playlists:** Static. No personalization.

**Enhancement candidates:**
- Featured scenes: weekly rotation of which 3 are featured
- Worship Playlists: a "Today's Pick" sub-section above Featured (matches Daily Hub's SongPickSection pattern)

### Community warmth

- **Music:** None today.
- **Shared mixes:** A friend can share a mix URL — that's the only social touch.

**Enhancement candidates:**
- "Friends praying with this scene" badge on scenes that another friend has played
- "Top 3 scenes among Worship Room users this week" chart

### Tell-a-friend

- **Music:** SharedMixHero is exactly this. But the share-mix-URL generation flow is buried inside SaveMixButton.

**Enhancement candidates:**
- Promote share affordance to Saved Mixes section (always visible "Share this mix" link on each card)
- Add "Share this scene" affordance on FeaturedSceneCard

### Competitor analysis

This research, summarized from WebSearch, surfaces precedents Worship Room could selectively adopt.

**Calm** ([Behance UX gallery](https://www.behance.net/gallery/218671419/Calm-Rest-App-UIUX-Design-Mobile-Design), [UX Case Study](https://usabilitygeek.com/ux-case-study-calm-mobile-app/), [2026 trends](https://elements.envato.com/learn/ux-ui-design-trends), [UI Sources](https://uisources.com/app/calm)): Soft azure gradient + naturalistic typeface. Player intentionally minimalist — only play/pause and stop, plus volume and favourite. **Deliberate lack of fast-forward** prevents skipping through programs. 2026 design trends emphasize "calmer micro-interactions" replacing extravagant motion. **Lesson:** the no-fast-forward stance is a strong anti-anxiety pattern — Worship Room's Sleep tab could adopt this for scripture and story playback (currently Bible audio has a transport bar; Sleep tab content is simpler). The 2026 calm-interface trend validates Worship Room's existing low-saturation chrome.

**Headspace Sleepcasts** ([Sleepcasts page](https://www.headspace.com/sleep/sleepcasts), [help article](https://help.headspace.com/hc/en-us/articles/360009450393-What-is-a-Sleepcast), [Spotify](https://open.spotify.com/playlist/5kxTrSYiq0QFoWXBP5wGYy)): Each Sleepcast = wind-down + immersive environment description (Living Biodomes, Coastal Campgrounds, Californian desert at night). **Remixed each night** so users can't memorize narrative, intentionally preventing anxious clock-tracking. **Lesson:** the remixed-each-night design is genius — it removes the anxiety of "I've heard this before, how long is left." Worship Room's Tonight's Scripture rotation hints at this; could push further by occasionally re-mixing scripture readings with different ambient backings.

**Hallow** ([Hallow main](https://hallow.com/), [features](https://hallow.com/features/), [TechForJesus review](https://cloud.techforjesus.org/tools/app/30-hallow.html)): Direct competitor. Compline + Night Prayer + Examen + Lectio Divinas. **3 length options (5/10/15 min)** + custom length (1/5/10/15/30/60). Set prayer reminders, invite others, track goals + streaks. Offline downloads. UX usability concern: difficulty finding daily routines. **Lesson:** the 3-length default option (5/10/15) is exactly the canonical Worship Room guided prayer pattern. Routine length awareness ("~12 min") is already canonical on RoutineCard. Hallow's "difficulty finding daily routines" complaint validates Worship Room's prominent Bedtime Routines placement on Sleep tab + dedicated `/music/routines` page.

**Endel** ([Endel main](https://endel.io/), [Soundscapes](https://endel.io/soundscapes), [Technology](https://endel.io/technology), [Musicology Now review](https://musicologynow.org/welcome-to-the-sound-wellness-revolution-endels-ai-generated-soundscapes-and-the-commodification-of-passive-listening/)): **Each soundscape comes with its own generative visual.** Adapts to time of day, weather, motion, heart rate. Pentatonic-scale-based for low cognitive load. **Lesson:** Endel's sound-visual coupling is the gold standard for "sound as atmosphere." Worship Room could adopt a lighter version — when a scene is active, the page background subtly shifts to reflect the scene's gradient. Direction Decision 1 (b) territory.

**Brain.fm** ([Brain.fm main](https://www.brain.fm/), [DesignWanted interview](https://designwanted.com/brain-fm-functional-music-for-focus/), [ScreensDesign showcase](https://screensdesign.com/showcase/brainfm-focus-music)): Categorized by mental state (Focus / Relax / Sleep). **Music designed gap-free** — no breaks, sharp changes, or abrupt instrumental additions. 3D spatial audio. Structured onboarding "visualizes its impact with charts." **Lesson:** the mental-state categorization maps exactly to Worship Room's tab structure (Worship / Ambient / Sleep). Brain.fm's "gap-free music" principle is worth thinking about for routine transitions — the current Worship Room routine player has hard cuts between steps.

**Spotify Daylist + Prompted Playlist** ([Daylist launch](https://edm.com/gear-tech/spotify-launches-personalized-playlist-daylist/), [Prompted Playlist news](https://newsroom.spotify.com/2025-12-10/spotify-prompted-playlists-algorithm-gustav-soderstrom/), [2026 expansion](https://newsroom.spotify.com/2026-01-22/prompted-playlists-expansion/)): Daylist updates throughout the day, transitions tempo morning → evening. **Prompted Playlist** (Beta 2026) lets users describe what they want in natural language. **Lesson:** time-of-day adaptive content is now industry-canonical. Direction Decision 8 territory. Natural-language playlist generation is out of scope for Spec 11 but a clear future opportunity given Worship Room's existing Gemini integration (BB-32 cache).

**Apple Music iOS 26.4** ([MacObserver redesign](https://www.macobserver.com/news/ios-26-4-brings-7-new-features-and-a-redesign-to-apple-music/), [GeekyGadgets Music 2026 overhaul](https://www.geeky-gadgets.com/apple-musics-2026-overhaul-the-game-changing-features-you-cant-miss/)): Larger album/playlist covers, **interactive live artwork (animated album covers)**, Playlist Playground (natural-language playlist generation). Personalized mixes (Favorites/Chill/Discovery). **Lesson:** larger album/playlist artwork is canonical now. Worship Room's WorshipPlaylistsTab could feature its hero embed even larger on desktop. Animated album covers are aspirational — out of scope but a clear cinematic enhancement direction.

**YouTube Music ambient channels** ([YouTube Ambient blog](https://blog.youtube/culture-and-trends/ambient-music/), [Ambient Nightscapes](https://www.youtube.com/channel/UCcKeTbUZa34td1DfziJid8g), [Ambient Worlds](https://www.youtube.com/ambientworlds)): Cinematic ambient music = looped visuals + immersive soundscapes designed for sleep/focus. Long runtime, atmospheric backgrounds. **Lesson:** this is the closest comp to Worship Room's scene + ambient mix combination. Ambient Nightscapes' "deep night atmospheres for late hours" is exactly what Worship Room's Sleep tab evening atmospheric variant could feel like. Direction Decision 1 (b) precedent.
