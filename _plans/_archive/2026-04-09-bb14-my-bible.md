### Subscribe Pattern

All three Bible stores (highlight, bookmark, notes) use a module-level `Set<() => void>` listeners pattern. `subscribe(fn)` adds to the set and returns an unsubscribe function. Listeners fire on every write mutation. The `useActivityFeed` hook subscribes to all three.

Meditation storage (`wr_meditation_history`) has **no subscribe method**. Meditation sessions won't change while the user is viewing the My Bible page (they'd have to navigate away and meditate). The hook reads meditation history on mount only; re-reading on focus could be a future enhancement but is unnecessary for BB-14.

## Edge Cases & Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| "From Daily Hub" shows only meditation | Show meditation cards with verseContext; omit journal/prayer | Journal entries aren't persisted to localStorage; prayers don't have verseContext. Shipping with partial coverage + descriptive empty state is better than blocking on a persistence redesign. |
| formatReference for activity items | Use `formatReference` from `lib/dailyHub/verseContext.ts` (takes raw args) | Avoids constructing a full `VerseSelection` object just to format a string. Simpler and already tested. |
| Verse text loading | Lazy-load chapter data per unique (book, chapter) pair; cache in a Map within the hook | Cards may reference many chapters; loading all at once would be expensive. Cache prevents redundant fetches for cards in the same chapter. |
| No `subscribe()` on meditation storage | Read on mount, no reactive subscription | Meditation sessions won't change while viewing My Bible. User must navigate away to meditate and come back. |
| Canonical sort order | Use `BIBLE_BOOKS.findIndex(b => b.slug === item.book)` for book ordering, then chapter, then verse | `BIBLE_BOOKS` array is already in canonical order. |
| Action menu positioning | Desktop: absolute positioned near right-click point. Mobile: fixed bottom sheet. | Follows AudioDrawer + verse action sheet patterns for consistent UX. |
| Book filter on mobile | Bottom sheet with scrollable list | Spec requirement. Matches existing mobile patterns. |
| Note reference links | Parse via `parseReferences()`, render as styled pill buttons, toast "Coming in a future update" on click | Spec says "toast stub until BB-38". |
| Bookmark label editing | Inline text input in action menu popover | Keep editing lightweight â€” no separate modal. |
| Long-press timing | 400ms via `setTimeout` in `onPointerDown`/`onPointerUp` | Spec requirement. Cancel on `onPointerMove` (>10px delta) and `onPointerUp` before timeout. |
| Max items display | No virtualization initially; all items render. Add virtualization only if perf testing shows scroll jank at 1000+ items. | Spec explicitly defers virtualization. |

### Step 2: useActivityFeed Hook + Verse Text Cache

**Objective:** Create the reactive hook that subscribes to stores, manages filter/sort state, loads verse text for cards, and exposes the filtered/sorted activity feed.

**Files to create/modify:**
- `src/hooks/bible/useActivityFeed.ts` â€” new file
- `src/hooks/bible/useVerseTextCache.ts` â€” new file, lazy verse text loader with Map cache

**Details:**

`useVerseTextCache` hook:
- Maintains a `Map<string, Map<number, string>>` keyed by `${book}:${chapter}` â†’ verse number â†’ text
- Exposes `getVerseText(book, chapter, startVerse, endVerse): string | null` (returns null while loading)
- Exposes `preloadChapters(items: ActivityItem[]): void` â€” batches unique (book, chapter) pairs and calls `loadChapterWeb` for each, updating the cache
- Uses `useRef` for the cache Map to avoid re-renders; triggers re-render via a version counter `useState`
- Caches results so repeated calls for the same chapter don't re-fetch

`useActivityFeed` hook:
- Parameters: none (filter/sort state is internal)
- Returns: `{ items: ActivityItem[], filter: ActivityFilter, sort: ActivitySort, setFilter, setSort, totalCounts: { highlights, notes, bookmarks, booksSet, meditations, streak }, isEmpty: boolean, isFilteredEmpty: boolean, clearFilters: () => void, getVerseText: (book, chapter, start, end) => string | null }`
- On mount: calls `loadAllActivity()`, computes `totalCounts`, subscribes to highlight/bookmark/note stores
- On store change: re-calls `loadAllActivity()`, re-computes
- On filter/sort change: calls `filterActivity` and `sortActivity` (memoized)
- Preloads verse text for visible items via `useVerseTextCache`
- Cleanup: unsubscribes all three stores on unmount

```typescript
// Subscription pattern
useEffect(() => {
  const reload = () => { /* re-load, re-filter, re-sort */ }
  const unsubs = [
    highlightStore.subscribe(reload),
    bookmarkStore.subscribe(reload),
    noteStore.subscribe(reload),
  ]
  return () => unsubs.forEach(fn => fn())
}, [])
```

**Responsive behavior:** N/A: no UI impact.

**Details:**

`useLongPress(callback, delay = 400)` hook:
- Returns `{ onPointerDown, onPointerUp, onPointerMove, onPointerCancel }`
- `onPointerDown`: records start position, starts `setTimeout(delay)`
- `onPointerUp` / `onPointerCancel`: clears timeout
- `onPointerMove`: if delta > 10px from start, clears timeout (prevents firing on scroll)
- On timeout fire: calls `callback(event)` with the pointer event for positioning
- Prevents default text selection during long-press via `touch-action: none` on the card

`ActivityActionMenu` component:
- Props: `{ item: ActivityItem, position: { x: number, y: number }, onClose: () => void, onMutate: () => void }`
- Dark frosted glass popover: `bg-[rgba(15,10,30,0.95)] backdrop-blur-[16px] border border-white/[0.12] rounded-xl shadow-[0_0_25px_rgba(139,92,246,0.06),0_4px_20px_rgba(0,0,0,0.3)]`
- Positioned via `position: fixed; top/left` based on pointer event coordinates, clamped to viewport bounds
- On mobile (< 640px): renders as a fixed bottom sheet instead of a positioned popover: `fixed bottom-0 left-0 right-0 rounded-t-2xl` with backdrop overlay
- Focus trap via `useFocusTrap(true, onClose)`
- Escape key closes via focus trap's onEscape callback
- Click outside overlay closes

After mutation: call `onMutate()` to trigger feed refresh (the store subscription handles this, but closing the menu cleanly is also needed).
