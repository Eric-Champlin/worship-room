- **Marker pattern** (`VerseBookmarkMarker.tsx`): `<span>` wrapper with `aria-hidden="true"`, `fontSize: '0.7em'`, inline SVG with `color: var(--bookmark-marker)`, `verticalAlign: 'baseline'`. Rendered inline in verse span.

- **BibleReader subscription** (`BibleReader.tsx` lines 131-146): `useState` initialized from store getter, `useEffect` to refresh on chapter change, second `useEffect` with `subscribe()` that calls the store getter on change. Same pattern used for highlights (lines 88-129) and bookmarks (lines 131-146).
- **CharacterCount component** (`CharacterCount.tsx`): Accepts `current`, `max`, `warningAt`, `dangerAt`, `visibleAt`, `id`, `className`. Has `aria-live="polite"` announcements on zone change. Reusable for BB-8 with `warningAt={9000}`, `dangerAt={10000}`.

## Design System Reminder

**Project-specific quirks that `/execute-plan` displays before every UI step:**

- The Bible reader uses `[data-reader-theme]` scoped CSS custom properties. All styling inside the reader MUST use `var(--token)` — NOT Tailwind dark-mode classes or raw hex values.
- Bookmark marker pattern: `<span>` wrapper with `aria-hidden="true"`, inline SVG at `0.85em` size, `color: var(--bookmark-marker)`, `verticalAlign: 'baseline'`, `fontSize: '0.7em'`. Note marker follows the same pattern with `var(--note-marker)`.
- The action sheet sub-view system uses `renderSubView({ selection, onBack, context })`. The component receives these props — it does NOT call `ctx.pushSubView` (that method doesn't exist). Sub-view navigation is handled by `VerseActionSheet` state.
- The note editor uses the reader's theme-scoped colors, NOT the Daily Hub's white textarea glow pattern. No `shadow-[...]` glow, no `animate-glow-pulse`, no cyan borders. Calm, minimal styling.
- FrostedCard is NOT used inside the action sheet sub-view. The sub-view has its own minimal styling within the sheet chrome.
- All tap targets must be at least 44px (spec AC-218).
- `prefers-reduced-motion` must be respected on sub-view open/close transitions (spec AC-220).
- Primary action filled state: `state?.active ? '[&>*]:fill-current'` applies fill to SVG children. Active color applied to wrapper `<span>`. Pattern from highlight and bookmark handlers.

**From recent execution logs (BB-7 + BB-7.5, same session):**
- No design system deviations found — both executed cleanly against the reader's theme token system.

## Edge Cases & Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Overlap detection | Match verse anywhere inside existing range | Spec req 4-5: "if a note already overlaps the selected verse(s), the editor loads the existing note body". Range does not shrink. |
| Empty body save | Do not persist notes with empty/whitespace-only body | No point saving blank notes; if user clears all text, treat as implicit delete intent |
| Paste beyond limit | Truncate to 10,000 chars | Spec req 16: "pasting beyond the limit truncates". Use `maxLength` + onChange guard. |
| Tab in textarea | Insert tab char | Spec req 17: "Tab inserts a tab character (not focus-next)". Capture Tab keydown, `preventDefault`, insert `\t`. |
| Crisis detection | NOT included | Spec explicitly says no crisis detection — notes are private, never sent to server or displayed publicly |
| Reference parser — ambiguous abbreviations | Require word boundary after abbreviation | "Am" could be "Amos" or the word "am". Require `\d+:\d+` pattern after to disambiguate. Chapter-only refs require full book name or unambiguous abbreviation. |
| Old NoteEditor.tsx | Left in place (deprecated) | Not imported by any production code. BB-14 will build the "My Bible" notes view and may use a new component. Cleanup is not BB-8 scope. |
| Reference link navigation | Toast placeholder | BB-38 hasn't shipped. Toast: "Jump-to-reference ships with deep linking" |
| Note marker icon | Pencil outline (PenLine from lucide, inline SVG) | Spec says "pencil or document outline icon". PenLine matches the Note action icon in the registry, maintains visual consistency. |
| `timeAgo` for "Edited X ago" | Adapter function using epoch ms → ISO → `timeAgo()` | Existing `timeAgo` returns full words ("minutes"). The spec shows "Edited 3 min ago" but abbreviation isn't mandatory — "Edited 3 minutes ago" is fine. The subtitle space is sufficient. |
| Store directory | `src/lib/bible/notes/store.ts` | Spec explicitly requests `notes/` subdirectory. Different from highlights/bookmarks which are at the `lib/bible/` root. |

### Step 4: useAutosave Hook

**Objective:** Create a generic, reusable autosave hook. TDD: write tests first, then implementation.

**Files to create:**
- `frontend/src/hooks/__tests__/useAutosave.test.ts`
- `frontend/src/hooks/useAutosave.ts`

**Details:**

**Hook API:**
```typescript
type AutosaveStatus = 'idle' | 'saving' | 'saved' | 'error'

interface UseAutosaveOptions {
  value: string
  onSave: (value: string) => void | Promise<void>
  delay?: number          // debounce ms, default 2000
  enabled?: boolean       // skip autosave when false (e.g., while loading)
}

interface UseAutosaveReturn {
  status: AutosaveStatus
  flush: () => void       // force immediate save (for close/Cmd+S)
  lastSavedAt: number | null  // epoch ms of last successful save
}
```

**Behavior:**
- On `value` change: reset debounce timer, set status to `'idle'` (or keep current if saving)
- After `delay` ms of no changes: call `onSave(value)`, status → `'saving'` → `'saved'` (on success) or `'error'` (on catch)
- `flush()`: cancel debounce timer, save immediately if value differs from last saved
- Skip save if `value` hasn't changed since last successful save (idempotent)
- `enabled: false` → no debounce timer fires, but `flush()` still works

**Tests (~12):**
1. Initial status is 'idle'
2. Status becomes 'saving' then 'saved' after delay
3. Debounce resets on each value change
4. `flush()` saves immediately
5. `flush()` no-ops when value unchanged since last save
6. Status becomes 'error' when onSave throws
7. `enabled: false` prevents auto-save
8. Skip save when value unchanged
9. `lastSavedAt` updates on successful save
10. Cleanup cancels pending timer on unmount
11. Custom delay is respected
12. Concurrent value changes during save are handled (latest value wins)

Use `vi.useFakeTimers()` for debounce testing. Use `renderHook` from `@testing-library/react`.

**Auth gating:** N/A — generic hook.

**Responsive behavior:** N/A: no UI impact.

```typescript
export interface ParsedReference {
  text: string           // matched text e.g. "John 3:16"
  startIndex: number     // char position in source text
  endIndex: number       // char position in source text
  bookSlug: string       // resolved slug e.g. "john"
  bookName: string       // display name e.g. "John"
  chapter: number
  startVerse?: number    // undefined for chapter-only refs
  endVerse?: number      // undefined for single verse or chapter-only
}

export function parseReferences(text: string): ParsedReference[]
```

**Implementation approach:**
1. Build a name → slug lookup map from `BIBLE_BOOKS` (full names + common abbreviations)
2. Build a regex that matches: `(BookName) (\d+)(?::(\d+)(?:[–-](\d+))?)?` where `BookName` alternates all names/abbreviations
3. For numbered books (1/2/3 prefix), handle both "1 John" and "1John" patterns
4. Word boundary enforcement to avoid matching inside prose words
5. Negative lookahead for time patterns: reject if followed by `\s*(?:am|pm|a\.m\.|p\.m\.)`

**Abbreviation map** (embedded in the module): Include common abbreviations like Gen, Ex, Lev, Num, Deut, Josh, Judg, Ps/Psalm, Prov, Isa, Jer, Matt, Mk, Lk, Jn, Rom, 1 Cor, 2 Cor, Gal, Eph, Phil, Col, 1 Thess, 2 Thess, 1 Tim, 2 Tim, Heb, Jas, 1 Pet, 2 Pet, 1 Jn, 2 Jn, 3 Jn, Rev.

```typescript
function editedAgo(epochMs: number): string {
  const seconds = Math.floor((Date.now() - epochMs) / 1000)
  if (seconds < 60) return 'Edited just now'
  const minutes = Math.floor(seconds / 60)
  if (minutes < 60) return `Edited ${minutes}m ago`
  const hours = Math.floor(minutes / 60)
  if (hours < 24) return `Edited ${hours}h ago`
  const days = Math.floor(hours / 24)
  return `Edited ${days}d ago`
}
```
