## Affected Frontend Routes

The image upload affordance lives in the testimony and question composers; `<PostImage>` (and its `<ImageLightbox>` overlay) renders inside `PrayerCard`, which is consumed at every Prayer Wall surface. The composer affordance ships behind the established per-type composer infrastructure. `<PostImage>` is decoupled from `postType` (W21) — it renders whenever `prayer.image` is non-null.

- `/prayer-wall` — composer affordance for testimony/question; mixed-feed `<PostImage>` rendering with chrome-color preservation
- `/prayer-wall/:id` — single-card render with lightbox
- `/prayer-wall/dashboard` — multi-tab feed
- `/prayer-wall/user/:id` — author profile feed
- `/my-prayers` — bookmarked prayers feed

The upload endpoint (`POST /api/v1/uploads/post-image`) has no UI route; it is consumed by `prayerWallApi.uploadImage(file)`.

## Architecture Context

**Recon ground truth (2026-05-09, verified on disk):**

### Storage (1.10e — already shipped, do NOT modify)

- `backend/src/main/java/com/worshiproom/storage/ObjectStorageAdapter.java` — interface with **6 methods**: `put(key, data, contentLength, contentType, metadata) → StoredObject`, `get(key) → Optional<StoredObjectStream>`, `exists(key) → boolean`, `delete(key) → boolean`, `list(prefix, maxResults) → List<StoredObjectSummary>`, `generatePresignedUrl(key, expiry) → String` (capped at `worshiproom.storage.max-presign-hours`).
- `StorageConfig.java` profile-gated wiring: dev → `LocalFilesystemStorageAdapter`, test/prod → `S3StorageAdapter` (MinIO Testcontainer in test, R2 in prod).
- `StorageKeyValidator.java` enforces `^[a-z0-9._/-]+$`, max 256 chars, no leading/trailing `/`, no `//`, no `..`. **All upload keys must conform** — 4.6b uses lowercase UUIDs (`UUID.toString()` already lowercase + hex).
- `StorageProperties.java` — POJO with `bucket`, `region`, `accessKey`, `secretKey`, `endpointUrl`, `localPath`, `devSigningSecret`, `maxPresignHours` (default 1).
- `DevStorageController.java` (`@Profile("dev")`) — serves filesystem objects via HMAC-signed `GET /dev-storage/{key}?expires=&signature=`. `LocalFilesystemStorageAdapter.computeHmac(key, expires, secret)` is the canonical signer.
- `StoredObjectStream` returns `stream()` — caller closes. Tests via MinIO Testcontainer (already in `pom.xml`).

### Posts package (3.5/3.6/3.7/4.4/4.5/4.6 — already shipped)

- `Post.java` entity at `backend/src/main/java/com/worshiproom/post/Post.java`. **NO image columns today** (verified via grep `image_url|imageUrl|image_alt|imageAlt` in `post/`). 4.6b adds `image_url VARCHAR(500) NULL` and `image_alt_text VARCHAR(500) NULL`.
- `PostService.createPost(authorId, request, idempotencyKey, requestId)` is `@Transactional`. The 9-step recipe: idempotency lookup → rate-limit → cross-field validation → HTML sanitization → crisis detection → entity build + save + flush + `entityManager.refresh()` → activity engine → response assembly → AFTER_COMMIT crisis event → idempotency cache store.
- `PostMapper.toDto()` and `toDtoList()` are the serialization seam. Both call a shared `buildDto()` helper. The mapper does NOT currently take any storage adapter — 4.6b injects `ObjectStorageAdapter` into PostMapper to build presigned URLs.
- `PostDto` is a Java record (line 3499 in OpenAPI; `backend/src/main/java/com/worshiproom/post/dto/PostDto.java` has 24 fields). 4.6b adds a 25th: `@Nullable PostImageDto image`.

- `PostExceptionHandler` is `@RestControllerAdvice(basePackages = "com.worshiproom.post")` (package-scoped, addendum item 6). 4.6b extends this advice with handlers for `ImageNotAllowedForPostTypeException`, `InvalidAltTextException`, `ImageClaimFailedException` — these new exceptions all extend `PostException`, so the existing `@ExceptionHandler(PostException.class)` already catches them via Java's most-specific dispatch.
- `PostsRateLimitService` uses Caffeine bounded bucket + bucket4j. **Cannot be parameterized** (its `Bandwidth` is per-day, hardcoded to `getRateLimit().getMaxPerDay()`). 4.6b creates a sibling `UploadRateLimitService` that copies the pattern with a per-hour bucket.

## Edge Cases & Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Storage architecture | Proxied uploads (NOT direct browser → R2 presigned PUT) | Server-side EXIF strip + resize + JPEG re-encode requires bytes flowing through the backend. Spec D1. |
| Output format | Always JPEG Q=85 (regardless of input) | Belt-and-suspenders metadata strip; simplifies rendering. Spec D2 + MPD-2. |
| Renditions | full (1920) / medium (960) / thumb (320) — long-edge resize, all stored | Spec D2. Storage cost ~3x but R2 storage is cheap; bandwidth savings on feed dominate. |
| Storage key shape | Pending: `posts/pending/{userId}/{uploadId}` (logical base; `/full.jpg` `/medium.jpg` `/thumb.jpg` hang off it). Claimed: `posts/{postId}` | Path includes userId so cross-user claim is trivially detectable. UUIDs are lowercase hex — pass `StorageKeyValidator`. Spec D17. |
| Two-step orphan strategy | MOVE-on-claim (during `PostService.createPost` transaction) | Simpler than transactional outbox. Move = copy+delete via adapter (no atomic move primitive on S3). Spec D14. |
| Alt-text policy | REQUIRED with submit-disabled enforcement when `imageUploadId` is set | Accessibility mandate. Spec D7 + W9 + W28. Whitespace-only treated as empty (`.trim().length > 0`). |
| Upload claim contract | Frontend sends `imageUploadId` (UUID), NOT URLs | Backend looks up pending record by ID + verifies userId match. Prevents cross-user claim. Spec W12. |
| Cross-user claim | Reject with 403 `IMAGE_CLAIM_FAILED` | The pending key's `{userId}` segment is compared to the authenticated user's UUID. Path-based check (no separate tracking table). Spec W12. |
| Double-claim | Reject second claim with 400 `IMAGE_CLAIM_FAILED` | After first claim's MOVE, the pending key no longer exists; the second lookup fails naturally. Spec W13. |
| Read URL strategy | Presigned-GET URLs (TTL = `STORAGE_MAX_PRESIGN_HOURS=1`) | Bucket stays private; no public-read configuration; per-URL TTL adds safety. Spec W10. |
| Read-time URL generation | At PostMapper serialization time (per request, no caching) | 50 posts × 3 renditions = 150 presigner calls per feed page. AWS SDK presigner is fast; caching deferred (W25). |
| Multipart cap | Spring config 10MB; service-level cap 5MB | Spring rejects oversized at the protocol layer (defense in depth). Service produces user-facing message. Spec D3. |
| HEIC handling | Reject client-side (extension + MIME) AND server-side (decode fails) | Two-layer enforcement. ImageIO without libheif cannot decode HEIC; rejection is automatic. Spec D5. |
| Dimension cap | 4000×4000 hard reject (after decode, before resize) | Larger images risk OOM at full color depth (4000×4000 ARGB = ~64MB). Spec D6. |
| EXIF orientation | Apply orientation BEFORE stripping (rotate pixel data) | Spec W19 — strip-first leaves sideways photos sideways. Order: read-orientation → rotate-pixels → strip-metadata → encode. |
| WebP input | Decode via TwelveMonkeys ImageIO extension; output forced to JPEG | Strip XMP + EXIF chunks via re-encode. Spec MPD-2 + W15. |
| Idempotency on upload | Reuse existing `Idempotency-Key` header pattern | Same shape as POST /posts. Caller may pass `Idempotency-Key`; same key → cached UploadResponse returned without re-uploading. Spec gate 1. |
| Cleanup task cadence | Daily `@Scheduled(cron = "0 0 3 * * ?")` (3 AM UTC) | Runs once per day; lists `posts/pending/` prefix; deletes anything older than 24h. Spec D14 + W17. |
| Logging discipline | Image bytes never logged. Alt text never logged. Storage keys logged (path identifiers). | Per addendum item 10 + spec W4. |
| Crisis detection on alt text | Concatenate `content + " " + altText` before `detectsCrisis()` | Alt text could carry crisis-language — must be detected. Spec gate 6. Single detector call, single AFTER_COMMIT event. |
| `<PostImage>` postType awareness | Decoupled — renders whenever `prayer.image` is non-null | Future post types may gain images without touching `<PostImage>`. Spec W21. |
| Lightbox uses full rendition | Yes (`prayer.image.full`) | NOT browser-upscaled medium. Spec W30. |
| Frontend remove-image | Local-only state clear, NO DELETE to backend | Cleanup cron handles unclaimed pending. Avoids race when user re-uploads. Spec D19. |
| `PostMapper` DI extension | Inject `ObjectStorageAdapter` + `StorageProperties` (for max-presign-hours) | Mapper builds presigned URLs at serialization. Acceptable because PostMapper is a `@Component` with no transactional concerns. |
| New error code in `api-error-codes.md` | Five new codes filed during execution | `INVALID_IMAGE_FORMAT`, `IMAGE_TOO_LARGE`, `IMAGE_DIMENSIONS_TOO_LARGE`, `INVALID_ALT_TEXT`, `IMAGE_CLAIM_FAILED`. `IMAGE_NOT_ALLOWED_FOR_POST_TYPE` ALSO added. |

### Step 4: ImageProcessingService

**Objective:** The pure-image pipeline — decode → orientation → dimension validation → resize → strip metadata → re-encode.

**Files to create:**
- `backend/src/main/java/com/worshiproom/upload/ImageProcessingService.java`

**Details:**

```java
package com.worshiproom.upload;

import com.drew.imaging.ImageMetadataReader;
import com.drew.metadata.Metadata;
import com.drew.metadata.exif.ExifIFD0Directory;
import org.imgscalr.Scalr;
import org.springframework.stereotype.Service;

import javax.imageio.ImageIO;
import java.awt.image.BufferedImage;
import java.io.ByteArrayInputStream;
import java.io.ByteArrayOutputStream;
import java.io.IOException;
import java.io.InputStream;

/**
 * Image processing pipeline for 4.6b uploads:
 *   1. Decode via ImageIO (JPEG/PNG/WebP via TwelveMonkeys WebP extension SPI)
 *   2. Read EXIF orientation BEFORE strip; rotate pixel data so output is upright
 *   3. Validate dimensions (≤4000×4000) AFTER decode (defense in depth — Spring max-file-size catches the byte-level cap)
 *   4. Resize to 3 renditions: full=1920, medium=960, thumb=320 (long-edge cap)
 *   5. Re-encode each rendition as JPEG Q=85 — discards all input metadata (belt-and-suspenders strip)
 *
 * <p>Returns three byte arrays in a record. Caller (UploadService) writes them to the
 * storage adapter under their respective keys.
 *
 * <p>Logs:
 *   - input format, decoded width/height, processing time (info)
 *   - decode failures or dimension rejections (warn) — NEVER the bytes themselves
 *   - alt text NEVER logged (alt text is supplied by caller and stripped on input here — this service has no alt-text view)
 */
@Service
public class ImageProcessingService {

    private static final int FULL_LONG_EDGE = 1920;
    private static final int MEDIUM_LONG_EDGE = 960;
    private static final int THUMB_LONG_EDGE = 320;
    private static final int MAX_DIMENSION = 4000;
    private static final float JPEG_QUALITY = 0.85f;

    public record ProcessedImage(byte[] full, byte[] medium, byte[] thumb) {}

    /**
     * Processes the input bytes through the full pipeline. Throws InvalidImageFormatException
     * on decode failure (covers HEIC, polyglot files, corrupt input). Throws
     * ImageDimensionsTooLargeException if the decoded image exceeds 4000×4000.
     */
    public ProcessedImage process(byte[] input, String declaredContentType) {
        try {
            // Step 1: read orientation from EXIF (BEFORE re-encode wipes it)
            int orientation = readExifOrientation(input);

            // Step 2: decode via ImageIO (SPI registers TwelveMonkeys WebP automatically)
            BufferedImage decoded;
            try (InputStream stream = new ByteArrayInputStream(input)) {
                decoded = ImageIO.read(stream);
            }
            if (decoded == null) {
                throw new InvalidImageFormatException("Decoded image is null — unsupported or corrupt format.");
            }

            // Step 3: dimension validation
            if (decoded.getWidth() > MAX_DIMENSION || decoded.getHeight() > MAX_DIMENSION) {
                throw new ImageDimensionsTooLargeException(decoded.getWidth(), decoded.getHeight());
            }

            // Step 4: apply orientation BEFORE strip (so pixel data is upright)
            BufferedImage upright = applyOrientation(decoded, orientation);

            // Step 5: resize to three renditions
            BufferedImage fullImg = Scalr.resize(upright, Scalr.Method.QUALITY, Scalr.Mode.AUTOMATIC, FULL_LONG_EDGE);
            BufferedImage mediumImg = Scalr.resize(upright, Scalr.Method.QUALITY, Scalr.Mode.AUTOMATIC, MEDIUM_LONG_EDGE);
            BufferedImage thumbImg = Scalr.resize(upright, Scalr.Method.QUALITY, Scalr.Mode.AUTOMATIC, THUMB_LONG_EDGE);

            // Step 6: encode each as JPEG Q=85
            return new ProcessedImage(
                encodeJpeg(fullImg, JPEG_QUALITY),
                encodeJpeg(mediumImg, JPEG_QUALITY),
                encodeJpeg(thumbImg, JPEG_QUALITY)
            );
        } catch (IOException e) {
            throw new InvalidImageFormatException("Image decode failed.");
        }
    }

    private int readExifOrientation(byte[] input) {
        try (InputStream stream = new ByteArrayInputStream(input)) {
            Metadata metadata = ImageMetadataReader.readMetadata(stream);
            ExifIFD0Directory dir = metadata.getFirstDirectoryOfType(ExifIFD0Directory.class);
            if (dir != null && dir.containsTag(ExifIFD0Directory.TAG_ORIENTATION)) {
                return dir.getInt(ExifIFD0Directory.TAG_ORIENTATION);
            }
        } catch (Exception e) {
            // No EXIF or metadata library failure — default to "normal" orientation (1)
        }
        return 1;
    }

    private BufferedImage applyOrientation(BufferedImage src, int orientation) {
        // EXIF orientation enum: 1=normal, 3=rotate-180, 6=rotate-90-CW, 8=rotate-90-CCW
        // 2/4/5/7 involve flips — handled identically; details in EXIF spec
        return switch (orientation) {
            case 3 -> Scalr.rotate(src, Scalr.Rotation.CW_180);
            case 6 -> Scalr.rotate(src, Scalr.Rotation.CW_90);
            case 8 -> Scalr.rotate(src, Scalr.Rotation.CW_270);
            default -> src; // 1, 2, 4, 5, 7 — treat as normal (flips not auto-corrected; rare)
        };
    }

    private byte[] encodeJpeg(BufferedImage img, float quality) throws IOException {
        // Standard ImageIO JPEG writer; quality via JPEGImageWriteParam. Uses default sRGB
        // color profile. Discards input metadata since we're re-encoding from a BufferedImage
        // (which only carries pixel data, not container metadata).
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        try (var ios = ImageIO.createImageOutputStream(baos)) {
            var writers = ImageIO.getImageWritersByFormatName("jpeg");
            if (!writers.hasNext()) throw new IOException("No JPEG writer available");
            var writer = writers.next();
            try {
                writer.setOutput(ios);
                var param = writer.getDefaultWriteParam();
                param.setCompressionMode(javax.imageio.ImageWriteParam.MODE_EXPLICIT);
                param.setCompressionQuality(quality);
                writer.write(null, new javax.imageio.IIOImage(img, null, null), param);
            } finally {
                writer.dispose();
            }
        }
        return baos.toByteArray();
    }
}
```

`UploadService.java` is the orchestrator:

```java
@Service
public class UploadService {

    private static final Logger log = LoggerFactory.getLogger(UploadService.class);
    private static final Duration PRESIGN_TTL = Duration.ofHours(1);
    private static final Set<String> ALLOWED_EXTENSIONS = Set.of("jpg", "jpeg", "png", "webp");
    private static final Set<String> HEIC_EXTENSIONS = Set.of("heic", "heif");

    private final ImageProcessingService imageProcessor;
    private final ObjectStorageAdapter storage;
    private final StorageProperties storageProperties;
    private final UploadRateLimitService rateLimitService;
    private final UploadProperties config;

    // constructor injects all five

    public UploadResponse upload(UUID userId, MultipartFile file, String requestId) {
        // 1. Rate limit
        rateLimitService.checkAndConsume(userId);

        // 2. Size check (defense in depth — Spring multipart cap is the protocol-layer safety net)
        if (file.getSize() > config.getMaxSizeBytes()) {
            throw new ImageTooLargeException(file.getSize());
        }

        // 3. Filename-based HEIC reject (extension OR original-filename suffix)
        String originalFilename = file.getOriginalFilename();
        String extension = extractExtension(originalFilename);
        if (HEIC_EXTENSIONS.contains(extension)) {
            throw new InvalidImageFormatException("HEIC images are not supported.");
        }
        // Allowed-extension hint (the actual format check is decode-based in ImageProcessingService)
        if (!ALLOWED_EXTENSIONS.contains(extension)) {
            throw new InvalidImageFormatException("Unsupported file extension: " + extension);
        }

        // 4. Read bytes (within MultipartFile limit, already validated)
        byte[] inputBytes;
        try {
            inputBytes = file.getBytes();
        } catch (IOException e) {
            throw new InvalidImageFormatException("Failed to read uploaded bytes.");
        }

        // 5. Process — decode + dimension validate + orientation + resize + re-encode
        long start = System.currentTimeMillis();
        ImageProcessingService.ProcessedImage processed = imageProcessor.process(inputBytes, file.getContentType());
        long elapsed = System.currentTimeMillis() - start;

        // 6. Store renditions at posts/pending/{userId}/{uploadId}/{rendition}.jpg
        UUID uploadId = UUID.randomUUID();
        String pendingBase = String.format("posts/pending/%s/%s",
                userId.toString(), uploadId.toString()); // both UUIDs are lowercase hex — pass StorageKeyValidator
        Map<String, String> metadata = Map.of(
                "user-id", userId.toString(),
                "upload-id", uploadId.toString(),
                "uploaded-at", Instant.now().toString()
        );
        try {
            storage.put(pendingBase + "/full.jpg", new ByteArrayInputStream(processed.full()),
                    processed.full().length, "image/jpeg", metadata);
            storage.put(pendingBase + "/medium.jpg", new ByteArrayInputStream(processed.medium()),
                    processed.medium().length, "image/jpeg", metadata);
            storage.put(pendingBase + "/thumb.jpg", new ByteArrayInputStream(processed.thumb()),
                    processed.thumb().length, "image/jpeg", metadata);
        } catch (RuntimeException e) {
            // If a partial put leaks, the cleanup cron sweeps anything in posts/pending older than TTL.
            log.warn("uploadStorageFailure userId={} uploadId={} requestId={}",
                    userId, uploadId, requestId);
            throw e; // bubbles to StorageExceptionHandler → 502 STORAGE_FAILURE
        }

        // 7. Build presigned-GET URLs
        String fullUrl = storage.generatePresignedUrl(pendingBase + "/full.jpg", PRESIGN_TTL);
        String mediumUrl = storage.generatePresignedUrl(pendingBase + "/medium.jpg", PRESIGN_TTL);
        String thumbUrl = storage.generatePresignedUrl(pendingBase + "/thumb.jpg", PRESIGN_TTL);

        // 8. Log — IDs only, no bytes, no alt text (the controller passes no alt text here anyway)
        log.info("uploadCompleted userId={} uploadId={} extension={} sizeBytes={} elapsedMs={} requestId={}",
                userId, uploadId, extension, file.getSize(), elapsed, requestId);

        return new UploadResponse(uploadId.toString(), fullUrl, mediumUrl, thumbUrl);
    }

    /**
     * Called from PostService.createPost during the image-claim flow.
     * Verifies pending exists + belongs to user; copies pending → claimed; deletes pending.
     * Returns the claimed base path (e.g., "posts/{postId}").
     *
     * Throws ImageClaimFailedException for: missing pending, cross-user claim, double-claim.
     */
    public String claimUpload(UUID userId, UUID uploadId, UUID postId) {
        String pendingBase = String.format("posts/pending/%s/%s", userId.toString(), uploadId.toString());
        String claimedBase = String.format("posts/%s", postId.toString());

        // Verify pending full rendition exists (covers cross-user too — different userId in the path → key absent for THIS user)
        if (!storage.exists(pendingBase + "/full.jpg")) {
            throw new ImageClaimFailedException();
        }

        // Move (copy + delete) all three renditions
        for (String rendition : List.of("full.jpg", "medium.jpg", "thumb.jpg")) {
            String src = pendingBase + "/" + rendition;
            String dst = claimedBase + "/" + rendition;
            // Copy: read pending stream, write to claimed key
            try (var streamOpt = storage.get(src)) {
                if (streamOpt.isEmpty()) {
                    throw new ImageClaimFailedException();
                }
                StoredObjectStream stream = streamOpt.get();
                // Read all bytes (renditions are bounded by 5MB, fits comfortably in memory)
                byte[] bytes;
                try (InputStream in = stream.stream()) {
                    bytes = in.readAllBytes();
                }
                storage.put(dst, new ByteArrayInputStream(bytes), bytes.length, "image/jpeg",
                        Map.of("post-id", postId.toString()));
            } catch (IOException e) {
                throw new ImageClaimFailedException();
            }
        }

        // Delete pending (idempotent — safe to call on already-deleted)
        for (String rendition : List.of("full.jpg", "medium.jpg", "thumb.jpg")) {
            storage.delete(pendingBase + "/" + rendition);
        }

        return claimedBase;
    }

    private String extractExtension(String filename) {
        if (filename == null) return "";
        int dot = filename.lastIndexOf('.');
        return dot < 0 ? "" : filename.substring(dot + 1).toLowerCase();
    }
}
```

`UploadIdempotencyService.java` mirrors `PostsIdempotencyService.java`:

- Keyed by `(userId, idempotencyKey)`; value `(bodyHash, UploadResponse)`

- For multipart uploads, `bodyHash` is the SHA-256 (or DJB2) of the file bytes — computed in the controller

`UploadController.java`:

```java
@RestController
@RequestMapping("/api/v1/uploads")
public class UploadController {

    private static final Logger log = LoggerFactory.getLogger(UploadController.class);

    private final UploadService uploadService;
    private final UploadIdempotencyService idempotencyService;

    public UploadController(UploadService uploadService, UploadIdempotencyService idempotencyService) {
        this.uploadService = uploadService;
        this.idempotencyService = idempotencyService;
    }

    @PostMapping(value = "/post-image", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
    public ResponseEntity<UploadResponse> uploadPostImage(
            @AuthenticationPrincipal AuthenticatedUser principal,
            @RequestParam("file") MultipartFile file,
            @RequestHeader(name = "Idempotency-Key", required = false) String idempotencyKey,
            HttpServletRequest request
    ) {
        String requestId = MDC.get("requestId");

        // Idempotency lookup — body hash is the file size + first-N-bytes hash (cheap, sufficient)
        int bodyHash = computeBodyHash(file);
        Optional<UploadResponse> cached =
                idempotencyService.lookup(principal.userId(), idempotencyKey, bodyHash);
        if (cached.isPresent()) {
            return ResponseEntity.ok(cached.get());
        }

        UploadResponse response = uploadService.upload(principal.userId(), file, requestId);
        idempotencyService.store(principal.userId(), idempotencyKey, bodyHash, response);
        return ResponseEntity.ok(response);
    }

    private int computeBodyHash(MultipartFile file) {
        // Cheap idempotency hash: filesize + first 4096 bytes of content
        try {
            byte[] head = file.getInputStream().readNBytes(4096);
            return Long.hashCode(file.getSize()) ^ Arrays.hashCode(head);
        } catch (IOException e) {
            return 0; // fallback — idempotency degrades to no-op
        }
    }
}
```

```java
        Instant cutoff = Instant.now().minus(Duration.ofHours(config.getPendingTtlHours()));
        log.info("pendingCleanupStarted cutoff={}", cutoff);

        int deleted = 0;
        int errors = 0;

        List<StoredObjectSummary> summaries = storage.list("posts/pending/", 1000);
        for (StoredObjectSummary summary : summaries) {
            // StoredObjectSummary should expose lastModified; if absent, fall back to skip-cleanup
            // (4.6b guarantees the metadata `uploaded-at` was attached — adapter's list() may
            // not return metadata; if so, fetch the object to read it. Plan accepts the

            Instant lastModified = summary.lastModified(); // verify field name during execution

                try {
                    if (storage.delete(summary.key())) {
                        deleted++;
                    }
                } catch (RuntimeException e) {
                    errors++;
                    log.warn("pendingCleanupItemFailed key={} error={}", summary.key(), e.getMessage());
                }
            }
        }
        log.info("pendingCleanupCompleted deleted={} errors={}", deleted, errors);
    }
}
```

```java
    @ExceptionHandler(UploadRateLimitException.class)

    public ResponseEntity<ProxyError> handleUpload(UploadException ex) {
        var requestId = MDC.get("requestId");
        log.info("Upload-domain rejection: code={} message={}", ex.getCode(), ex.getMessage());
        return ResponseEntity
                .status(ex.getStatus())
                .body(ProxyError.of(ex.getCode(), ex.getMessage(), requestId));
    }

    @ExceptionHandler(MaxUploadSizeExceededException.class)
    public ResponseEntity<ProxyError> handleMaxUploadSize(MaxUploadSizeExceededException ex) {
        // Spring's protocol-layer 10MB cap (multipart) hits before our 5MB service-layer cap.
```

```typescript
export async function uploadImage(file: File): Promise<UploadResponse> {
  const formData = new FormData()
  formData.append('file', file)

  const token = getStoredToken()
  if (!token) throw new AnonymousWriteAttemptError()

  const baseUrl = import.meta.env.VITE_API_BASE_URL ?? ''
  const response = await fetch(`${baseUrl}/api/v1/uploads/post-image`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${token}` },
    body: formData,
  })

  if (!response.ok) {
    const errorBody = await response.json().catch(() => ({}))
    throw new ApiError(response.status, errorBody?.code ?? 'UNKNOWN', errorBody?.message ?? 'Upload failed', errorBody?.requestId)
  }

  return response.json() as Promise<UploadResponse>
}
```

```tsx
interface ImageUploadProps {
  onUploadSuccess: (uploadId: string, mediumUrl: string) => void
  onUploadRemoved: () => void

  onAltTextChange: (value: string) => void
  helperText?: string

export function ImageUpload({ onUploadSuccess, onUploadRemoved, altText, onAltTextChange, helperText }: ImageUploadProps) {
  const [state, setState] = useState<'idle' | 'uploading' | 'preview' | 'error'>('idle')
  const [previewUrl, setPreviewUrl] = useState<string | null>(null)
  const [errorMessage, setErrorMessage] = useState<string | null>(null)
  const [isDragOver, setIsDragOver] = useState(false)
  const fileInputRef = useRef<HTMLInputElement>(null)

  const ACCEPT_MIME = 'image/jpeg,image/png,image/webp'
  const MAX_SIZE_BYTES = 5 * 1024 * 1024
  const HEIC_EXTENSIONS = new Set(['heic', 'heif'])

  const handleFile = useCallback(async (file: File) => {
    // Client-side HEIC reject
    const ext = file.name.split('.').pop()?.toLowerCase() ?? ''
    if (HEIC_EXTENSIONS.has(ext) || file.type.includes('heic') || file.type.includes('heif')) {
      setState('error')
      setErrorMessage("HEIC images aren't supported yet. Open the Photos app, share the image, and choose JPEG.")
      return
    }
    // Client-side size reject
    if (file.size > MAX_SIZE_BYTES) {
      setState('error')
      setErrorMessage('Image is larger than 5 MB. Try a smaller version.')
      return
    }
    setState('uploading')
    setErrorMessage(null)
    try {
      const result = await prayerWallApi.uploadImage(file)
      setPreviewUrl(result.mediumUrl)
      setState('preview')
      onUploadSuccess(result.uploadId, result.mediumUrl)
    } catch (e) {
      setState('error')
      setErrorMessage(e instanceof ApiError ? e.message : 'Upload failed. Please try again.')
    }
  }, [onUploadSuccess])

  // Drag-drop handlers
  const handleDragOver = useCallback((e: React.DragEvent) => {
    e.preventDefault()
    setIsDragOver(true)
  }, [])
  const handleDragLeave = useCallback(() => setIsDragOver(false), [])
  const handleDrop = useCallback((e: React.DragEvent) => {
    e.preventDefault()
    setIsDragOver(false)
    const file = e.dataTransfer.files[0]
    if (file) handleFile(file)
  }, [handleFile])

  const handleRemove = useCallback(() => {
    setPreviewUrl(null)
    setState('idle')
    setErrorMessage(null)
    onUploadRemoved()  // parent clears alt text + uploadId state
  }, [onUploadRemoved])

  // Render: drag-drop wrapper + state-specific UI
  return (
    <div
      onDragOver={handleDragOver}
      onDragLeave={handleDragLeave}
      onDrop={handleDrop}
      className={cn(
        'mt-3 rounded-lg border border-white/10 bg-white/[0.04] p-3',
        isDragOver && 'border-violet-400/50 bg-violet-500/[0.06]'
      )}
    >
      {/* Idle state: 'Add a photo' button + helper text */}
      {state === 'idle' && (
        <>
          <button
            type="button"
            onClick={() => fileInputRef.current?.click()}
            className="inline-flex items-center gap-2 rounded-full bg-white/[0.07] border border-white/[0.12] px-4 py-2 text-sm text-white hover:bg-white/[0.12] min-h-[44px]"
          >
            <ImagePlus className="h-4 w-4" aria-hidden="true" />
            Add a photo
          </button>
          <input
            ref={fileInputRef}
            type="file"
            accept={ACCEPT_MIME}
            className="sr-only"
            onChange={(e) => {
              const file = e.target.files?.[0]
              if (file) handleFile(file)
            }}
          />
          {helperText && (
            <p className="mt-2 text-xs text-white/60">{helperText}</p>
          )}
          {isDragOver && (
            <p className="mt-2 text-sm font-medium text-violet-200" role="status">Drop image here</p>
          )}
        </>
      )}

      {/* Uploading state: shimmer skeleton */}
      {state === 'uploading' && (
        <div className="flex items-center gap-3" role="status" aria-live="polite">
          <div className="h-20 w-20 animate-pulse rounded-md bg-white/10" />
          <p className="text-sm text-white/80">Uploading…</p>
        </div>
      )}

      {/* Preview state: image + Remove + alt-text input */}
      {state === 'preview' && previewUrl && (
        <>
          <div className="flex items-start gap-3">
            <img
              src={previewUrl}
              alt={altText || 'Uploaded image preview'}
              className="h-32 w-32 rounded-md object-cover"
            />
            <button
              type="button"
              onClick={handleRemove}
              className="inline-flex items-center gap-1 rounded-full px-3 py-1.5 text-sm text-white/80 hover:bg-white/10 min-h-[44px]"
              aria-label="Remove image"
            >
              <X className="h-4 w-4" aria-hidden="true" />
              Remove
            </button>
          </div>
          <label className="mt-3 block">
            <span className="text-sm font-medium text-white/80">
              Describe this image for screen readers
            </span>
            <input
              type="text"
              value={altText}
              onChange={(e) => onAltTextChange(e.target.value)}
              placeholder="A short description of what's in the photo"
              maxLength={500}
              required
              aria-required="true"
              className="mt-1 w-full rounded-md border border-white/10 bg-white/[0.04] px-3 py-2 text-white placeholder:text-white/50 focus:border-violet-400/60 focus:ring-2 focus:ring-violet-400/30 focus:outline-none"
            />
            <span className="mt-1 block text-xs text-white/60">
              Required for accessibility — screen readers will read this aloud.
            </span>
          </label>
        </>
      )}

      {/* Error state: gentle error + retry */}
      {state === 'error' && errorMessage && (
        <div role="alert" className="rounded-md border border-red-400/30 bg-red-950/30 p-3 text-sm text-red-100">
          {errorMessage}
          <button
            type="button"
            onClick={() => setState('idle')}
            className="ml-2 underline hover:no-underline"
          >
            Try again
          </button>
        </div>
      )}
    </div>
  )
}
```
