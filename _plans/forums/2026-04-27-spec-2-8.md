## Architecture Context

### Backend (Phase 2 deliverables, all merged on `forums-wave-continued`)

Three pure services with zero Spring dependencies, instantiable via `new ServiceClass()`:

```java
// backend/src/main/java/com/worshiproom/activity/
class FaithPointsService {
    public FaithPointsResult calculate(Set<ActivityType> activities, int currentTotalPoints);
}

class StreakService {
    public StreakResult updateStreak(StreakStateData currentState, LocalDate today);
    public boolean isFreeRepairAvailable(LocalDate lastFreeRepairDate, LocalDate currentWeekStart);
}

class BadgeService {
    public BadgeResult checkBadges(BadgeCheckContext context, Set<String> alreadyEarnedBadgeIds);
}
```

Supporting records (all in `com.worshiproom.activity` or `com.worshiproom.activity.dto`):

- `FaithPointsResult(int basePoints, int activityCount, int pointsEarned, int totalPoints, int currentLevel, String currentLevelName, int pointsToNextLevel, boolean levelUp, MultiplierTier multiplierTier)`
- `StreakStateData(int currentStreak, int longestStreak, LocalDate lastActiveDate)` with `static fresh()` factory
- `StreakResult(StreakStateData newState, StreakTransition transition, int previousStreak, boolean shouldCaptureForRepair)`
- `StreakTransition` enum: `FIRST_EVER`, `SAME_DAY`, `INCREMENT`, `RESET`
- `BadgeCheckContext(int currentStreak, int longestStreak, int currentLevel, int previousLevel, Set<ActivityType> todayActivities, ActivityCountsSnapshot activityCounts, int friendCount, boolean allActivitiesWereTrueBefore, List<ReadingPlanProgress> readingPlanProgress, BibleProgressSnapshot bibleProgress, List<MeditationSession> meditationHistory, GratitudeEntriesSnapshot gratitudeEntries, LocalVisitsSnapshot localVisits, List<ListeningSession> listeningHistory)`
- `BadgeResult(List<String> newlyEarnedBadgeIds, List<BadgeDefinition> newlyEarnedDefinitions)`
- `LevelInfo` and `LevelThresholds.levelForPoints(int points)` for level computation

The composition algorithm `ActivityService.recordActivity` does (Spec 2.6):

1. Branch on `firstTimeToday`:
   - If FALSE (same-day repeat): zero deltas, `SAME_DAY` transition, no badge check, no level up.
   - If TRUE: compute `oldSet` and `newSet` activity sets; call `faithPointsService.calculate(oldSet, 0)` and `faithPointsService.calculate(newSet, 0)`; delta = `newSet.pointsEarned - oldSet.pointsEarned`; new total = current + delta; compute new level via `LevelThresholds.levelForPoints`; call `streakService.updateStreak(streakData, today)`; build `BadgeCheckContext` with new streak/level/today's activities; call `badgeService.checkBadges(context, alreadyEarned)`.
2. Repair eligibility = `lastFreeRepairDate == null || lastFreeRepairDate < currentWeekStart`.

The drift test composes these three pure services directly. It does NOT call `ActivityService` (which is transactional, requires DB and Spring).

### Composition algorithm replicated in the harness (per spec Decision 5)

```text
runScenario(input) -> output:
  if input.todaysActivitiesBefore[input.newActivityType] === true:
    // Same-day-repeat early-return path (Decision 6)
    return {
      pointsEarned: 0,
      newTotalPoints: input.currentTotalPoints,
      newCurrentLevel: input.currentLevel,
      levelUp: false,
      newCurrentStreak: input.currentStreak,
      newLongestStreak: input.longestStreak,
      streakTransition: "SAME_DAY",
      newBadgeIds: [],
      isFreeRepairAvailableAfter: pureFreeRepairCheck(input.lastFreeRepairDate, input.currentWeekStart),
    }

  // First-time-today path
  oldSet = activeActivities(input.todaysActivitiesBefore)
  newSet = oldSet ∪ {input.newActivityType}

  oldPoints = calculateDailyPoints(oldSet) or FaithPointsService.calculate(oldSet, 0).pointsEarned
  newPoints = calculateDailyPoints(newSet) or FaithPointsService.calculate(newSet, 0).pointsEarned
  pointsDelta = newPoints - oldPoints
  newTotalPoints = input.currentTotalPoints + pointsDelta

  levelInfo = getLevelForPoints(newTotalPoints) or LevelThresholds.levelForPoints(newTotalPoints)
  levelUp = levelInfo.level > input.currentLevel

  streakResult = StreakService.updateStreak(streakDataFromInput, today)  // backend
            OR  updateStreak(input.today, streakDataFromInput)            // frontend
  // Frontend lacks a transition enum — derive in the harness:
  //   oldLastActive == null   -> FIRST_EVER
  //   oldLastActive == today  -> SAME_DAY (only on same-day path; not reachable here since branch handled)
  //   oldLastActive == today.minusDays(1) -> INCREMENT
  //   else -> RESET

  context = buildBadgeContext(newStreak, levelInfo.level, input.currentLevel, newSet, input.activity counts and data sources)
  newBadgeIds = BadgeService.checkBadges(context, alreadyEarned).newlyEarnedBadgeIds  // backend
            OR  checkForNewBadges(context, alreadyEarnedAsRecord)                      // frontend
  newBadgeIds = newBadgeIds.sort()  // alphabetical for stable comparison

  isFreeRepairAvailableAfter = lastFreeRepairDate == null || lastFreeRepairDate < currentWeekStart

  return { pointsEarned: pointsDelta, newTotalPoints, newCurrentLevel: levelInfo.level, levelUp,
           newCurrentStreak, newLongestStreak, streakTransition, newBadgeIds, isFreeRepairAvailableAfter }
```

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Fixture format | JSON (`_test_fixtures/activity-engine-scenarios.json`) | Per spec Decision 1 — both Vitest and Jackson parse natively, no new deps. |

| Number of scenarios | 36 (target; minimum 30 per acceptance criterion) | Per spec Decision 2. Categories: 3 first-time, 4 streak continuation, 3 streak boundaries, 5 level-up, 4 multiplier tiers, 5 activity-milestone badges, 4 data-source badges, 2 badge repeatability, 3 idempotency, 3 repair eligibility = 36. |
| Backend invocation surface | The three pure services directly (`FaithPointsService`, `StreakService`, `BadgeService`) | Per spec Divergence 2 — calling `ActivityService.recordActivity` would require DB, transactions, principal context. The drift test wants pure calculation parity. |
| Same-day-repeat behavior | Early-return in harness on both sides | Per spec Decision 6 — replicates production code's idempotency check. Returns zero deltas, `SAME_DAY` transition. |
| `streakTransition` derivation on frontend | Inline in harness (frontend has no enum) | Per spec Decision 8. Logic: `lastActiveDate == null` → `FIRST_EVER`; `lastActiveDate == today.minus(1)` → `INCREMENT`; same-day path → `SAME_DAY`; else → `RESET`. |
| Badge ID ordering | Sorted alphabetically in BOTH fixture and harness output | Per spec Decision 7 — badges are a set, not an ordered list. Stable comparison without depending on category iteration order. |
| Out-of-scope categories | Grace days, grief pause, repair APPLICATION, welcome badge, challenge badges | Per spec Divergence 1 + Out of Scope. Features not implemented on either side, or manually granted (welcome). |
| BadgeCheckContext data sources on backend | Fixture provides them; backend uses them | Per spec Divergence 3 — production backend currently passes empties (no persistence yet for these 6 sources), but the drift test exercises calculation parity. Future specs that add backend persistence won't need fixture changes. |
| Where to compute expected values | One-shot generator: temporary Java main method calling the three pure services with fixture inputs, output pasted into JSON, generator file deleted | Per spec "Notes for Eric" — avoids circular self-validation. The generator is run ONCE during execution; the test then asserts against the COMMITTED expected values. Future code changes that drift from the implementation will fail the committed expected values, surfacing the drift. |
| One-shot generator location | `backend/src/test/java/com/worshiproom/activity/_DriftFixtureGenerator.java` (note leading underscore for clarity), then DELETE after use | Underscore prefix sorts it next to the test files but signals "scaffolding"; deletion at end of step 7 keeps the committed test surface clean per spec Files NOT to modify list. |

| Repair eligibility check | Inline pure 2-line check in both harnesses (`lastFreeRepairDate == null \|\| lastFreeRepairDate < currentWeekStart`) | Per spec Recon Fact C — frontend `isFreeRepairAvailable()` reads localStorage; the harness inlines the pure check using fixture inputs to avoid bypassing localStorage AND staying parallel to backend's `StreakService.isFreeRepairAvailable(LocalDate, LocalDate)`. |

1. Append a new entry to the `scenarios` array in the relevant fixture file.

package com.worshiproom.activity;

import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.fasterxml.jackson.databind.json.JsonMapper;
import com.fasterxml.jackson.datatype.jsr310.JavaTimeModule;
import com.worshiproom.activity.constants.LevelThresholds;

import org.junit.jupiter.params.ParameterizedTest;
import org.junit.jupiter.params.provider.MethodSource;

import java.io.File;
import java.io.IOException;
import java.time.LocalDate;
import java.time.LocalDateTime;
import java.util.*;
import java.util.stream.Stream;
import java.util.stream.StreamSupport;

import static org.assertj.core.api.Assertions.assertThat;

class DriftDetectionTest {

    private static final ObjectMapper MAPPER = JsonMapper.builder()
        .addModule(new JavaTimeModule())
        .build();

    private final FaithPointsService faithPointsService = new FaithPointsService();
    private final StreakService streakService = new StreakService();
    private final BadgeService badgeService = new BadgeService();

    @ParameterizedTest(name = "{0}")
    @MethodSource("allScenarios")
    void scenarioMatchesBackendComputation(DriftScenario scenario) {
        DriftResult actual = runBackend(scenario.input());
        assertThat(actual)
            .as("Drift detected in scenario: %s — %s", scenario.id(), scenario.description())
            .isEqualTo(scenario.expected());
    }

    static Stream<DriftScenario> allScenarios() throws IOException {
        // Path is relative to the backend module's working directory (where mvn runs).
        // The repo root sits at ../ from backend/.
        File fixtureFile = new File("../_test_fixtures/activity-engine-scenarios.json");
        JsonNode root = MAPPER.readTree(fixtureFile);
        JsonNode scenariosNode = root.get("scenarios");
        return StreamSupport.stream(scenariosNode.spliterator(), false)
            .map(node -> {
                try {
                    return MAPPER.treeToValue(node, DriftScenario.class);
                } catch (Exception e) {
                    throw new RuntimeException("Failed to deserialize scenario: " + node.path("id").asText(), e);
                }
            });
    }

    private DriftResult runBackend(DriftInput input) {
        // Same-day-repeat early return (Spec Decision 6).
        if (input.todaysActivitiesBefore().getOrDefault(input.newActivityType(), false)) {
            return new DriftResult(
                0,
                input.currentTotalPoints(),
                input.currentLevel(),
                false,
                input.currentStreak(),
                input.longestStreak(),
                "SAME_DAY",
                List.of(),
                isFreeRepairAvailable(input)
            );
        }

        // First-time-today path. Build oldSet and newSet of ActivityType.
        Set<ActivityType> oldSet = activeActivities(input.todaysActivitiesBefore());
        Set<ActivityType> newSet = EnumSet.copyOf(oldSet);
        newSet.add(ActivityType.fromWireValue(input.newActivityType()));

        // Faith points delta.
        int oldPoints = oldSet.isEmpty()
            ? 0
            : faithPointsService.calculate(oldSet, 0).pointsEarned();
        int newPoints = faithPointsService.calculate(newSet, 0).pointsEarned();
        int pointsDelta = newPoints - oldPoints;
        int newTotalPoints = input.currentTotalPoints() + pointsDelta;

        // Level computation.
        LevelThresholds.LevelInfo levelInfo = LevelThresholds.levelForPoints(newTotalPoints);
        boolean levelUp = levelInfo.level() > input.currentLevel();

        // Streak update.
        StreakStateData currentStreakState = new StreakStateData(
            input.currentStreak(),
            input.longestStreak(),
            input.lastActiveDate()
        );
        StreakResult streakResult = streakService.updateStreak(currentStreakState, input.today());

        // Badge check — build context from new state and fixture data sources.
        // Increment activity count for the recorded type (per the production incrementCount call).
        Map<String, Integer> incrementedCounts = new HashMap<>(input.activityCounts());
        ActivityType type = ActivityType.fromWireValue(input.newActivityType());
        // Use the wire value as the count-key — matches the frontend ActivityCounts shape and the
        // backend ActivityCountsSnapshot field naming (camelCase, e.g., "pray", "prayerWall").
        // The CountType enum is checked first; activities like "mood" don't have a counter.
        var countTypeOpt = CountType.fromWireValue(input.newActivityType());
        if (countTypeOpt.isPresent()) {
            String key = countTypeOpt.get().wireValue();
            incrementedCounts.merge(key, 1, Integer::sum);
        }

        ActivityCountsSnapshot countsSnapshot = new ActivityCountsSnapshot(
            incrementedCounts.getOrDefault("pray", 0),
            incrementedCounts.getOrDefault("journal", 0),
            incrementedCounts.getOrDefault("meditate", 0),
            incrementedCounts.getOrDefault("listen", 0),
            incrementedCounts.getOrDefault("prayerWall", 0),
            incrementedCounts.getOrDefault("readingPlan", 0),
            incrementedCounts.getOrDefault("gratitude", 0),
            incrementedCounts.getOrDefault("reflection", 0),
            input.encouragementsSent(),
            input.fullWorshipDays(),
            input.challengesCompleted(),
            input.intercessionCount(),
            input.bibleChaptersRead(),
            input.prayerWallPosts()
        );

        BadgeCheckContext context = new BadgeCheckContext(
            streakResult.newState().currentStreak(),
            streakResult.newState().longestStreak(),
            levelInfo.level(),
            input.currentLevel(),
            EnumSet.copyOf(newSet),
            countsSnapshot,
            input.friendCount(),
            false, // allActivitiesWereTrueBefore — production passes false; idempotency already handled above
            buildReadingPlans(input),
            buildBibleProgress(input),
            buildMeditationHistory(input),
            buildGratitudeEntries(input),
            new LocalVisitsSnapshot(input.localVisitsTotal()),
            buildListeningHistory(input)
        );

        Set<String> alreadyEarned = new HashSet<>(input.alreadyEarnedBadgeIds());
        BadgeResult badgeResult = badgeService.checkBadges(context, alreadyEarned);

        List<String> sortedBadges = new ArrayList<>(badgeResult.newlyEarnedBadgeIds());
        Collections.sort(sortedBadges);

        return new DriftResult(
            pointsDelta,
            newTotalPoints,
            levelInfo.level(),
            levelUp,
            streakResult.newState().currentStreak(),
            streakResult.newState().longestStreak(),
            streakResult.transition().name(),
            sortedBadges,
            isFreeRepairAvailable(input)
        );
    }

    private Set<ActivityType> activeActivities(Map<String, Boolean> map) {
        Set<ActivityType> result = EnumSet.noneOf(ActivityType.class);
        for (var entry : map.entrySet()) {
            if (Boolean.TRUE.equals(entry.getValue())) {
                result.add(ActivityType.fromWireValue(entry.getKey()));
            }
        }
        return result;
    }

    private boolean isFreeRepairAvailable(DriftInput input) {
        return streakService.isFreeRepairAvailable(input.lastFreeRepairDate(), input.currentWeekStart());
    }

    // Helper builders — translate fixture's simple shapes into the Phase 2 DTO types.
    // For the 36 scenarios shipped in this spec, only data-source-dependent badge scenarios
    // populate these collections; the rest pass empty.

    private List<ReadingPlanProgress> buildReadingPlans(DriftInput input) {
        return input.readingPlanProgress().stream()
            .map(p -> new ReadingPlanProgress(p.planSlug(),
                p.completedAt() == null ? null : LocalDateTime.parse(p.completedAt())))
            .toList();
    }

    private BibleProgressSnapshot buildBibleProgress(DriftInput input) {
        Map<String, Set<Integer>> map = new HashMap<>();
        input.bibleProgress().forEach((book, list) -> map.put(book, new HashSet<>(list)));
        return new BibleProgressSnapshot(map);
    }

    private List<MeditationSession> buildMeditationHistory(DriftInput input) {
        return input.meditationHistory().stream()
            .map(m -> new MeditationSession(LocalDateTime.parse(m.occurredAt()), m.durationSeconds()))
            .toList();
    }

    private GratitudeEntriesSnapshot buildGratitudeEntries(DriftInput input) {
        return new GratitudeEntriesSnapshot(
            input.gratitudeEntryDates().stream().map(LocalDate::parse).toList()
        );
    }

    private List<ListeningSession> buildListeningHistory(DriftInput input) {
        return input.listeningHistory().stream()
            .map(l -> new ListeningSession(LocalDateTime.parse(l.occurredAt()), l.durationSeconds()))
            .toList();
    }
}

// Seed localStorage from the input data sources that frontend `checkForNewBadges` reads.
function seedLocalStorageFromInput(input: DriftInput): void {
  // Reading plan progress: the badge engine reads { [slug]: { completedAt: string | null } }.
  const planMap: Record<string, { completedAt: string | null }> = {};
  for (const p of input.readingPlanProgress) {
    planMap[p.planSlug] = { completedAt: p.completedAt };
  }
  localStorage.setItem('wr_reading_plan_progress', JSON.stringify(planMap));

  // Bible progress: { [book]: number[] }
  localStorage.setItem('wr_bible_progress', JSON.stringify(input.bibleProgress));

  // Gratitude entries: badge engine reads via getGratitudeEntries() (from gratitude-storage).
  // The storage shape is GratitudeEntry[] with a `date` field. Build a minimal shape.
  const gratitudeEntries = input.gratitudeEntryDates.map(date => ({ date, items: [] }));
  localStorage.setItem('wr_gratitude_entries', JSON.stringify(gratitudeEntries));

  // Meditation history: badge engine reads via getMeditationHistory().
  // Shape: MeditationSession[] with sessionDate (string YYYY-MM-DD), durationSeconds, type.
  const meditationSessions = input.meditationHistory.map(m => ({
    sessionDate: m.occurredAt.slice(0, 10),
    durationSeconds: m.durationSeconds,
    type: 'breathing',
  }));
  localStorage.setItem('wr_meditation_history', JSON.stringify(meditationSessions));

  // Local visits: badge engine reads via getUniqueVisitedPlaces().
  // Shape: LocalVisit[]. To reach `localVisitsTotal` unique places, fabricate placeId 1..N.
  const visits = Array.from({ length: input.localVisitsTotal }, (_, i) => ({
    placeId: `place-${i}`,
    visitedAt: input.today,
    name: `Place ${i}`,
  }));
  localStorage.setItem('wr_local_visits', JSON.stringify(visits));

  // Listening history: not used by any of the 36 scenarios for badges, but seed the key with empty array.
  localStorage.setItem('wr_listening_history', JSON.stringify(input.listeningHistory));
}

const ACTIVITY_KEYS: ActivityType[] = [
  'mood', 'pray', 'listen', 'prayerWall', 'readingPlan',
  'meditate', 'journal', 'gratitude', 'reflection',
  'challenge', 'localVisit', 'devotional',
];

function activeActivities(map: Record<string, boolean>): Set<ActivityType> {
  const result = new Set<ActivityType>();
  for (const key of ACTIVITY_KEYS) {
    if (map[key]) result.add(key);
  }
  return result;
}

function buildDailyActivities(active: Set<ActivityType>): DailyActivities {
  const activities = freshDailyActivities();
  for (const key of active) {
    (activities as unknown as Record<ActivityType, boolean>)[key] = true;
  }
  return activities;
}

function deriveStreakTransitionFrontend(
  oldLastActive: string | null,
  today: string,
): 'FIRST_EVER' | 'SAME_DAY' | 'INCREMENT' | 'RESET' {
  if (oldLastActive === null) return 'FIRST_EVER';
  if (oldLastActive === today) return 'SAME_DAY';
  // Compute today.minus(1 day) as YYYY-MM-DD.
  const todayDate = new Date(today + 'T00:00:00');
  const yesterday = new Date(todayDate);
  yesterday.setDate(yesterday.getDate() - 1);
  const yesterdayStr = yesterday.toISOString().slice(0, 10);
  if (oldLastActive === yesterdayStr) return 'INCREMENT';
  return 'RESET';
}

function pureFreeRepairAvailable(
  lastFreeRepairDate: string | null,
  currentWeekStart: string,
): boolean {
  if (lastFreeRepairDate === null) return true;
  return lastFreeRepairDate < currentWeekStart;
}

function runFrontend(input: DriftInput): DriftExpected {
  // Same-day-repeat early return (Spec Decision 6).
  if (input.todaysActivitiesBefore[input.newActivityType] === true) {
    return {
      pointsEarned: 0,
      newTotalPoints: input.currentTotalPoints,
      newCurrentLevel: input.currentLevel,
      levelUp: false,
      newCurrentStreak: input.currentStreak,
      newLongestStreak: input.longestStreak,
      streakTransition: 'SAME_DAY',
      newBadgeIds: [],
      isFreeRepairAvailableAfter: pureFreeRepairAvailable(
        input.lastFreeRepairDate,
        input.currentWeekStart,
      ),
    };
  }

  const oldSet = activeActivities(input.todaysActivitiesBefore);
  const newSet = new Set(oldSet);
  newSet.add(input.newActivityType);

  const oldDaily = oldSet.size === 0
    ? { points: 0, multiplier: 1 }
    : calculateDailyPoints(buildDailyActivities(oldSet));
  const newDaily = calculateDailyPoints(buildDailyActivities(newSet));
  const pointsDelta = newDaily.points - oldDaily.points;
  const newTotalPoints = input.currentTotalPoints + pointsDelta;

  const levelInfo = getLevelForPoints(newTotalPoints);
  const levelUp = levelInfo.level > input.currentLevel;

  const currentStreakData: StreakData = {
    currentStreak: input.currentStreak,
    longestStreak: input.longestStreak,
    lastActiveDate: input.lastActiveDate,
  };
  const newStreak = updateStreak(input.today, currentStreakData);
  const streakTransition = deriveStreakTransitionFrontend(input.lastActiveDate, input.today);

  // Build badge context. Increment activity count for the recorded type.
  const incrementedCounts: ActivityCounts = { ...input.activityCounts };
  // Only increment if the activity type maps to an ActivityCounts field. The mapping mirrors
  // the backend CountType enum: mood/devotional/challenge/localVisit have NO counter.
  const COUNT_KEYS: Record<string, keyof ActivityCounts | null> = {
    pray: 'pray', journal: 'journal', meditate: 'meditate', listen: 'listen',
    prayerWall: 'prayerWall', readingPlan: 'readingPlan',
    gratitude: 'gratitude', reflection: 'reflection',
    mood: null, devotional: null, challenge: null, localVisit: null,
  };
  const countKey = COUNT_KEYS[input.newActivityType];
  if (countKey) {
    incrementedCounts[countKey] = (incrementedCounts[countKey] ?? 0) + 1;
  }

  const earned: Record<string, BadgeEarnedEntry> = {};
  for (const id of input.alreadyEarnedBadgeIds) {
    earned[id] = { earnedAt: input.today };
  }

  const context: BadgeCheckContext = {
    streak: newStreak,
    level: levelInfo.level,
    previousLevel: input.currentLevel,
    todayActivities: buildDailyActivities(newSet),
    activityCounts: incrementedCounts,
    friendCount: input.friendCount,
    allActivitiesWereTrueBefore: false,
  };

  const newBadgeIds = checkForNewBadges(context, earned).slice().sort();

  return {
    pointsEarned: pointsDelta,
    newTotalPoints,
    newCurrentLevel: levelInfo.level,
    levelUp,
    newCurrentStreak: newStreak.currentStreak,
    newLongestStreak: newStreak.longestStreak,
    streakTransition,
    newBadgeIds,
    isFreeRepairAvailableAfter: pureFreeRepairAvailable(
      input.lastFreeRepairDate,
      input.currentWeekStart,
    ),
  };
}
