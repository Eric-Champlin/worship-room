## Architecture Context

**Existing patterns this plan anchors on (verified at recon):**

- **Activity engine (`backend/src/main/java/com/worshiproom/activity/`):** Phase 2 shipped 10/10 specs. `ActivityType` enum at line 21-34 has 13 values (`MOOD`, `PRAY`, …, `INTERCESSION`). `ActivityService.recordActivity(userId, request)` is the canonical entry point; runs `@Transactional` inside `ActivityController` at `/api/v1/activity`. The service runs the first-time-today gate, fires `FaithPointsService`/`StreakService`/`BadgeService`, persists everything in one transaction, and emits `milestone_events` rows (Spec 2.5.4b).
- **Activity counts table (`activity_counts`):** Schema is `(user_id UUID, count_type VARCHAR(50), count_value INTEGER, last_updated TIMESTAMP WITH TIME ZONE)` with composite PK. NEW `CountType` values do NOT require a schema migration — the table is polymorphic on `count_type` string. Changeset `2026-04-25-007-create-activity-counts-table.xml`.
- **CountType enum** (`activity/CountType.java`): 14 values keyed by camelCase wire string. `fromWireValue` returns `Optional` (resilient lookup so unknown DB rows are skipped). Adding `QUICK_LIFT("quickLift")` is a one-line addition.

- **`BadgeService`** (`activity/BadgeService.java`): Pure function — no Spring deps, no repo access. 15 eligibility-category helpers run in fixed order against a `BadgeCheckContext`. `intercessor(c)` (line 239-244) is the canonical sibling for a count-of-activity badge with a single threshold: `if (c.activityCounts().intercessionCount() >= BadgeThresholds.INTERCESSIONS) return List.of("prayerwall_25_intercessions");`. - **`BadgeCatalog`** (`activity/constants/BadgeCatalog.java`): 173 lines, 58 badges, metadata-only `Map<String, BadgeDefinition>`. Prayer-wall milestones block at lines 108-114. NEW `faithful_watcher` slots between `prayerwall_10_posts` and `prayerwall_25_intercessions` (or after — pick by alphabetic match against the frontend BADGE_DEFINITIONS ordering for parity).
- **`BadgeThresholds`** (`activity/constants/BadgeThresholds.java`): Two shapes — `LinkedHashMap<ActivityType, int[]> ACTIVITY_MILESTONES` and `public static final int X` constants. Single-threshold count-of-activity badges (Intercessions, Local Support visits, Local first visit, Gratitude consecutive streak, Listen 10 hours) use the constant shape. **Faithful Watcher uses constant `public static final int QUICK_LIFTS = 10;`** to match.

## API Changes

| Method | Endpoint | Auth | Rate Limit | Request Body | Response |
|--------|----------|------|-----------|-------------|----------|
| POST | `/api/v1/quick-lift/start` | Required (JWT) | 10/min/user (configurable) | `{ "postId": "<uuid>" }` | `200 OK` with `ProxyResponse<{ sessionId: UUID, serverStartedAt: Instant }>` |
| POST | `/api/v1/quick-lift/{sessionId}/complete` | Required (JWT) | 30/min/user (defense-in-depth, configurable) | (none — `sessionId` in path) | `200 OK` with `ProxyResponse<{ activityRecorded: boolean, pointsAwarded: int, badgesUnlocked: List<NewBadge> }>` |

**Error responses (all `ProxyError` shape per `03-backend-standards.md`):**
- `400 BAD_REQUEST` with code `TIMING_TOO_EARLY` on `/complete` when `(server_now - session.started_at) < 30s`
- `401 UNAUTHORIZED` on either endpoint when JWT missing/invalid
- `403 FORBIDDEN` with code `FORBIDDEN` on `/complete` when caller's `userId != session.user_id` (W3: do NOT reveal existence with 404)
- `404 NOT_FOUND` with code `NOT_FOUND` on `/start` when `postId` does not exist, or on `/complete` when `sessionId` does not exist
- `409 CONFLICT` with code `ACTIVE_SESSION_EXISTS` on `/start` when an active session for `(userId, postId)` already exists
- `409 CONFLICT` with code `ALREADY_COMPLETED` on `/complete` when session is already terminal (completed OR cancelled)
- `429 TOO_MANY_REQUESTS` with code `QUICK_LIFT_START_RATE_LIMITED` (or `_COMPLETE_RATE_LIMITED`) + `Retry-After` header

## Spec-Category-Specific Guidance

**This is a feature spec (full-stack, NOT dual-write, NOT cutover, NOT backend-only).** Quick Lift is a brand-new server-authoritative feature with no localStorage primary path. The frontend calls the backend directly. Apply:

- **Backend-only-style step structure for the server side:** Liquibase → entity → repository → service → controller → exception/handler → SecurityConfig → properties — in that order.
- **Activity engine integration is NOT dual-write:** server records the activity inside the complete transaction (W7 atomic). Frontend MUST NOT call `useFaithPoints.recordActivity` with the standard backend dual-write path, or it would double-insert. Either (a) extend `recordActivity` with a `{skipBackendDualWrite?: boolean}` option, or (b) introduce a sibling local-only updater. **Decision: extend `recordActivity` with the option flag** (see Edge Cases & Decisions table). The localStorage path still runs so the UI reflects the new activity for today's checklist + points display + badge celebration.
- **Universal Rule 17 (cutover a11y smoke test) does NOT apply** — Spec 6.2 is a feature spec, not a cutover spec. `/code-review` + `/verify-with-playwright` cover the per-spec a11y requirements.

## Edge Cases & Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| **Frontend activity recording after Quick Lift complete success** | Extend `useFaithPoints.recordActivity(type, sourceFeature, options?)` with `options.skipBackendDualWrite: boolean`. After successful POST `/complete`, frontend calls `recordActivity('quickLift', 'quickLift-overlay', { skipBackendDualWrite: true })`. The localStorage flow runs (today's activities, faith points, streak, badge calculation) but `postActivityToBackend` is NOT called. | Backend already records the activity inside the complete transaction (W7). Calling the default dual-write would double-insert. Idempotency in `recordActivity` (line 154 `if (todayEntry[type]) return;`) handles repeat Quick Lifts gracefully. Drift-detection test still passes because both sides count the same +20 points and Faithful Watcher threshold. |
| **Server timing source** | DB `NOW()` for `started_at` INSERT default (via `<column ... defaultValueComputed="NOW()">` in changeset + `@Column(insertable=false, updatable=false)` on the entity field, then `entityManager.refresh(saved)` after `save()` per L1-cache trap). Application `Instant.now()` for elapsed check at complete time. | NTP-synced on Railway (R14). DB-source eliminates JVM/container clock skew at insertion time. Application-source at complete time is acceptable since the comparison is server-side. The 50ms typical JVM clock drift is negligible compared to the 30,000ms window. |
| **L1-cache trap on `started_at`** | `QuickLiftSession` entity declares `started_at` as `@Column(name = "started_at", insertable=false, updatable=false)`. `QuickLiftService.start()` calls `entityManager.refresh(saved)` after `quickLiftSessionRepository.save(...)` so the returned entity has the DB-populated `started_at`. Integration test asserts non-null `serverStartedAt` in start response. | Phase 3 Execution Reality Addendum item 2 + Spec 3.5/3.6 precedent. Without `refresh()`, the in-memory entity has `null` for `started_at` even after the SQL INSERT populates it. Regression-guard test makes the trap obvious. |
| **Atomic state transition on complete** | Complete uses native SQL UPDATE: `UPDATE quick_lift_sessions SET completed_at = ? WHERE id = ? AND completed_at IS NULL AND cancelled_at IS NULL` returning affected row count. If row count == 0, the session is already terminal → throw `QuickLiftException.alreadyCompleted()`. **Same DB transaction wraps `activityService.recordActivity` per W7.** | Concurrent complete attempts must produce exactly one success. SQL-side guard via `WHERE completed_at IS NULL` is the canonical SQL atomicity pattern. Anti-farming test verifies this. Pure JPA `if (entity.getCompletedAt() != null)` check + entity save would race. |
| **Bulk DELETE in cleanup job** | Use JPQL `@Modifying(clearAutomatically=true, flushAutomatically=true) @Query("DELETE FROM QuickLiftSession qls WHERE qls.completedAt IS NULL AND qls.cancelledAt IS NULL AND qls.startedAt < :cutoff")` in batched chunks of 1000. Loop until affected rows < 1000. | Phase 3 Execution Reality Addendum item 3 — both flags required on every `@Modifying`. JPQL preferred over native SQL for type safety; if Hibernate parser issues arise, fall back to JdbcTemplate native (acceptable per addendum). Bounded batch DELETE prevents long table lock at scale (W13). |
| **Rate-limit on /complete** | Same Caffeine-bounded `(UUID userId → Bucket)` pattern as /start, but with looser limits (30/min/user instead of 10/min/user). New `QuickLiftCompleteRateLimitConfig` + `QuickLiftCompleteRateLimitService` siblings to the start versions. | Defense-in-depth — a malicious client racing many sessions in parallel would mostly hit the start limit, but if start tokens are spread across multiple users the complete endpoint is the fallback gate. 30/min gives 1 complete every 2 seconds — far above any legitimate use. |
| **Faithful Watcher integration shape** | Add `CountType.QUICK_LIFT("quickLift")` enum value; extend `ActivityCountsSnapshot` with `int quickLiftCount` field; add `ActivityType.QUICK_LIFT → CountType.QUICK_LIFT` to `ActivityService.ACTIVITY_TO_COUNT` map; add `BadgeThresholds.QUICK_LIFTS = 10`; add `faithfulWatcher(c)` helper in `BadgeService` mirroring `intercessor(c)`. NO schema change to `activity_counts` (polymorphic on `count_type` string). | The brief's "BadgeCatalog with declarative criteria" architecture is R-OVR-2-overridden as wrong-shaped. The established pattern is `ActivityCountsSnapshot` + `BadgeService` switch. Same shape as Intercessor. Auto-increment via `ActivityService.recordActivity` → `ACTIVITY_TO_COUNT.get(QUICK_LIFT) == QUICK_LIFT` → `activityCountsService.incrementCount(userId, QUICK_LIFT)`. |
| **Frontend ActivityCounts parity** | Frontend `ActivityCounts` interface gets `quickLiftCount: number`; `FRESH_ACTIVITY_COUNTS` gets `quickLiftCount: 0`; `incrementActivityCount` switch gets `'quickLift'` case (likely already wildcard-safe — verify in plan recon); `badge-engine.ts checkForNewBadges` gets Faithful Watcher branch matching backend exactly. | Drift-detection test (`PointValuesTest.quickLiftPointsMatchFrontend`) asserts parity. Without frontend parity, the localStorage-canonical path miscalculates badges, dashboard shows wrong "today's progress," and dual-write mode diverges. |

    public QuickLiftStartRateLimitService(QuickLiftStartRateLimitConfig config) {
        this.config = config;
        this.userBuckets = Caffeine.newBuilder()
            .maximumSize(config.getBucketCacheSize())
            .expireAfterAccess(Duration.ofMinutes(5))   // strictly > 1-min refill window
            .build();
    }

    public void checkAndConsume(UUID userId) {
        Bucket bucket = userBuckets.get(userId, k -> newBucket());
        ConsumptionProbe probe = bucket.tryConsumeAndReturnRemaining(1);
        if (!probe.isConsumed()) {
            long retryAfterSec = Math.max(1L, TimeUnit.NANOSECONDS.toSeconds(probe.getNanosToWaitForRefill()));
            throw QuickLiftException.startRateLimited(retryAfterSec);
        }
    }

    private Bucket newBucket() {
        int max = config.getRequestsPerMinute();
        // Bucket4j Bandwidth.classic with intervally refill matches the documented "X per minute" semantics.
        Bandwidth limit = Bandwidth.classic(max, Refill.intervally(max, Duration.ofMinutes(1)));
        return Bucket.builder().addLimit(limit).build();
    }
}

```java
@Service
public class QuickLiftService {

    private static final Logger log = LoggerFactory.getLogger(QuickLiftService.class);
    static final long MINIMUM_DWELL_MS = 30_000L;   // 30 seconds

    private final QuickLiftSessionRepository sessionRepository;
    private final PostRepository postRepository;   // for existence check at /start
    private final ActivityService activityService;
    private final EntityManager entityManager;
    private final QuickLiftStartRateLimitService startRateLimit;
    private final QuickLiftCompleteRateLimitService completeRateLimit;

    // Constructor injection

    @Transactional
    public QuickLiftStartResponse start(UUID userId, UUID postId) {
        startRateLimit.checkAndConsume(userId);

        // Verify post exists; 404 if not. CASCADE FK ensures we don't insert orphan.
        if (!postRepository.existsById(postId)) throw QuickLiftException.notFound();

        // 409 if active session exists for (user, post)
        if (sessionRepository.findActiveByUserAndPost(userId, postId).isPresent()) {
            throw QuickLiftException.activeSessionExists();
        }

        QuickLiftSession session = new QuickLiftSession(userId, postId);
        QuickLiftSession saved = sessionRepository.save(session);
        // L1-cache trap: started_at is DB-default + insertable=false. Without refresh,
        // saved.getStartedAt() returns null. Phase 3 Execution Reality Addendum item 2.
        entityManager.refresh(saved);

        log.info("Quick Lift started userId={} postId={} sessionId={}", userId, postId, saved.getId());

        return new QuickLiftStartResponse(saved.getId(), saved.getStartedAt());
    }

    @Transactional
    public QuickLiftCompleteResponse complete(UUID userId, UUID sessionId) {
        completeRateLimit.checkAndConsume(userId);

        QuickLiftSession session = sessionRepository.findById(sessionId)
            .orElseThrow(QuickLiftException::notFound);

        // Cross-user attempt: 403, not 404 (W3 — do not leak existence via status differences)
        if (!session.getUserId().equals(userId)) throw QuickLiftException.forbidden();

        // Server-authoritative timing — never trust client time (Gate-G-SERVER-TIMING-HARD)
        long elapsedMs = Duration.between(session.getStartedAt(), OffsetDateTime.now(ZoneOffset.UTC)).toMillis();
        if (elapsedMs < MINIMUM_DWELL_MS) {
            throw QuickLiftException.timingTooEarly();
        }

        // Atomic state transition. Returns 0 if already terminal → 409.
        int affected = sessionRepository.markCompleted(sessionId, userId, OffsetDateTime.now(ZoneOffset.UTC));
        if (affected == 0) throw QuickLiftException.alreadyCompleted();

        // Activity recording INSIDE the same transaction (W7). If this throws, both
        // the session UPDATE AND the activity write roll back; user sees a clean error.
        ActivityRequest req = new ActivityRequest(ActivityType.QUICK_LIFT, "quickLift-overlay", Map.of("postId", session.getPostId().toString()));
        ActivityResponseData activity = activityService.recordActivity(userId, req);

        List<NewBadge> badges = activity.newBadges();   // includes faithful_watcher if 10th completion
        int pointsAwarded = activity.pointsDelta();

        log.info("Quick Lift completed userId={} sessionId={} pointsAwarded={} badgesUnlocked={}",
            userId, sessionId, pointsAwarded, badges.size());

        return new QuickLiftCompleteResponse(true, pointsAwarded, badges);
    }
}

@Component
public class QuickLiftCleanupJob {

    private static final Logger log = LoggerFactory.getLogger(QuickLiftCleanupJob.class);
    static final int BATCH_SIZE = 1000;
    static final Duration ABANDONED_AFTER = Duration.ofMinutes(5);

    private final QuickLiftSessionRepository sessionRepository;

    public QuickLiftCleanupJob(QuickLiftSessionRepository sessionRepository) {
        this.sessionRepository = sessionRepository;
    }

    /**
     * Run every 15 minutes at minute :00 / :15 / :30 / :45. Cron format is
     * second minute hour dayOfMonth month dayOfWeek (Spring 6-field).
     */
    @Scheduled(cron = "0 */15 * * * *", zone = "UTC")
    @Transactional
    public void cleanup() {
        OffsetDateTime cutoff = OffsetDateTime.now(ZoneOffset.UTC).minus(ABANDONED_AFTER);

        int totalDeleted = 0;
        int deleted;
        do {
            deleted = sessionRepository.deleteAbandonedBatch(cutoff, BATCH_SIZE);
            totalDeleted += deleted;
        } while (deleted >= BATCH_SIZE);

        if (totalDeleted > 0) {
            log.info("quickLiftCleanupPruned count={}", totalDeleted);
        }
    }
}

export function playWindChime(): void {
  if (!isSoundEnabled()) return;
  try {
    cachedContext = cachedContext ?? new AudioContext();
    const ctx = cachedContext;
    if (ctx.state === 'suspended') ctx.resume();
    const now = ctx.currentTime;
    [{ freq: 523.25, delay: 0 }, { freq: 783.99, delay: 0.1 }].forEach(({ freq, delay }) => {

      const gain = ctx.createGain();
      osc.type = 'sine';

      gain.gain.exponentialRampToValueAtTime(0.0001, now + delay + 1.5);

        ? "You've already started a Quick Lift on this post — finish that one first."
        : code === 'QUICK_LIFT_START_RATE_LIMITED'
        ? "Please slow down. Try again in a moment."
        : "We couldn't start that. Please try again.";
      setState({ phase: 'error', code, message });
    }
  }, [postId]);

  const complete = useCallback(async (sessionId: string) => {
    setState({ phase: 'completing', sessionId });
    try {
      const res = await apiFetch<{ data: QuickLiftCompleteResponse; meta: { requestId: string } }>(
        `/api/v1/quick-lift/${sessionId}/complete`,
        { method: 'POST' },
      );
      setState({ phase: 'complete', response: res.data });
    } catch (err: any) {
      const code = err?.code ?? 'NETWORK_ERROR';
      const message = code === 'TIMING_TOO_EARLY'
        ? "Almost there. Give it the full 30 seconds."
        : code === 'ALREADY_COMPLETED'
        ? "This Quick Lift is already done."
        : "We couldn't complete that. Please try again.";
      setState({ phase: 'error', code, message });
    }
  }, []);

  const reset = useCallback(() => setState({ phase: 'idle' }), []);

  return { state, start, complete, reset };
}
```

import { playWindChime } from '@/lib/quickLiftSound';
import { useQuickLift } from '@/hooks/useQuickLift';
import type { QuickLiftCompleteResponse } from '@/types/quickLift';

const QUICK_LIFT_DURATION_MS = 30_000;
const RING_CIRCUMFERENCE = 2 * Math.PI * 90;   // r=90 SVG circle
const TITLE_ID = 'quick-lift-overlay-title';

interface QuickLiftOverlayProps {
  isOpen: boolean;
  postId: string;
  postExcerpt: string;
  onCancel: () => void;
  onComplete: (response: QuickLiftCompleteResponse) => void;
}

export function QuickLiftOverlay({ isOpen, postId, postExcerpt, onCancel, onComplete }: QuickLiftOverlayProps) {
  const containerRef = useFocusTrap(isOpen, onCancel);
  const { state, start, complete, reset } = useQuickLift(postId);
  const [reducedMotion] = useState(() =>
    typeof window !== 'undefined' && window.matchMedia('(prefers-reduced-motion: reduce)').matches
  );
  const [progressMilestone, setProgressMilestone] = useState(0);   // 0/25/50/75/100 for SR announcements
  const completedRef = useRef(false);

  // Start session on open
  useEffect(() => {
    if (isOpen && state.phase === 'idle') start();
    if (!isOpen) reset();
  }, [isOpen, state.phase, start, reset]);

  // Drive completion at exactly 30 seconds after serverStartedAt
  useEffect(() => {
    if (state.phase !== 'running' || completedRef.current) return;
    const remaining = Math.max(0, QUICK_LIFT_DURATION_MS - (Date.now() - state.serverStartedAt));
    const timer = window.setTimeout(() => {
      completedRef.current = true;
      complete(state.sessionId);
    }, remaining);
    return () => window.clearTimeout(timer);
  }, [state, complete]);

  // SR progress announcements at 0/25/50/75/100 checkpoints
  useEffect(() => {
    if (state.phase !== 'running') return;
    const intervals = [0.25, 0.5, 0.75, 1.0].map((frac, i) => {
      const remaining = Math.max(0, frac * QUICK_LIFT_DURATION_MS - (Date.now() - state.serverStartedAt));
      return window.setTimeout(() => setProgressMilestone((i + 1) * 25), remaining);
    });
    return () => intervals.forEach((t) => window.clearTimeout(t));
  }, [state]);

  // On complete: wind chime + call onComplete + auto-close after 2 seconds
  useEffect(() => {
    if (state.phase !== 'complete') return;
    playWindChime();
    onComplete(state.response);
    const t = window.setTimeout(onCancel, 2000);
    return () => window.clearTimeout(t);
  }, [state, onComplete, onCancel]);

  if (!isOpen) return null;

  const excerpt = postExcerpt.length > 80 ? `${postExcerpt.slice(0, 80)}…` : postExcerpt;

  return (
    <div
      role="dialog"
      aria-modal="true"
      aria-labelledby={TITLE_ID}
      className="fixed inset-0 z-50 flex items-center justify-center bg-black/70 backdrop-blur-sm p-4"
      onClick={onCancel}
    >
      <div
        ref={containerRef}
        className={cn(
          'relative w-full max-w-md rounded-3xl border border-violet-400/40 bg-hero-bg shadow-frosted-accent p-8 text-white',
        )}
        onClick={(e) => e.stopPropagation()}
        data-reduced-motion={reducedMotion ? 'true' : 'false'}
      >
        <button
          type="button"
          onClick={onCancel}
          aria-label="Cancel Quick Lift"
          className="absolute right-3 top-3 flex h-11 w-11 items-center justify-center rounded-full text-white/70 hover:text-white hover:bg-white/[0.06] focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-violet-400/60"
          data-testid="quick-lift-overlay-close"
        >
          <X className="h-5 w-5" aria-hidden="true" />
        </button>

        <h2 id={TITLE_ID} className="sr-only">Quick Lift in prayer</h2>

        <p className="mb-6 text-sm italic text-white/70 leading-relaxed">"{excerpt}"</p>

        {/* SVG progress ring — CSS animation on stroke-dashoffset (60fps GPU) */}
        <div className="flex justify-center mb-6">
          <svg viewBox="0 0 200 200" className="h-44 w-44" aria-hidden="true">
            <circle cx="100" cy="100" r="90" fill="none" stroke="rgba(139,92,246,0.15)" strokeWidth="6" />
            <circle
              cx="100" cy="100" r="90" fill="none"
              stroke="rgb(167,139,250)"
              strokeWidth="6"
              strokeLinecap="round"
              strokeDasharray={RING_CIRCUMFERENCE}
              strokeDashoffset={RING_CIRCUMFERENCE}
              transform="rotate(-90 100 100)"
              className={cn(
                'quick-lift-ring',
                reducedMotion ? 'quick-lift-ring--reduced' : 'quick-lift-ring--smooth',
              )}
            />
          </svg>
        </div>

        {/* Progress announcement region — SR-only, polite */}
        <div role="progressbar"
          aria-valuemin={0} aria-valuemax={100} aria-valuenow={progressMilestone}
          aria-valuetext={`${progressMilestone}% through Quick Lift`}
          aria-live="polite"
          className="sr-only">
          {progressMilestone}% through Quick Lift
        </div>

        {state.phase === 'complete' ? (
          <p className="text-center text-lg font-medium text-white" aria-live="polite">Thank you.</p>
        ) : (
          <p className="text-center text-sm text-white/60">Praying alongside</p>
        )}

        {state.phase === 'error' && (
          <p role="alert" className="mt-3 rounded-md border border-red-400/30 bg-red-950/30 p-3 text-sm text-red-100">

}

| 6 | Wind chime is a NEW module at `lib/quickLiftSound.ts`, NOT added to the standard 6-sound `lib/sound-effects.ts` palette | The wind chime is feature-specific (only used by Quick Lift completion).
