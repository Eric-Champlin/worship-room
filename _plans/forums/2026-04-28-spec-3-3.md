## Architecture Context

### Existing Backend Structure

Recon (Explore agent + direct file reads) confirmed the following patterns are firmly established and MUST be matched:

**Package layout** — `com.worshiproom.{domain}/` with everything for that domain in one package: entity, ID class (composite keys only), enum + AttributeConverter pairs, repository, service, controller, mapper helpers (DisplayNameResolver), exception base + subclasses, exception handlers, DTOs in `dto/` subpackage. Verified against `mute/`, `friends/`, `user/`, `activity/`. Spec 3.3 establishes `com.worshiproom.post/` following the same convention.

**JPA entity convention** (verified against `User.java`, `FriendRelationship.java`, `UserMute.java`):
- `@Entity` + `@Table(name = "...")`
- `@Id @Column(...)` with manual UUID assignment in `@PrePersist` (NOT `@GeneratedValue`) — note: `03-backend-standards.md` says `@GeneratedValue(strategy = GenerationType.UUID)`, but EVERY existing entity uses manual `@PrePersist`. Match existing code, not the rule file. (DB default is `gen_random_uuid()` so reads work either way; manual `@PrePersist` matches the established pattern.)
- Timestamp type is **`OffsetDateTime`** (consistent across User, FriendRelationship, UserMute) — matches `TIMESTAMP WITH TIME ZONE` columns. Hibernate property `hibernate.jdbc.time_zone=UTC` is set in `application.properties`.
- Enum handling: `@Convert(converter = XxxConverter.class)` with custom `AttributeConverter<EnumClass, String>` (NOT `@Enumerated(EnumType.STRING)`). The converter's `value()` / `fromValue()` methods own the DB-string mapping. Pattern verified in `DisplayNamePreferenceConverter`, `FriendRelationshipStatusConverter`.
- `protected` no-arg constructor (JPA requirement) + one all-args constructor.
- Equals/hashCode on `id` only.
- NO Lombok anywhere in the codebase — explicit getters/setters.

**Service convention** (verified against `MuteService`, `FriendsService`):
- `@Service` + constructor injection (no `@Autowired` on fields)
- `@Transactional` on writes; `@Transactional(readOnly = true)` on reads
- Custom checked-style RuntimeException subclasses (e.g., `MuteException → SelfActionException`, `UserNotFoundException`)
- Services return DTOs, NOT entities (mapper invocation done in service)
- For pagination: services use Spring Data `Pageable` + `Page<T>` from repository; FriendsService.searchUsers shows the pattern but doesn't expose pagination meta yet (returns plain `List<DTO>`). **Spec 3.3 is the first spec to expose pagination meta in the API response** — see Step 9.

**Controller convention** (verified against `MuteController`, `ActivityController`, `FriendsController`):
- `@RestController` (sometimes with class-level `@RequestMapping`, sometimes with per-method `@PostMapping("/api/v1/...")`)
- Auth via `@AuthenticationPrincipal AuthenticatedUser principal` — record `(UUID userId, boolean isAdmin)`
- `@Valid @RequestBody DTO body` for request validation
- Return type is `ResponseEntity<ProxyResponse<T>>` where `ProxyResponse` is the standard `{data, meta}` envelope
- `requestId` resolved via `MDC.get("requestId")` (set by `RequestIdFilter` at `HIGHEST_PRECEDENCE`)
- INFO-level controller log on entry (e.g., `log.info("Activity received userId={} ...")`)

**Optional auth pattern (CRITICAL — Watch-For #6 from spec):**
The codebase has no existing pattern for endpoints that work BOTH authenticated AND anonymously. Recon resolved the architectural decision:

- `PublicPaths.PATTERNS` is the single source of truth for both `SecurityConfig.requestMatchers(...).permitAll()` AND `JwtAuthenticationFilter.shouldNotFilter()`. **Adding posts paths there would make `JwtAuthenticationFilter` SKIP them entirely**, which means valid tokens would NOT extract a principal — breaking personalized filtering for authenticated users.
- `JwtAuthenticationFilter.doFilterInternal` (line 96–101) already has the right behavior for non-PublicPath URLs: missing header → pass through with no principal; valid header → principal extracted; invalid header → 401.
- The blocker: `SecurityConfig` declares `.requestMatchers("/api/v1/**").authenticated()` which rejects no-token requests on non-PublicPath URLs.

**Resolution:** Introduce a sibling list `PublicPaths.OPTIONAL_AUTH_PATTERNS` (kept in the same file for discoverability). `SecurityConfig` permits BOTH lists with `permitAll()`. `JwtAuthenticationFilter.shouldNotFilter()` continues to check ONLY `PATTERNS` (not the new list). Net effect: posts endpoints accept anonymous AND personalize when a valid token is present; an invalid token still returns 401. This is exactly the spec's requested behavior in Watch-For #6.

**Exception handler convention** (verified against `MuteExceptionHandler`, `ProxyExceptionHandler`):
- `@RestControllerAdvice(basePackages = "com.worshiproom.{domain}")` — package-scoped to prevent cross-package contamination
- Single `@ExceptionHandler({DomainException}.class)` catches all subclasses
- Returns `ResponseEntity<ProxyError>` where `ProxyError` is record `(String code, String message, String requestId, Instant timestamp)`
- `requestId` from `MDC.get("requestId")`; `timestamp` from `Instant.now()`
- Validation handlers (`MethodArgumentNotValidException`, `ConstraintViolationException`, `HandlerMethodValidationException`, `MissingServletRequestParameterException`) → 400 `INVALID_INPUT`. Pattern is duplicated across packages because each package's advice is `basePackages`-scoped — this is intentional. Spec 3.3 follows the same pattern with its own `PostValidationExceptionHandler`.

**Test infrastructure** (verified against `AbstractIntegrationTest`, `AbstractDataJpaTest`, `ActivityControllerIntegrationTest`):
- `AbstractIntegrationTest` extends `@SpringBootTest` + uses singleton Testcontainers PostgreSQL via `TestContainers.registerJdbcProperties(registry)` in `@DynamicPropertySource`. Subclasses add `@AutoConfigureMockMvc`.
- `AbstractDataJpaTest` extends `@DataJpaTest` + `@AutoConfigureTestDatabase(replace = Replace.NONE)`. Same singleton container. Auto-rollback per test.
- Liquibase context override pinned to `"test"` so dev-seed and the prayer-wall mock seed (changeset 021, `context="dev"`) are SKIPPED during tests. **Tests that need post data MUST seed it themselves.**
- Auth tokens generated via `JwtService.generateToken(userId, isAdmin)` for integration tests
- Test count target: 40-50 (slightly above L target due to visibility-predicate criticality; spec confirms acceptable)

**Database tables involved (already exist):**

- `users` (Spec 1.3) — joined for non-anonymous author resolution

**Decisions resolved during planning (recorded for audit):**

1. **Author-posts mute semantic** (Watch-For #2) — chose **empty-array** over 404. Rationale: preserves asymmetric mute contract (Sarah doesn't know she's muted; viewer just sees no posts), consistent with feed-read mute behavior, doesn't leak "Sarah was muted" via differential responses.
2. **Single-post mute semantic** (Divergence 1, Watch-For #3) — mute filter NOT applied. Direct ID lookups bypass mute (e.g., notification or shared-link path).
3. **404 (not 403) on visibility-excluded post** (Watch-For #4) — anti-enumeration; don't leak post existence.
4. **404 NOT returned for nonexistent username** (Watch-For #5) — empty array, 200 OK; anti-enumeration.
5. **Optional auth via new `PublicPaths.OPTIONAL_AUTH_PATTERNS` list** (Watch-For #6) — see Architecture Context. Cleanest path that preserves invalid-token-rejection while allowing anonymous AND authenticated reads.
6. **Single-post detail returns ONLY post fields** (Divergence 2) — comments come from Spec 3.4's `GET /posts/{id}/comments` endpoint. No inlined comments.
7. **Page/limit pagination** (Divergence 3) — NOT cursor-based per master plan acceptance criterion. `meta` includes `page`, `limit`, `totalCount`, `totalPages`, `hasNextPage`, `hasPrevPage`, `requestId`.
8. **Visibility predicate centralized in `PostSpecifications.visibleTo(viewerId)`** (Divergence 4) — JPA `Specification<Post>` API. Recon confirmed no existing Specification usage; this spec establishes the pattern.
9. **Username path resolution: kebab-case derivation OR UUID** (Divergence 5) — Phase 8.1 will replace with `users.username` lookup. Followup recorded in `_plans/post-1.10-followups.md` (file may need creating; if not present, write a single-line entry as part of Step 5).
10. **`OffsetDateTime` for timestamps** — confirmed via recon against `User`, `FriendRelationship`, `UserMute`. Match existing convention; do NOT introduce `Instant` or `LocalDateTime`.

|----------|--------|-----------|

| Mute filter on `/posts/{id}` | NOT applied | Direct lookups bypass discovery filters; spec Divergence 1 |

### Step 3: Create `UserResolverService` (kebab-case username → UUID resolution)

**Objective:** Resolve the `{username}` path param from `GET /api/v1/users/{username}/posts` to a `UUID`. Per Spec Divergence 5, accepts kebab-case (`sarah-johnson` → `first_name='Sarah' AND lower(last_name)='johnson'`) OR a literal UUID. When Phase 8.1 ships `users.username`, this service swaps to a single-column lookup.

**Files to create:**
- `backend/src/main/java/com/worshiproom/post/UserResolverService.java`

**Files to modify:**
- `_plans/post-1.10-followups.md` — append a single-line entry: `- [ ] Replace kebab-case username derivation in UserResolverService.resolveByKebabCase() with users.username lookup when Phase 8.1 ships (round3-phase08-spec01-username-system).` Create the file if it does not exist.

**Details:**
```java
package com.worshiproom.post;

import com.worshiproom.user.User;
import com.worshiproom.user.UserRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.Optional;
import java.util.UUID;

@Service
@Transactional(readOnly = true)
public class UserResolverService {

    private final UserRepository userRepository;

    public UserResolverService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    /**
     * Resolves the {username} path param to a UUID. Accepts:
     *   1. A literal UUID string ("00000000-0000-0000-0000-000000000101")
     *   2. A kebab-case identifier derived from first_name + '-' + last_name
     *      (lowercased; "sarah-johnson", "mary-anne-smith")
     *
     * Returns Optional.empty() if neither path resolves. Caller decides whether
     * to 404 or return empty array (Spec 3.3 returns empty array for the
     * /users/{username}/posts endpoint per anti-enumeration policy).
     *
     * Followup: Phase 8.1 (round3-phase08-spec01-username-system) replaces this
     * kebab-case derivation with users.username column lookup. See
     * _plans/post-1.10-followups.md.
     */
    public Optional<UUID> resolve(String identifier) {
        if (identifier == null || identifier.isBlank()) {
            return Optional.empty();
        }
        // UUID first — cheaper than DB hit
        try {
            return Optional.of(UUID.fromString(identifier));
        } catch (IllegalArgumentException ignored) {
            // Not a UUID; try kebab-case
        }
        return resolveByKebabCase(identifier);
    }

    private Optional<UUID> resolveByKebabCase(String kebab) {
        int lastDash = kebab.lastIndexOf('-');
        if (lastDash <= 0 || lastDash == kebab.length() - 1) {
            return Optional.empty();
        }
        String firstNameKebab = kebab.substring(0, lastDash);
        String lastNameKebab = kebab.substring(lastDash + 1);
        // Multi-word first names: "mary-anne-smith" → first="mary anne", last="smith"
        // Recon should verify whether dev seed has multi-word names — if not,
        // single-word first/last is sufficient. The mock seed (Spec 3.2) has
        // single-word first/last names only (Sarah Johnson, David Chen, etc.)
        // so simple last-dash split works for the seeded cases.
        return userRepository.findByFirstNameIgnoreCaseAndLastNameIgnoreCase(
            firstNameKebab.replace('-', ' '),
            lastNameKebab
        ).map(User::getId);
    }
}
```

### Step 4: Create `PostSpecifications` (THE LOAD-BEARING STEP)

**Objective:** Centralize the visibility predicate, mute filter, and individual filter clauses as composable `Specification<Post>` building blocks. This is the spec's load-bearing correctness concern — a buggy `visibleTo()` leaks private content.

**Files to create:**
- `backend/src/main/java/com/worshiproom/post/PostSpecifications.java`

**Details:**

The class is a final utility with private constructor and static factory methods:

```java
package com.worshiproom.post;

import com.worshiproom.friends.FriendRelationship;
import com.worshiproom.mute.UserMute;
import jakarta.annotation.Nullable;
import jakarta.persistence.criteria.Predicate;
import jakarta.persistence.criteria.Root;
import jakarta.persistence.criteria.Subquery;
import org.springframework.data.jpa.domain.Specification;

import java.util.UUID;

public final class PostSpecifications {

    private PostSpecifications() {}

    /**
     * Canonical visibility predicate from master plan Spec 7.7 (line 6307–6321).
     * Side-by-side alignment in the JavaDoc body so future reviewers can verify.
     */
    public static Specification<Post> visibleTo(@Nullable UUID viewerId) {
        return (root, query, cb) -> {
            // posts.is_deleted = FALSE
            Predicate notDeleted = cb.isFalse(root.get("isDeleted"));

            // posts.moderation_status IN ('approved', 'flagged')
            Predicate moderationVisible = root.get("moderationStatus")
                .in(ModerationStatus.APPROVED, ModerationStatus.FLAGGED);

            // Visibility branches
            Predicate publicPost = cb.equal(root.get("visibility"), PostVisibility.PUBLIC);

            Predicate friendsPost;
            Predicate privatePost;
            Predicate ownPost;

            if (viewerId == null) {
                // Unauthenticated viewer: only public posts
                friendsPost = cb.disjunction();  // always false
                privatePost = cb.disjunction();
                ownPost = cb.disjunction();
            } else {
                // Authenticated viewer
                // friends: posts.visibility='friends' AND EXISTS friend_relationships row
                Subquery<Integer> frSubquery = query.subquery(Integer.class);
                Root<FriendRelationship> fr = frSubquery.from(FriendRelationship.class);
                frSubquery.select(cb.literal(1)).where(
                    cb.equal(fr.get("userId"), root.get("userId")),
                    cb.equal(fr.get("friendUserId"), viewerId),
                    cb.equal(fr.get("status"), com.worshiproom.friends.FriendRelationshipStatus.ACTIVE)
                );
                friendsPost = cb.and(
                    cb.equal(root.get("visibility"), PostVisibility.FRIENDS),
                    cb.exists(frSubquery)
                );

                privatePost = cb.and(
                    cb.equal(root.get("visibility"), PostVisibility.PRIVATE),
                    cb.equal(root.get("userId"), viewerId)
                );

                ownPost = cb.equal(root.get("userId"), viewerId);
            }

            Predicate visibilityClause = cb.or(publicPost, friendsPost, privatePost, ownPost);
            return cb.and(notDeleted, moderationVisible, visibilityClause);
        };
    }

    /**
     * Mute filter — composes WITH visibleTo() for feed and author-posts endpoints,
     * NOT for single-post detail (spec Divergence 1).
     *
     * Implemented as Specification subquery (NOT per-row MuteService.isMuted call)
     * to fold into single SELECT and avoid N+1.
     */
    public static Specification<Post> notMutedBy(@Nullable UUID viewerId) {
        if (viewerId == null) {
            return (root, query, cb) -> cb.conjunction();  // no mute filter for anonymous viewers
        }
        return (root, query, cb) -> {
            Subquery<Integer> muteSubquery = query.subquery(Integer.class);
            Root<UserMute> um = muteSubquery.from(UserMute.class);
            muteSubquery.select(cb.literal(1)).where(
                cb.equal(um.get("muterId"), viewerId),
                cb.equal(um.get("mutedId"), root.get("userId"))
            );
            return cb.not(cb.exists(muteSubquery));
        };
    }

    public static Specification<Post> byAuthor(UUID authorUserId) {
        return (root, query, cb) -> cb.equal(root.get("userId"), authorUserId);
    }

    public static Specification<Post> byCategory(@Nullable String category) {
        if (category == null) return (root, query, cb) -> cb.conjunction();
        return (root, query, cb) -> cb.equal(root.get("category"), category);
    }

    public static Specification<Post> byPostType(@Nullable PostType postType) {
        if (postType == null) return (root, query, cb) -> cb.conjunction();
        return (root, query, cb) -> cb.equal(root.get("postType"), postType);
    }

    public static Specification<Post> byQotdId(@Nullable String qotdId) {
        if (qotdId == null) return (root, query, cb) -> cb.conjunction();
        return (root, query, cb) -> cb.equal(root.get("qotdId"), qotdId);
    }

    public static Specification<Post> isAnswered() {
        return (root, query, cb) -> cb.isTrue(root.get("isAnswered"));
    }
}
```

**Side-by-side verification of `visibleTo()` against canonical pseudo-SQL** (from spec line 214–229 / master plan line 6307–6321):

| Pseudo-SQL line | Java translation |
|-----------------|------------------|
| `WHERE posts.is_deleted = FALSE` | `cb.isFalse(root.get("isDeleted"))` |
| `AND posts.moderation_status IN ('approved', 'flagged')` | `root.get("moderationStatus").in(APPROVED, FLAGGED)` |
| `posts.visibility = 'public'` | `cb.equal(root.get("visibility"), PUBLIC)` |
| `OR (posts.visibility = 'friends' AND :viewer_id IS NOT NULL AND EXISTS (SELECT 1 FROM friend_relationships fr WHERE fr.user_id = posts.user_id AND fr.friend_user_id = :viewer_id AND fr.status = 'active'))` | `null-check via if-branch` + `cb.and(cb.equal(visibility, FRIENDS), cb.exists(subquery))` where subquery joins on `fr.userId = post.userId AND fr.friendUserId = viewerId AND fr.status = ACTIVE` |
| `OR (posts.visibility = 'private' AND posts.user_id = :viewer_id)` | `cb.and(cb.equal(visibility, PRIVATE), cb.equal(userId, viewerId))` |
| `OR posts.user_id = :viewer_id` | `cb.equal(userId, viewerId)` |

**The friend_relationships subquery direction matters:** master plan SQL says `fr.user_id = posts.user_id AND fr.friend_user_id = :viewer_id`. This means the relationship row's `user_id` (relationship author) is the post's author, and `friend_user_id` is the viewer. Translated: `fr.userId = root.get("userId")` (post's userId) and `fr.friendUserId = viewerId`. **Reversing these leaks private content.** The Specification has them in the correct direction; tests in Step 14 verify both directions of friendship behavior (viewer→author vs author→viewer; only the direction matching the predicate should expose posts).

**Friend status is `ACTIVE`, not `BLOCKED`:** The status enum value is `FriendRelationshipStatus.ACTIVE`. Using a different value (e.g., `'pending'` from `friend_requests` — a different table) would silently drop legitimate friend posts.

**Guardrails (DO NOT):**
- Do NOT collapse the four visibility branches into a single `cb.or(...)` outside the if-branch — the unauthenticated branch must NOT generate the subquery (avoids unnecessary `EXISTS` against `friend_relationships` for anonymous viewers; also makes the predicate testable in isolation).
- Do NOT use string literals (`"approved"`, `"public"`) — use the enum constants. The `AttributeConverter` translates them on the wire.
- Do NOT call `MuteService.isMuted()` per row from the service — that's N+1. The Specification subquery is the only correct shape.
- Do NOT reverse `fr.userId` and `fr.friendUserId`. Both must match the master plan SQL exactly (recon verified the FriendRelationship entity uses `userId` and `friendUserId` field names).
- Do NOT add a `byVisibility(PostVisibility)` Specification — visibility is owned by `visibleTo(viewerId)`. A separate filter would let callers request `?visibility=private`, which conflicts with the predicate's invariant.
- Do NOT add a sort Specification — sort is applied via `Pageable` / `Sort` in `PostService` (Step 9), not via Specification. Sort whitelist validation (per Watch-For #13) happens at the controller layer.

**Objective:** Convert `Post` entity to `PostDto`, with two responsibilities: (a) anonymous-handling (return `AuthorDto(null, "Anonymous", null)` regardless of author), (b) author resolution for non-anonymous posts (call `DisplayNameResolver.resolve(user)` + read `user.getAvatarUrl()`).

- `backend/src/main/java/com/worshiproom/post/PostMapper.java`

**Details:**
```java
package com.worshiproom.post;

import com.worshiproom.post.dto.AuthorDto;
import com.worshiproom.post.dto.PostDto;
import com.worshiproom.user.DisplayNameResolver;
import com.worshiproom.user.User;
import com.worshiproom.user.UserRepository;
import org.springframework.stereotype.Component;

import java.util.List;
import java.util.Map;
import java.util.Set;
import java.util.UUID;
import java.util.function.Function;
import java.util.stream.Collectors;

@Component
public class PostMapper {

    private final UserRepository userRepository;

    public PostMapper(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    /**
     * Single-post mapping. Performs ONE user lookup for non-anonymous posts.
     * For lists, prefer toDtoList() to avoid N+1.
     */
    public PostDto toDto(Post post) {
        AuthorDto author;
        if (post.isAnonymous()) {
            author = new AuthorDto(null, "Anonymous", null);
        } else {
            User user = userRepository.findById(post.getUserId())
                .orElseThrow(() -> new IllegalStateException(
                    "Post " + post.getId() + " references missing user " + post.getUserId()));
            author = new AuthorDto(
                user.getId(),
                DisplayNameResolver.resolve(user),
                user.getAvatarUrl()
            );
        }
        return buildDto(post, author);
    }

    /**
     * Batch mapping. Loads all required users in a SINGLE query, avoiding N+1.
     */
    public List<PostDto> toDtoList(List<Post> posts) {
        // Collect non-anonymous author UUIDs only — anonymous posts don't need user data
        Set<UUID> authorIds = posts.stream()
            .filter(p -> !p.isAnonymous())
            .map(Post::getUserId)
            .collect(Collectors.toSet());
        Map<UUID, User> userById = userRepository.findAllById(authorIds).stream()
            .collect(Collectors.toMap(User::getId, Function.identity()));
        return posts.stream().map(p -> {
            AuthorDto author;
            if (p.isAnonymous()) {
                author = new AuthorDto(null, "Anonymous", null);
            } else {
                User u = userById.get(p.getUserId());
                if (u == null) {
                    throw new IllegalStateException(
                        "Post " + p.getId() + " references missing user " + p.getUserId());
                }
                author = new AuthorDto(u.getId(), DisplayNameResolver.resolve(u), u.getAvatarUrl());
            }
            return buildDto(p, author);
        }).toList();
    }

    private PostDto buildDto(Post p, AuthorDto author) {
        return new PostDto(
            p.getId(),
            p.getPostType().value(),                  // enum → DB string for JSON
            p.getContent(),
            p.getCategory(),
            p.isAnonymous(),
            p.getChallengeId(),
            p.getQotdId(),
            p.getScriptureReference(),
            p.getScriptureText(),
            p.getVisibility().value(),
            p.isAnswered(),
            p.getAnsweredText(),
            p.getAnsweredAt(),
            p.getModerationStatus().value(),
            p.isCrisisFlag(),
            p.getPrayingCount(),
            p.getCandleCount(),
            p.getCommentCount(),
            p.getBookmarkCount(),
            p.getCreatedAt(),
            p.getUpdatedAt(),
            p.getLastActivityAt(),
            author
        );
    }
}
```

import org.springframework.data.domain.Pageable;
import org.springframework.data.domain.Sort;
import org.springframework.data.jpa.domain.Specification;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;
import java.util.Optional;
import java.util.UUID;

@Service
@Transactional(readOnly = true)
public class PostService {

    private final PostRepository postRepository;
    private final PostMapper postMapper;
    private final UserResolverService userResolverService;

    public PostService(PostRepository postRepository, PostMapper postMapper,
                       UserResolverService userResolverService) {
        this.postRepository = postRepository;
        this.postMapper = postMapper;
        this.userResolverService = userResolverService;
    }

    public PostListResponse listFeed(
        @org.springframework.lang.Nullable UUID viewerId,
        int page, int limit,
        @org.springframework.lang.Nullable String category,
        @org.springframework.lang.Nullable PostType postType,
        @org.springframework.lang.Nullable String qotdId,
        SortKey sort,
        String requestId
    ) {
        Specification<Post> spec = Specification.where(PostSpecifications.visibleTo(viewerId))
            .and(PostSpecifications.notMutedBy(viewerId))
            .and(PostSpecifications.byCategory(category))
            .and(PostSpecifications.byPostType(postType))
            .and(PostSpecifications.byQotdId(qotdId));
        if (sort == SortKey.ANSWERED) {
            spec = spec.and(PostSpecifications.isAnswered());
        }
        Pageable pageable = PageRequest.of(page - 1, limit, sortFor(sort));
        Page<Post> resultPage = postRepository.findAll(spec, pageable);
        List<PostDto> dtos = postMapper.toDtoList(resultPage.getContent());
        return new PostListResponse(dtos, buildMeta(page, limit, resultPage.getTotalElements(), requestId));
    }

    public PostDto getById(UUID postId, @org.springframework.lang.Nullable UUID viewerId) {
        // Mute filter intentionally NOT applied here — direct ID lookups bypass discovery filters
        Specification<Post> spec = Specification.where(PostSpecifications.visibleTo(viewerId))
            .and((root, query, cb) -> cb.equal(root.get("id"), postId));
        Optional<Post> post = postRepository.findOne(spec);
        Post resolved = post.orElseThrow(PostNotFoundException::new);
        return postMapper.toDto(resolved);
    }

    public PostListResponse listAuthorPosts(
        String username,
        @org.springframework.lang.Nullable UUID viewerId,
        int page, int limit,
        SortKey sort,
        String requestId
    ) {
        Optional<UUID> authorId = userResolverService.resolve(username);
        if (authorId.isEmpty()) {
            // Anti-enumeration: empty array, not 404
            return new PostListResponse(List.of(), buildMeta(page, limit, 0L, requestId));
        }
        Specification<Post> spec = Specification.where(PostSpecifications.visibleTo(viewerId))
            .and(PostSpecifications.notMutedBy(viewerId))
            .and(PostSpecifications.byAuthor(authorId.get()));
        if (sort == SortKey.ANSWERED) {
            spec = spec.and(PostSpecifications.isAnswered());
        }
        Pageable pageable = PageRequest.of(page - 1, limit, sortFor(sort));
        Page<Post> resultPage = postRepository.findAll(spec, pageable);
        List<PostDto> dtos = postMapper.toDtoList(resultPage.getContent());
        return new PostListResponse(dtos, buildMeta(page, limit, resultPage.getTotalElements(), requestId));
    }

    private static Sort sortFor(SortKey key) {
        return switch (key) {
            case BUMPED -> Sort.by(Sort.Direction.DESC, "lastActivityAt");
            case RECENT -> Sort.by(Sort.Direction.DESC, "createdAt");
            case ANSWERED -> Sort.by(Sort.Direction.DESC, "answeredAt");
        };
    }

    private static PostListMeta buildMeta(int page, int limit, long totalCount, String requestId) {
        int totalPages = totalCount == 0 ? 0 : (int) Math.ceil((double) totalCount / limit);
        boolean hasNext = page < totalPages;
        boolean hasPrev = page > 1 && totalPages > 0;
        return new PostListMeta(page, limit, totalCount, totalPages, hasNext, hasPrev, requestId);
    }

    public enum SortKey {
        BUMPED, RECENT, ANSWERED;

        public static SortKey parse(String raw, SortKey defaultValue) {
            if (raw == null || raw.isBlank()) return defaultValue;
            return switch (raw.toLowerCase()) {
                case "bumped" -> BUMPED;
                case "recent" -> RECENT;
                case "answered" -> ANSWERED;
                default -> throw new InvalidSortException(raw);
            };
        }
    }
}
```

package com.worshiproom.post;

import com.worshiproom.auth.AuthenticatedUser;
import com.worshiproom.post.dto.PostDto;
import com.worshiproom.post.dto.PostListResponse;
import com.worshiproom.proxy.common.ProxyResponse;
import jakarta.validation.constraints.Max;
import jakarta.validation.constraints.Min;
import jakarta.validation.constraints.NotBlank;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.slf4j.MDC;
import org.springframework.http.ResponseEntity;
import org.springframework.security.core.annotation.AuthenticationPrincipal;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;
import org.springframework.validation.annotation.Validated;

import java.util.UUID;

@RestController
@RequestMapping("/api/v1")
@Validated
public class PostController {

    private static final Logger log = LoggerFactory.getLogger(PostController.class);

    private static final java.util.Set<String> VALID_CATEGORIES = java.util.Set.of(
        "health", "mental-health", "family", "work", "grief",
        "gratitude", "praise", "relationships", "other", "discussion"
    );

    private final PostService postService;

    public PostController(PostService postService) {
        this.postService = postService;
    }

    @GetMapping("/posts")
    public ResponseEntity<PostListResponse> listFeed(
        @AuthenticationPrincipal AuthenticatedUser principal,
        @RequestParam(defaultValue = "1") @Min(1) int page,
        @RequestParam(defaultValue = "20") @Min(1) @Max(50) int limit,
        @RequestParam(required = false) String category,
        @RequestParam(required = false) String postType,
        @RequestParam(required = false) String qotdId,
        @RequestParam(required = false, defaultValue = "bumped") String sort
    ) {
        UUID viewerId = principal == null ? null : principal.userId();
        log.info("Posts feed requested viewerId={} page={} limit={} category={} postType={} sort={}",
            viewerId, page, limit, category, postType, sort);
        validateCategory(category);
        PostType pt = parsePostType(postType);
        PostService.SortKey sortKey = PostService.SortKey.parse(sort, PostService.SortKey.BUMPED);
        PostListResponse body = postService.listFeed(
            viewerId, page, limit, category, pt, qotdId, sortKey, MDC.get("requestId"));
        return ResponseEntity.ok(body);
    }

    @GetMapping("/posts/{id}")
    public ResponseEntity<ProxyResponse<PostDto>> getById(
        @AuthenticationPrincipal AuthenticatedUser principal,
        @PathVariable UUID id
    ) {
        UUID viewerId = principal == null ? null : principal.userId();
        log.info("Post detail requested viewerId={} postId={}", viewerId, id);
        PostDto post = postService.getById(id, viewerId);
        return ResponseEntity.ok(ProxyResponse.of(post));
    }

    @GetMapping("/users/{username}/posts")
    public ResponseEntity<PostListResponse> listAuthorPosts(
        @AuthenticationPrincipal AuthenticatedUser principal,
        @PathVariable @NotBlank String username,
        @RequestParam(defaultValue = "1") @Min(1) int page,
        @RequestParam(defaultValue = "20") @Min(1) @Max(50) int limit,
        @RequestParam(required = false, defaultValue = "recent") String sort
    ) {
        UUID viewerId = principal == null ? null : principal.userId();
        log.info("Author posts requested viewerId={} username={} page={} limit={} sort={}",
            viewerId, username, page, limit, sort);
        PostService.SortKey sortKey = PostService.SortKey.parse(sort, PostService.SortKey.RECENT);
        PostListResponse body = postService.listAuthorPosts(
            username, viewerId, page, limit, sortKey, MDC.get("requestId"));
        return ResponseEntity.ok(body);
    }

    private static void validateCategory(String category) {
        if (category != null && !VALID_CATEGORIES.contains(category)) {
            throw new InvalidCategoryException(category);
        }
    }

    private static PostType parsePostType(String raw) {
        if (raw == null || raw.isBlank()) return null;
        try {
            return PostType.fromValue(raw);
        } catch (IllegalArgumentException e) {
            throw new InvalidPostTypeException(raw);
        }
    }
}

Either style is acceptable per existing convention.

**`@Validated` on the class:** required for Spring to evaluate `@Min` / `@Max` on `@RequestParam` arguments. Without it, the constraints are silently ignored.

import com.worshiproom.proxy.common.ProxyError;
import jakarta.validation.ConstraintViolationException;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.slf4j.MDC;
import org.springframework.core.Ordered;

            MDC.get("requestId"),
            Instant.now()

}
