Brief D5/D12 code samples show `extends RuntimeException`; that is wrong for this codebase. **All 4.7b exceptions MUST extend `PostException`** (Plan-Time Divergence #1).

  - **Exception advice:** `PostExceptionHandler` is `@RestControllerAdvice(basePackages = "com.worshiproom.post")`. Generic `handlePost(PostException)` handler (line 52) routes any `PostException` subclass via most-specific-handler dispatch. `InvalidHelpTagException` rides this default path (no per-class handler needed); `HelpTagsNotAllowedForPostTypeException` gets a dedicated handler (matches `ImageNotAllowedForPostTypeException` pattern).
  - **Edit-window mechanism:** `EditWindowExpiredException` extends `PostException` and emits **409 CONFLICT** with code `EDIT_WINDOW_EXPIRED` (NOT 400 as the spec brief incorrectly states in Section 3 / Section 8 / Section 11). ## Edge Cases & Decisions

| Decision | Choice | Rationale |
|---|---|---|
| HTTP code for `EDIT_WINDOW_EXPIRED` on help_tags edit | **409 CONFLICT** (NOT 400 as spec brief states) | Existing `EditWindowExpiredException` returns 409. Phase 3 Addendum item 1 explicitly requires 409 for edit-window endpoints. Spec brief Section 3 / Section 8 / Section 11 references to "400 EDIT_WINDOW_EXPIRED" are wrong; the codebase wins. (Plan-Time Divergence #2) |
| Base class for new exceptions | `extends PostException` | Spec brief D5/D12 code samples show `extends RuntimeException`. `PostException` is the canonical post-domain base (Spec 3.5+); every other post exception extends it. Using the base ensures `PostExceptionHandler.handlePost` routes the new exceptions automatically. (Plan-Time Divergence #1) |
| Where to log rejected `postType` for `HelpTagsNotAllowedForPostTypeException` | Dedicated `@ExceptionHandler` in `PostExceptionHandler` (mirrors `handleImageNotAllowed` line 42–50) | Audit trail value — operators want to see which post type tripped the gate. Generic `handlePost` works but loses the wireValue field. |
| Where to log rejected enum value for `InvalidHelpTagException` | Generic `handlePost` (no dedicated handler) | The exception's message already includes the invalid value (e.g., `"Invalid help tag: pizza"`). No additional context needed for audit. |
| Storage format | `VARCHAR(200) NOT NULL DEFAULT ''` comma-separated | Master plan body recommendation; matches D2. PostgreSQL `TEXT[]` would require array operators throughout the query layer for marginal benefit. 200 chars > 5 × longest enum value (`just_prayer` = 11 chars + 4 commas + headroom). |
| Empty Set vs null in `Post.helpTagsRaw` | `private String helpTagsRaw = ""` (initializer in entity) | W19 — Hibernate may surface NULL from a transitional row before the migration default fires. Initializer guarantees the entity instance is always coherent. |
| Canonical sort order | Server normalizes input on both create and update; frontend trusts API order | D3 / W15. Single source of truth for ordering. Frontend rendering iterates `displayableTags` directly without client-side sort. |
| Dedup behavior | Silent (Set semantics) | D4 / W8. A buggy client sending `"meals,meals"` should not get a 400. Idempotent dedup is friendlier. Unknown values DO throw — different concern. |
| Rendering rule for `just_prayer` | NEVER renders as a pill; filtered out before iteration | D8 / D6 / W5 / MPD-5. The most subtle rule. Test coverage: dedicated WaysToHelpPills test asserts `[just_prayer]` renders nothing; `[meals, just_prayer]` renders only Meals. |

`HelpTag.java`:

```java
package com.worshiproom.post;

import java.util.Arrays;
import java.util.Comparator;
import java.util.LinkedHashSet;
import java.util.Set;

public enum HelpTag {
    MEALS("meals"),
    RIDES("rides"),
    ERRANDS("errands"),
    VISITS("visits"),
    JUST_PRAYER("just_prayer");

    private final String wireValue;

    HelpTag(String wireValue) { this.wireValue = wireValue; }

    public String wireValue() { return wireValue; }

    /**
     * Parse a wire-value string into the enum, throwing
     * {@link InvalidHelpTagException} on unknown / null / empty values.
     * Case-sensitive (e.g., "MEALS" rejects).
     */
    public static HelpTag fromWireValue(String raw) {
        if (raw == null) throw new InvalidHelpTagException("null");
        if (raw.isEmpty()) throw new InvalidHelpTagException("");
        return Arrays.stream(values())
                .filter(t -> t.wireValue.equals(raw))
                .findFirst()
                .orElseThrow(() -> new InvalidHelpTagException(raw));
    }

    /** Canonical declaration order is the canonical sort order (D3). */
    public static final Comparator<HelpTag> CANONICAL_ORDER = Comparator.comparingInt(HelpTag::ordinal);
}
```

`InvalidHelpTagException.java`:

```java
package com.worshiproom.post;

import org.springframework.http.HttpStatus;

/**
 * 400 INVALID_HELP_TAG — emitted when a HelpTag wire-value is unknown.
 * Thrown from {@link HelpTag#fromWireValue(String)}. Message includes the
 * rejected value for debugging; PostExceptionHandler routes via the generic
 * handlePost(PostException) path.
 */
public class InvalidHelpTagException extends PostException {

    private final String invalidValue;

    public InvalidHelpTagException(String invalidValue) {
        super(HttpStatus.BAD_REQUEST, "INVALID_HELP_TAG",
              String.format("Invalid help tag: %s", invalidValue));
        this.invalidValue = invalidValue;
    }

    public String getInvalidValue() { return invalidValue; }
}
```

```java
/**
 * Spec 4.7b — Validate, dedupe, sort, and serialize a help_tags input.
 *
 * <p>Rules:
 * <ul>
 *   <li>null or empty input → empty string ("").</li>
 *   <li>Each non-blank value parsed via {@link HelpTag#fromWireValue} (throws on unknown).</li>
 *   <li>Whitespace-only entries silently filtered (D4).</li>
 *   <li>Duplicates silently deduped via Set semantics (D4).</li>
 *   <li>Output sorted in canonical order (D3) and joined with comma.</li>
 * </ul>
 */
private static String normalizeHelpTags(Set<String> input) {
    if (input == null || input.isEmpty()) return "";
    Set<HelpTag> parsed = new TreeSet<>(HelpTag.CANONICAL_ORDER);
    for (String raw : input) {
        if (raw == null) continue;
        String trimmed = raw.trim();
        if (trimmed.isEmpty()) continue;
        parsed.add(HelpTag.fromWireValue(trimmed));  // throws InvalidHelpTagException on unknown
    }
    return parsed.stream()
            .map(HelpTag::wireValue)
            .collect(Collectors.joining(","));
}
```
