**`verse_surfacing_log` schema:**

```sql
CREATE TABLE verse_surfacing_log (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL,
    verse_id VARCHAR(64) NOT NULL,           -- stable verse JSON entry id (e.g., "psalm-23-1")
    surfaced_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    trigger_type VARCHAR(20) NOT NULL,        -- 'post_compose' | 'comment' | 'reading_time'
    CONSTRAINT fk_verse_surfacing_log_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    CONSTRAINT chk_verse_surfacing_log_trigger CHECK (trigger_type IN ('post_compose','comment','reading_time'))
);
CREATE INDEX idx_verse_surfacing_log_user_surfaced_desc ON verse_surfacing_log (user_id, surfaced_at DESC);
```

**Note on `dismissed_at` omission (vs brief):** Brief §11 lists `dismissed_at` as a column. Plan-Time Divergence #9 removes it: cooldown lives in Redis; 3-in-a-row dismissal tracking lives in `wr_verse_dismissals` (localStorage, per brief §"Dismissal + suppression flow"). No server-side path writes the column and no read path consumes it — it would ship as dead schema. If Phase 13 analytics later wants server-side dismissal data, a dedicated spec can add the column + endpoint.

**Rollback:** `DROP TABLE verse_surfacing_log;` (index drops with the table).

**Migration safety:** Brand-new table; no existing data. `ON DELETE CASCADE` to `users(id)` ensures user-account deletion (Spec 10.11) sweeps the log.

## Edge Cases & Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| `discussion` post category — not in spec's 9 categories | Map `discussion` explicitly to empty tag set; selection returns `verse: null, reason: 'no_match'`. | Brief lists 9; `PostController.VALID_CATEGORIES` has 10. `discussion` is structurally different (auto-filled, not user-chosen). Explicit empty-set entry in `CATEGORY_TAG_MAPPING` documents the omission rather than silently 404-ing; a future contributor adding `discussion` to the mapping must consciously override the empty set. |
| Crisis-flag read seam shape | New interface `com.worshiproom.safety.CrisisFlagGate` with method `boolean isUserCrisisFlagged(UUID userId)`. Pre-Phase-10 impl `PrePhase10CrisisFlagGate implements CrisisFlagGate` queries `PostRepository.countCrisisFlaggedPostsByAuthorSince(userId, now().minus(48, HOURS))`. | Read-side gate is structurally different from `CrisisAlertService` (write/side-effect). Same package so Phase 10 can swap impls from one location. Single named interface, one wiring point — per W2/W5. The query reads from existing `Post.crisis_flag + author_id + created_at` columns; no new column needed. |
| Redis cooldown — fail OPEN vs CLOSED | Direct `RedisTemplate<String, String>` usage with `try/catch (DataAccessException)` returning "cooldown active" on the catch path. NOT `@Cacheable`. | Spec's existing `CircuitBreakingCacheManager` fails OPEN (bypasses on Redis failure). 6.8 requires the OPPOSITE per D-FailClosed: Redis unreachable → DENY surfacing. Using `@Cacheable` would inherit the fail-open pattern silently. Direct RedisTemplate makes the fail-closed branch explicit and testable. |
| Determinism: hash function | `Math.abs(userId.toString().hashCode()) + dayOfYear` then `% filteredSet.size()`. | `String.hashCode()` is specified stable across JVMs (Javadoc contract). `UUID.hashCode()` mixes timestamp bits — less stable across reboots if UUIDs get re-cached. `Math.abs` handles `Integer.MIN_VALUE` edge case explicitly. |
| Day-of-year timezone | UTC. `LocalDate.now(ZoneOffset.UTC).getDayOfYear()`. | Mirrors `QotdService` (canonical daily-rotation pattern in the codebase). Determinism crosses day boundary at 00:00 UTC for all users. |
| `wr_settings.verseFindsYou.enabled` namespace | Top-level (NOT nested under `prayerWall.*`). | Brief §11 is explicit. The brief's reasoning: Verse-Finds-You is a cross-surface feature (Prayer Wall triggers + future surfaces) and warrants its own namespace; `prayerWall.*` is for Prayer-Wall-specific UI prefs. `deepMerge` in `settings-storage.ts` handles existing-user defaults correctly. |
| Reading-time "active engagement" signal | Foreground tab (`document.visibilityState === 'visible'`) + scroll event within the last 30s + accumulated active time > 5 minutes. Counter resets when tab backgrounded; resumes when foregrounded. | Brief §6 R6: "foreground tab + scroll activity, not an idle tab." Wall-clock alone catches idle tabs left open; visibility + scroll signal catches active engagement. 30s scroll-event window is the de-facto Prayer Wall scroll cadence (cards are tall; users scroll ~every 10–20s when actively reading). |
| Reading-time trigger scope: PrayerWall vs PrayerWallDashboard | `/prayer-wall` only. Excluded from `/prayer-wall/dashboard`. | Brief §"Affected Frontend Routes" mentioned `dashboard` as Plan-Recon to confirm. Decision: dashboard is the user's OWN feed (their posts, their bookmarks), not a public feed. Surfacing scripture to a user browsing their own past posts is a different emotional context than surfacing while reading others' prayers. Plan-Time Divergence #5 documents this. |
| Verse stable id format | `book-chapter-startVerse-endVerse` (kebab-case), e.g. `psalm-23-1`, `lamentations-3-22-23`. Single-verse entries use just the start. | Stable across translations (translation locked to WEB), URL-safe, log-greppable. The 180 entries in `verse-finds-you.json` carry an explicit `id` field; CC's schema enforces this. |
| `excluded_contexts` enforcement | Step 5 filters BOTH `tags` (intersection with context tags must be non-empty) AND `excluded_contexts` (intersection with context tags must be empty). A verse `hope`-tagged with `excluded_contexts: ['grief']` queried against a grief context is filtered OUT. | Defensive lever per D-ExcludedContexts; testable directly per T5. |
| Cooldown Redis key shape | `verse:cooldown:<userId>` with TTL of 24h. Value: ISO8601 `surfaced_at` timestamp string (used to compute `cooldown_until` for response). | Single key per user, no scan; TTL handles expiration without manual cleanup. Format matches existing `RedisTemplate<String, String>` serializer config. |
| Session-suppression after dismissal (frontend) | Local React state inside `useVerseFindsYou` hook — NOT persisted across page reloads. | Session = the React app lifecycle. Reloading the page is a "new session" semantically. The 24h cooldown (server-side) is the cross-session enforcement; session-suppression is the within-session UX. |
| 3-in-a-row dismissal counter (`wr_verse_dismissals`) | Plain `{ count: number, lastResetAt: number }` in localStorage. Increment on dismissal; reset to 0 on engagement (user does NOT dismiss the surfaced verse, OR taps Save). Surface the off-ramp prompt exactly once when count reaches 3; persist a `promptShown: boolean` to avoid re-showing. | Simple plain-object storage matches the spec's pattern for `wr_*` keys. No reactive store needed (single consumer). |
| Save-this-verse rate limit (client-side) | Track `savedVerseIds: string[]` in component-local state for the session. Save button disables after first save for the same verseId. Backend imposes no separate rate limit — Save is a pure client-side PNG generation, no server call. | Spec §"Save-this-verse" says "Rate-limited to 1 save per verse per user", and "No server-side image logging." Client-side per-session enforcement is consistent with "no server-side image logging" — there's no server endpoint to rate-limit. Cross-session re-save is fine (the PNG itself is deterministic given the verse). |

`VerseSurfacingLog.java`:

```java
package com.worshiproom.verse;

import jakarta.persistence.*;
import java.time.OffsetDateTime;
import java.util.UUID;

@Entity
@Table(name = "verse_surfacing_log")
public class VerseSurfacingLog {

    @Id
    @GeneratedValue(strategy = GenerationType.UUID)
    private UUID id;

    @Column(name = "user_id", nullable = false)
    private UUID userId;

    @Column(name = "verse_id", nullable = false, length = 64)
    private String verseId;

    @Column(name = "surfaced_at", nullable = false, insertable = false, updatable = false)
    private OffsetDateTime surfacedAt;

    @Column(name = "trigger_type", nullable = false, length = 20)
    private String triggerType;

    protected VerseSurfacingLog() {}

    public VerseSurfacingLog(UUID userId, String verseId, String triggerType) {
        this.userId = userId;
        this.verseId = verseId;
        this.triggerType = triggerType;
    }

    public UUID getId() { return id; }
    public UUID getUserId() { return userId; }
    public String getVerseId() { return verseId; }
    public OffsetDateTime getSurfacedAt() { return surfacedAt; }
    public String getTriggerType() { return triggerType; }
}
```

`VerseSurfacingLogRepository.java`:

```java
package com.worshiproom.verse;

import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Modifying;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;
import org.springframework.stereotype.Repository;

import java.time.OffsetDateTime;
import java.util.UUID;

@Repository
public interface VerseSurfacingLogRepository extends JpaRepository<VerseSurfacingLog, UUID> {

    @Modifying(clearAutomatically = true, flushAutomatically = true)
    @Query("DELETE FROM VerseSurfacingLog v WHERE v.surfacedAt < :cutoff")

}
```

- Do NOT name the repo method something Spring Data can derive a query for if the query body differs — the explicit `@Query` is required for the cooldown lookup because we need ordering.

```java
import com.fasterxml.jackson.core.JsonParser;

import org.slf4j.Logger;

/**
 * Loads + caches the curated verse set from
 * {@code backend/src/main/resources/verses/verse-finds-you.json}.
 *
 * <p>On parse failure, returns an EMPTY catalog so the surfacing pipeline returns
 * {@code verse: null, reason: 'no_match'} (Gate-G-SILENT-FAILURE / D-SilentFailure)
 * rather than crashing.

@Component
public class VerseCatalog {

    private static final Logger log = LoggerFactory.getLogger(VerseCatalog.class);
    private static final String RESOURCE_PATH = "verses/verse-finds-you.json";

    private List<CuratedVerse> entries = List.of();
    private Map<String, List<String>> categoryTagMapping = Map.of();
    private Set<String> knownTags = Set.of();

    @PostConstruct
    void load() {
        try {
            ObjectMapper mapper = new ObjectMapper();
            mapper.configure(JsonParser.Feature.ALLOW_COMMENTS, true);
            mapper.configure(JsonParser.Feature.ALLOW_TRAILING_COMMA, true);
            CatalogFile parsed = mapper.readValue(
                new ClassPathResource(RESOURCE_PATH).getInputStream(),
                CatalogFile.class
            );
            this.entries = parsed.entries() == null ? List.of() : parsed.entries();
            this.categoryTagMapping = parsed.categoryTagMapping() == null ? Map.of() : parsed.categoryTagMapping();
            this.knownTags = parsed.knownTags() == null ? Set.of() : Set.copyOf(parsed.knownTags());
            log.info("verseCatalogLoaded entries={} categories={} knownTags={}",
                entries.size(), categoryTagMapping.size(), knownTags.size());
        } catch (Exception e) {
            // Catch BOTH IOException (file missing, IO error) AND RuntimeException subclasses
            // like JsonParseException (malformed JSON, extra comma, unbalanced quotes). Per
            // Gate-G-SILENT-FAILURE and T17 (JSON parse failure → no_match, no 500), the
            // catalog wrapper MUST never let a parse failure escape @PostConstruct — that
            // would crash Spring context startup. An empty catalog is the safe degraded state.
            log.error("verseCatalogParseFailure path={} message={}", RESOURCE_PATH, e.getMessage());
            // Silent failure: empty catalog → pipeline returns no_match
        }
    }

    public List<CuratedVerse> entries() { return entries; }
    public Map<String, List<String>> categoryTagMapping() { return categoryTagMapping; }
    public Set<String> knownTags() { return knownTags; }

        @JsonProperty("category_tag_mapping") Map<String, List<String>> categoryTagMapping,
        @JsonProperty("known_tags") List<String> knownTags,
```

```java
package com.worshiproom.verse;

import org.springframework.stereotype.Component;

import java.time.LocalDate;
import java.time.ZoneOffset;
import java.util.*;

/**
 * The 7-step deterministic selection algorithm for Spec 6.8.
 *
 * <p>Step 1 (crisis-flag) and Step 2 (cooldown) and Step 3 (disabled) live in
 * {@link VerseFindsYouService} — gating logic that returns null with a reason
 * code before the engine is even called.
 *
 * <p>This engine handles steps 4–7: tag derivation from category context,
 * filtering, deterministic selection.
 */
@Component
public class VerseSelectionEngine {

    private final VerseCatalog catalog;

    public VerseSelectionEngine(VerseCatalog catalog) {
        this.catalog = catalog;
    }

    /**
     * @param userId  caller user id (used in deterministic hash)
     * @param category the post category (e.g. "grief", "mental-health"); may be null
     * @return Optional.empty() if no verse matches the context; populated otherwise.
     */
    public Optional<CuratedVerse> select(UUID userId, String category) {
        if (category == null || category.isBlank()) return Optional.empty();
        List<String> contextTags = catalog.categoryTagMapping().get(category);
        if (contextTags == null || contextTags.isEmpty()) return Optional.empty(); // Step 4 — no mapping → no_match

        Set<String> contextSet = new HashSet<>(contextTags);

        // Step 5 — filter to entries matching ANY context tag AND NOT in excluded_contexts
        List<CuratedVerse> filtered = catalog.entries().stream()
            .filter(v -> v.tags().stream().anyMatch(contextSet::contains))
            .filter(v -> v.excludedContexts() == null || v.excludedContexts().stream().noneMatch(contextSet::contains))
            .toList();

        if (filtered.isEmpty()) return Optional.empty();

        // Step 6 — deterministic selection: (Math.abs(userId.toString().hashCode()) + dayOfYear) % size
        int dayOfYear = LocalDate.now(ZoneOffset.UTC).getDayOfYear();
        int hashComponent = Math.abs(userId.toString().hashCode());
        int index = (hashComponent + dayOfYear) % filtered.size();

        // Step 7 — return
        return Optional.of(filtered.get(index));
    }
}
```

- Do NOT introduce randomness anywhere in the selection — determinism is load-bearing per D-Determinism.
- Do NOT use `UUID.hashCode()` directly — it mixes timestamp bits. Use `userId.toString().hashCode()` (stable per Java spec).
- Do NOT use `LocalDate.now()` without timezone — must be UTC to match QotdService canonical pattern.
- Do NOT add a "fallback to default verse" branch on any filter-empty case.

```java
package com.worshiproom.verse;

import com.worshiproom.safety.CrisisFlagGate;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.dao.DataAccessException;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.time.Duration;
import java.time.OffsetDateTime;
import java.time.ZoneOffset;
import java.util.Optional;
import java.util.UUID;

/**
 * Orchestrates the 7-step Verse-Finds-You surfacing pipeline (Spec 6.8).
 *
 * <p>Step 1 — crisis suppression via {@link CrisisFlagGate} (currently a 48h read of the
 * user's own crisis-flagged posts; Phase 10 widens this to all crisis-detection sources).
 * Step 2 — 24h cooldown via Redis (FAIL CLOSED: Redis unreachable → return cooldown).
 * Step 3 — disabled toggle (passed in by caller from settings).
 * Steps 4–7 — delegated to {@link VerseSelectionEngine}.
 *
 * <p>All non-surfacing paths return a {@link SurfacingResult} with {@code verse=empty}
 * and a {@link SurfacingReason}. NO failure path throws to the controller; the only
 * exceptions that propagate are auth (401) and rate-limit (429) — both handled at the
 * filter layer.
 */
@Service
public class VerseFindsYouService {

    private static final Logger log = LoggerFactory.getLogger(VerseFindsYouService.class);
    private static final Duration COOLDOWN_DURATION = Duration.ofHours(24);
    private static final String COOLDOWN_KEY_PREFIX = "verse:cooldown:";

    private final CrisisFlagGate crisisFlagGate;
    private final RedisTemplate<String, String> redisTemplate;
    private final VerseSelectionEngine engine;
    private final VerseSurfacingLogRepository logRepository;

    public VerseFindsYouService(
        CrisisFlagGate crisisFlagGate,
        RedisTemplate<String, String> redisTemplate,
        VerseSelectionEngine engine,
        VerseSurfacingLogRepository logRepository
    ) {
        this.crisisFlagGate = crisisFlagGate;
        this.redisTemplate = redisTemplate;
        this.engine = engine;
        this.logRepository = logRepository;
    }

    @Transactional
    public SurfacingResult surface(UUID userId, TriggerType trigger, String category, boolean enabled) {
        // Step 3 — toggle (cheap; do first to avoid Redis cost when disabled)
        if (!enabled) return SurfacingResult.disabled();

        // Step 1 — crisis suppression (Gate-G-CRISIS-SEAM)
        try {
            if (crisisFlagGate.isUserCrisisFlagged(userId)) {
                return SurfacingResult.crisis();
            }
        } catch (RuntimeException e) {
            // Pre-Phase-10 no-flag-source path: if the gate impl ever fails (e.g., DB blip),
            // treat as "no flag readable" per Brief §0 — fall through, do NOT crash.
            log.warn("crisisFlagGateFailure userId={} message={}", userId, e.getMessage());
        }

        // Step 2 — Redis cooldown (FAIL CLOSED per D-FailClosed)
        OffsetDateTime lastSurfaced;
        try {
            String stored = redisTemplate.opsForValue().get(COOLDOWN_KEY_PREFIX + userId);
            lastSurfaced = stored == null ? null : OffsetDateTime.parse(stored);
        } catch (DataAccessException e) {
            // Redis unreachable → DENY (D-FailClosed). NOT bypass-and-allow.
            log.warn("verseCooldownRedisFailureFailingClosed userId={} message={}", userId, e.getMessage());
            return SurfacingResult.cooldown(OffsetDateTime.now(ZoneOffset.UTC).plus(COOLDOWN_DURATION));
        } catch (RuntimeException e) {
            // Malformed stored value (parse failure) → safe degrade: treat as no cooldown, fall through.
            // This is NOT a Redis failure — it's a data-shape failure that shouldn't deny surfacing.
            log.warn("verseCooldownParseFailure userId={} message={}", userId, e.getMessage());
            lastSurfaced = null;
        }
        if (lastSurfaced != null && lastSurfaced.plus(COOLDOWN_DURATION).isAfter(OffsetDateTime.now(ZoneOffset.UTC))) {
            return SurfacingResult.cooldown(lastSurfaced.plus(COOLDOWN_DURATION));
        }

        // Steps 4–7 — selection
        Optional<CuratedVerse> selected = engine.select(userId, category);
        if (selected.isEmpty()) return SurfacingResult.noMatch();

        CuratedVerse verse = selected.get();

        // Persist: Redis cooldown key (TTL 24h) + verse_surfacing_log row.
        // Redis failure on write is logged-and-swallowed — selection succeeded; the cooldown
        // miss is a degradation, not a denial. The next selection within 24h would over-surface,
        // which we treat as acceptable vs denying a verse the user is entitled to.
        try {
            redisTemplate.opsForValue().set(
                COOLDOWN_KEY_PREFIX + userId,
                OffsetDateTime.now(ZoneOffset.UTC).toString(),
                COOLDOWN_DURATION
            );
        } catch (DataAccessException e) {
            log.warn("verseCooldownRedisWriteFailure userId={} message={}", userId, e.getMessage());
        }

        VerseSurfacingLog logEntry = new VerseSurfacingLog(userId, verse.id(), trigger.dbValue());
        logRepository.save(logEntry);

        return SurfacingResult.success(verse);
    }
}
```

```java
import com.worshiproom.auth.AuthenticatedUser;
import com.worshiproom.verse.dto.VerseFindsYouResponse;
import com.worshiproom.verse.dto.VerseDto;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.http.ResponseEntity;
import org.springframework.security.core.annotation.AuthenticationPrincipal;
import org.springframework.web.bind.annotation.*;

import java.util.UUID;

@RestController
@RequestMapping("/api/v1")
public class VerseFindsYouController {

    private static final Logger log = LoggerFactory.getLogger(VerseFindsYouController.class);

    private final VerseFindsYouService service;
    private final VerseFindsYouRateLimitService rateLimit;

    public VerseFindsYouController(VerseFindsYouService service,
                                    VerseFindsYouRateLimitService rateLimit) {
        this.service = service;
        this.rateLimit = rateLimit;
    }

    @GetMapping("/verse-finds-you")
    public ResponseEntity<VerseFindsYouResponse> get(
        @AuthenticationPrincipal AuthenticatedUser principal,
        @RequestParam("trigger") String trigger,
        @RequestParam(value = "context", required = false) String context,
        @RequestParam(value = "enabled", required = false, defaultValue = "false") boolean enabled
    ) {
        UUID userId = principal.userId();
        rateLimit.consumeOrThrow(userId);

        TriggerType triggerType;
        try {
            triggerType = TriggerType.fromQueryParam(trigger);
        } catch (IllegalArgumentException e) {
            // Invalid trigger → 400 INVALID_INPUT (handled by VerseExceptionHandler)
            throw new InvalidTriggerException(trigger);
        }

        log.info("verseFindsYouRequested userId={} trigger={} context={} enabled={}",
            userId, trigger, context, enabled);

        SurfacingResult result = service.surface(userId, triggerType, context, enabled);

        VerseFindsYouResponse body = new VerseFindsYouResponse(
            result.verse().map(v -> new VerseDto(v.reference(), v.text())).orElse(null),
            result.cooldownUntil(),
            result.reason() == null ? null : result.reason().name().toLowerCase()
        );

        return ResponseEntity.ok(body);
    }
}
```

```java
import org.springframework.transaction.annotation.Transactional;

@Component
public class VerseSurfacingLogCleanupJob {

    private static final Logger log = LoggerFactory.getLogger(VerseSurfacingLogCleanupJob.class);
    static final Duration RETENTION = Duration.ofDays(30);

    private final VerseSurfacingLogRepository repository;

    public VerseSurfacingLogCleanupJob(VerseSurfacingLogRepository repository) {

    @Scheduled(cron = "0 30 3 * * ?", zone = "UTC")

    public void cleanup() {
        OffsetDateTime cutoff = OffsetDateTime.now(ZoneOffset.UTC).minus(RETENTION);
        int deleted = repository.deleteOlderThan(cutoff);
        if (deleted > 0) {
            log.info("verseSurfacingLogPruned count={}", deleted);
        }
    }
}
```

```typescript
const KEY = 'wr_verse_dismissals'

export function getVerseDismissals(): { count: number; promptShown: boolean } {
  try {
    const raw = localStorage.getItem(KEY)
    if (!raw) return { count: 0, promptShown: false }
    const parsed = JSON.parse(raw)
    return {
      count: typeof parsed?.count === 'number' ? parsed.count : 0,
      promptShown: parsed?.promptShown === true,
    }
  } catch { return { count: 0, promptShown: false } }
}

export function incrementDismissalCount(): { count: number; promptShown: boolean } {
  const current = getVerseDismissals()
  const next = { count: current.count + 1, promptShown: current.promptShown }
  saveDismissals(next)
  return next
}

export function resetDismissalCount(): void {
  saveDismissals({ count: 0, promptShown: false })
}

export function markPromptShown(): void {
  const current = getVerseDismissals()
  saveDismissals({ count: current.count, promptShown: true })
}

function saveDismissals(v: { count: number; promptShown: boolean }): void {
  try { localStorage.setItem(KEY, JSON.stringify(v)) } catch {}
}
```

```typescript
import { useCallback, useRef, useState } from 'react'
import { useSettings } from './useSettings'
import { apiFetch } from '@/services/api'   // existing api wrapper
import type { TriggerType, VerseSurfacingResult } from '@/types/verse-finds-you'
import { incrementDismissalCount, resetDismissalCount, markPromptShown } from '@/services/verse-dismissals-storage'

export function useVerseFindsYou() {
  const { settings } = useSettings()
  const [verse, setVerse] = useState<VerseSurfacingResult | null>(null)
  const sessionSuppressed = useRef(false)
  const savedVerseIds = useRef<Set<string>>(new Set())

  const trigger = useCallback(async (triggerType: TriggerType, context?: string) => {
    if (!settings.verseFindsYou.enabled) return  // Gate-G-DEFAULT-OFF / W28 — NO API call
    if (sessionSuppressed.current) return        // Session-level suppression after a dismissal
    const params = new URLSearchParams({
      trigger: triggerType,
      enabled: 'true',
      ...(context ? { context } : {}),
    })
    try {
      const result = await apiFetch<VerseSurfacingResult>(`/api/v1/verse-finds-you?${params}`)
      if (result.verse) {
        setVerse(result)
      } else {
        setVerse(null)
      }
    } catch {
      // 401, 429, network — silent failure on the client too (W10)
      setVerse(null)
    }
  }, [settings.verseFindsYou.enabled])

  const dismiss = useCallback(() => {
    setVerse(null)
    sessionSuppressed.current = true
    const next = incrementDismissalCount()
    if (next.count >= 3 && !next.promptShown) {
      return { showOffRampPrompt: true }
    }
    return { showOffRampPrompt: false }
  }, [])

  const acknowledgeEngagement = useCallback(() => {
    resetDismissalCount()
  }, [])

  const acknowledgePromptShown = useCallback(() => {
    markPromptShown()
  }, [])

  const markSaved = useCallback((verseId: string) => {
    savedVerseIds.current.add(verseId)
    resetDismissalCount()   // saving counts as engagement
  }, [])

  const wasSaved = useCallback((verseId: string) => savedVerseIds.current.has(verseId), [])

  return { verse, trigger, dismiss, acknowledgeEngagement, acknowledgePromptShown, markSaved, wasSaved }
}
```

### Step 14: Trigger insertion — reading-time on `PrayerWall.tsx`

**Objective:** Detect >5 minutes of active engagement on `/prayer-wall` (foreground tab + scroll-event-within-last-30s) and fire the reading-time trigger using the last-viewed post's category as context.

**Files to modify:**
- `frontend/src/pages/PrayerWall.tsx` — add an active-engagement timer effect.
- `frontend/src/hooks/useActiveEngagement.ts` — NEW hook that tracks foreground time + recent-scroll signal and fires a callback after a configurable threshold.

**Details:**

`useActiveEngagement.ts`:

```typescript
import { useEffect, useRef } from 'react'

interface Options {
  thresholdMs: number          // e.g. 5 * 60 * 1000
  scrollWindowMs: number       // e.g. 30 * 1000 — "active" requires scroll within this window
  onActive: () => void
}

export function useActiveEngagement({ thresholdMs, scrollWindowMs, onActive }: Options): void {
  const accumulatedActiveMs = useRef(0)
  const lastTickAt = useRef<number | null>(null)
  const lastScrollAt = useRef<number>(Date.now())
  const fired = useRef(false)

  useEffect(() => {
    const onScroll = () => { lastScrollAt.current = Date.now() }
    window.addEventListener('scroll', onScroll, { passive: true })
    const tick = setInterval(() => {
      const now = Date.now()
      const inForeground = document.visibilityState === 'visible'
      const recentScroll = now - lastScrollAt.current < scrollWindowMs
      if (inForeground && recentScroll && lastTickAt.current != null) {
        accumulatedActiveMs.current += now - lastTickAt.current
      }
      lastTickAt.current = now
      if (!fired.current && accumulatedActiveMs.current >= thresholdMs) {
        fired.current = true
        onActive()
      }
    }, 1000)
    return () => {
      window.removeEventListener('scroll', onScroll)
      clearInterval(tick)
    }
  }, [thresholdMs, scrollWindowMs, onActive])
}
```

**Last-viewed category tracking — committed approach:** Use a single `IntersectionObserver` instantiated at the `PrayerWall.tsx` page level that observes a `data-prayer-category` attribute on each rendered `PrayerCard`. On each intersection-changed callback, the category of the LAST card to cross the `isIntersecting=true` threshold (within the top 60% of the viewport) wins and is written to `lastViewedCategoryRef.current`. Plain ref — no state, no re-render. **Do NOT add an `onVisible` prop to `PrayerCard`** — that prop-drills observation responsibility onto every card and creates one observer per card (perf regression). One page-level observer, queried by attribute. Modify `PrayerCard.tsx` only to add `data-prayer-category={category}` to its outer wrapper element (a one-line change, no prop addition).

`PrayerWall.tsx`:

```tsx

useEffect(() => {
  const observer = new IntersectionObserver(
    (entries) => {
      // Pick the last entry that crossed into view (most recently scrolled-past)
      for (const e of entries) {
        if (e.isIntersecting) {
          const category = e.target.getAttribute('data-prayer-category')
          if (category) lastViewedCategoryRef.current = category
        }
      }
    },
    { threshold: 0.6 }
  )
  document.querySelectorAll('[data-prayer-category]').forEach((el) => observer.observe(el))
  // NB: re-observe when the feed list mutates — use a MutationObserver or re-run
  // this effect with a dependency on the feed-length to re-attach.
  return () => observer.disconnect()
}, [posts.length])

useActiveEngagement({
  thresholdMs: 5 * 60 * 1000,
  scrollWindowMs: 30 * 1000,
  onActive: () => {

```
