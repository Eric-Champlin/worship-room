## Architecture Context

**Composers in scope (Plan-Recon R1):**
- `frontend/src/components/prayer-wall/InlineComposer.tsx` ‚Äî single component, switches behavior via `postType` prop across the 5 canonical post types (`prayer_request | testimony | question | discussion | encouragement`). NOT unmounted on close ‚Äî `isOpen` toggles `aria-hidden` + `inert` + `max-h-0`/`max-h-[800px]`. Same component instance receives different `postType` prop values across the user's session.
- `frontend/src/components/prayer-wall/QotdComposer.tsx` ‚Äî separate composer for QOTD responses. Also not unmounted on close. Submits as `postType: 'discussion'` with a `qotdId` link, but the draft is conceptually QOTD-scoped, so it gets a synthetic key `'qotd'` (NOT `'discussion'`) ‚Äî a draft typed into the QOTD composer should NOT clobber an in-progress discussion draft and vice versa.

**Composers NOT in scope (Plan-Recon R1 exclusion):**
- `CommentInput.tsx` ‚Äî single-line input for comments, per-`prayerId`. Drafting a single sentence per individual post under 500 chars doesn't carry the same emotional/time investment as a multi-paragraph composer. Excluded to (a) keep scope tight per Gate-G-NO-SCOPE-CREEP, (b) preserve 6.6b-deferred-1 territory unchanged.
- `ComposerChooser.tsx` ‚Äî chooser only, no textarea.
- `ScriptureReferenceInput.tsx` ‚Äî sub-component, not a draft surface.

**Existing patterns to match:**
- **Storage service module:** `frontend/src/services/verse-dismissals-storage.ts` (Spec 6.8) ‚Äî pure CRUD, try/catch around `localStorage.getItem`/`setItem`, exports named functions plus a `*_KEY` constant. No reactive subscribe ‚Äî single-consumer per draft entry.
- **Draft-textarea pattern:** `wr_journal_draft` and `wr_prayer_draft` (Pray/Journal tabs) ‚Äî per-device, no userId tagging, draft survives auth modal interaction. Worship Room's existing draft idiom.
- **CRUD verb conventions** (`04-frontend-standards.md` ¬ß "CRUD Verb Conventions"): `get*` for sync reads, `set*` / `upsert*` for writes, `remove*` for delete. Hook returns are state, not actions; service is the action surface.
- **AuthContext logout:** `frontend/src/contexts/AuthContext.tsx` ‚Äî `logout()` clears auth state but preserves user data. We extend it with a single `clearAllComposerDrafts()` call so logout DOES clear drafts (per brief R6 conservative default + advisor note: simplest defensible rule, matches the limited blast radius of localStorage drafts).
- **FrostedCard tier system** (`09-design-system.md` ¬ß "FrostedCard Tier System"): the restore prompt uses `FrostedCard variant="default"` with `rounded-3xl`, `border-white/[0.12]`, `bg-white/[0.07]` (post-Visual-Rollout values).
- **Button variants:** `<Button variant="gradient" size="sm">Restore draft</Button>` for the primary action, `<Button variant="ghost" size="sm">Start fresh</Button>` for secondary.
- **Animation tokens:** `frontend/src/constants/animation.ts` ‚Äî `ANIMATION_DURATIONS.base` (`'base'` Tailwind class = 250 ms) for prompt fade-in, `ANIMATION_EASINGS.decelerate` for entering motion.

**Post-type enum (Plan-Recon R1 reconciliation):**
- Canonical name: `PostType` from `frontend/src/constants/post-types.ts` (5 values: `prayer_request | testimony | question | discussion | encouragement`).
- The brief refers to "the post-type enum" generically; the spec-time recon table flagged that `PostTypeApi` (a sibling string-literal union in `types/api/prayer-wall.ts`) shares the same five strings. The InlineComposer prop signature uses `PostType`, so the hook keys on `PostType` for clarity. - `useAuth()` returns `{ isAuthenticated: boolean, user: AuthUser | null }`.
- `AuthUser.id` is the canonical user identifier (`string`).
- `AuthContext.logout()` is the cleanup entry point.

### Step 1: Create `composer-drafts-storage.ts` service module

**Objective:** Pure CRUD module for the `wr_composer_drafts` localStorage key. Matches the `verse-dismissals-storage.ts` pattern exactly.

**Files to create/modify:**
- `frontend/src/services/composer-drafts-storage.ts` ‚Äî NEW. Exports: `COMPOSER_DRAFTS_KEY`, `DRAFT_EXPIRY_MS`, type `DraftKey`, type `DraftRecord`, type `DraftStore`, functions `getDraft(key)`, `setDraft(key, content)`, `removeDraft(key)`, `clearAllComposerDrafts()`, `isDraftExpired(record)`.

**Details:**

```ts
// services/composer-drafts-storage.ts
import type { PostType } from '@/constants/post-types'

export const COMPOSER_DRAFTS_KEY = 'wr_composer_drafts'
/** 7 days in milliseconds ‚Äî drafts older than this are silently discarded. */
export const DRAFT_EXPIRY_MS = 7 * 24 * 60 * 60 * 1000

/**
 * Composer draft key. `PostType` values plus a synthetic `'qotd'` for the
 * separate QOTD composer (whose draft is conceptually distinct from a
 * `'discussion'` draft despite the same submit type).
 */
export type DraftKey = PostType | 'qotd'

export interface DraftRecord {
  content: string
  updatedAt: number
}

export type DraftStore = Partial<Record<DraftKey, DraftRecord>>

const EMPTY: DraftStore = {}

function readStore(): DraftStore {
  try {
    const raw = localStorage.getItem(COMPOSER_DRAFTS_KEY)
    if (!raw) return { ...EMPTY }
    const parsed = JSON.parse(raw)
    if (!parsed || typeof parsed !== 'object') return { ...EMPTY }
    return parsed as DraftStore
  } catch {
    return { ...EMPTY }
  }
}

function writeStore(store: DraftStore): void {
  try {
    localStorage.setItem(COMPOSER_DRAFTS_KEY, JSON.stringify(store))
  } catch {
    // localStorage may be unavailable (private browsing, quota); silent failure
    // is acceptable ‚Äî drafts are courtesy, not contract.
  }
}

export function isDraftExpired(record: DraftRecord, now: number = Date.now()): boolean {
  return now - record.updatedAt > DRAFT_EXPIRY_MS
}

/**
 * Returns the draft for `key` if it exists AND is within the 7-day TTL.
 * Expired drafts are silently removed as a side effect and `null` is returned.
 */
export function getDraft(key: DraftKey): DraftRecord | null {
  const store = readStore()
  const record = store[key]
  if (!record) return null
  if (isDraftExpired(record)) {
    removeDraft(key)
    return null
  }
  return record
}

export function setDraft(key: DraftKey, content: string): void {
  const store = readStore()
  store[key] = { content, updatedAt: Date.now() }
  writeStore(store)
}

export function removeDraft(key: DraftKey): void {
  const store = readStore()
  if (!(key in store)) return
  delete store[key]
  writeStore(store)
}

export function clearAllComposerDrafts(): void {
  try {
    localStorage.removeItem(COMPOSER_DRAFTS_KEY)
  } catch {
    // Same as writeStore ‚Äî silent failure is acceptable.
  }
}
```

**Guardrails (DO NOT):**
- DO NOT export raw `localStorage` access; only the named functions.
- DO NOT throw on missing/malformed data ‚Äî return empty.
- DO NOT log draft content anywhere (W8 ‚Äî no PII leak via debug output).
- DO NOT add userId tagging ‚Äî clear-on-logout (R6) is the chosen rule.
- DO NOT add subscribe/notify ‚Äî this is a CRUD module, not a reactive store. Each composer is the sole consumer of its own postType slot.

### Step 2: Create `RestoreDraftPrompt.tsx` component

**Objective:** Inline-card UI with "You have a saved draft from {timeAgo}. Restore it?" + Restore / Start fresh buttons. Mounted by the composer when a draft exists.

**Details:**

```tsx
import { useEffect, useRef } from 'react'
import { FrostedCard } from '@/components/homepage/FrostedCard'
import { Button } from '@/components/ui/Button'
import { timeAgo } from '@/lib/time'

export interface RestoreDraftPromptProps {
  /** Timestamp (ms) when the draft was last updated. Drives the relative label. */
  draftTimestamp: number
  /** Called when the user taps "Restore draft". Caller is responsible for
   *  applying the draft content to its own state. */
  onRestore: () => void
  /** Called when the user taps "Start fresh". Caller is responsible for
   *  discarding the draft from storage. */
  onDiscard: () => void
}

export function RestoreDraftPrompt({
  draftTimestamp,
  onRestore,
  onDiscard,
}: RestoreDraftPromptProps) {
  const restoreButtonRef = useRef<HTMLButtonElement>(null)

  // Move focus to the primary button when the prompt appears so keyboard
  // users land on the recommended action. NOT a focus trap ‚Äî the prompt is
  // a status announcement, not a modal.
  useEffect(() => {
    restoreButtonRef.current?.focus()
  }, [])

  return (
    <div
      role="status"
      aria-live="polite"
      className="mb-3 motion-safe:animate-fade-in"
    >
      <FrostedCard variant="default" as="div" className="p-4">
        <p className="text-sm text-white/80">
          You have a saved draft from {timeAgo(new Date(draftTimestamp))}. Restore it?
        </p>
        <div className="mt-3 flex items-center gap-2">
          <Button
            ref={restoreButtonRef}
            type="button"
            variant="gradient"
            size="sm"
            onClick={onRestore}
            className="min-h-[44px]"
          >
            Restore draft
          </Button>
          <Button
            type="button"
            variant="ghost"
            size="sm"
            onClick={onDiscard}
            className="min-h-[44px]"
          >
            Start fresh
          </Button>
        </div>
      </FrostedCard>
    </div>
  )
}
```

**Objective:** React hook that ties the storage service to a composer's lifecycle. Handles auto-save (5s debounce via dirty flag), restore prompt visibility, clear-on-success, expiry check on open, and `postType` change mid-lifecycle.

**Files to create/modify:**
- `frontend/src/hooks/useComposerDraft.ts` ‚Äî NEW.

**Details:**

```ts
// hooks/useComposerDraft.ts
import { useEffect, useRef, useState } from 'react'
import {
  type DraftKey,
  type DraftRecord,
  getDraft,
  setDraft,
  removeDraft,
} from '@/services/composer-drafts-storage'

/** Auto-save tick interval in ms. Brief ¬ß2 + W1 ‚Äî every 5 seconds, dirty-flag pattern. */
export const COMPOSER_DRAFT_TICK_MS = 5000

export interface UseComposerDraftOptions {
  /** Draft key ‚Äî a PostType value, or 'qotd' for the QOTD composer. */
  draftKey: DraftKey
  /** Current textarea content. The hook reads this on each tick. */
  content: string
  /** Whether the composer is currently open/visible. Auto-save and restore-prompt
   *  visibility gate on this flag (NOT on React component mount ‚Äî composers
   *  stay mounted across open/close cycles via aria-hidden/inert). */
  isOpen: boolean
}

export interface UseComposerDraftResult {
  /** The draft to offer for restoration. `null` when no fresh draft exists,
   *  when the user has dismissed the prompt this open session, or when the
   *  user has already restored (caller transitions UI). */
  draftToRestore: DraftRecord | null
  /** Called when the user taps "Restore draft" ‚Äî returns the saved content
   *  for the caller to apply to its own state. Also dismisses the prompt
   *  (subsequent renders return draftToRestore: null for this open session). */
  restoreDraft: () => string
  /** Called when the user taps "Start fresh" ‚Äî removes the draft from storage
   *  and dismisses the prompt. */
  discardDraft: () => void
  /** Called by the composer on successful submit ‚Äî removes the draft from
   *  storage. Idempotent: safe to call when no draft exists. */
  clearDraft: () => void
}

/**
 * Per-composer draft hook. Auto-saves textarea content to localStorage every
 * 5 seconds while the user is actively editing (dirty-flag pattern, not
 * per-keystroke throttle).
 *
 * Lifecycle:
 *  - On `isOpen` transition false ‚Üí true: re-reads the draft for `draftKey`
 *    (which may have changed since the last open). If a fresh (<7-day) draft
 *    exists, `draftToRestore` is non-null until the user resolves it.
 *  - On `draftKey` change (e.g., InlineComposer's postType prop changed): same
 *    as a reopen ‚Äî re-read the draft for the new key, reset dirty flag.
 *  - On content change: marks dirty.
 *  - Every 5 seconds while `isOpen`: if dirty AND content.trim() !== '', writes
 *    content to localStorage and clears dirty. Otherwise no-op.
 *  - On unmount or `isOpen` transition true ‚Üí false: clears the interval.
 *
 * @see services/composer-drafts-storage.ts for the underlying CRUD.
 */
export function useComposerDraft({
  draftKey,
  content,
  isOpen,
}: UseComposerDraftOptions): UseComposerDraftResult {
  const [draftToRestore, setDraftToRestore] = useState<DraftRecord | null>(null)
  const dirtyRef = useRef(false)
  const contentRef = useRef(content)
  const lastSavedRef = useRef<string | null>(null)
  // Track the most recent key the hook has seen so we can detect prop changes.
  const lastKeyRef = useRef<DraftKey | null>(null)

  // Keep `contentRef` in sync without depending on it in the interval effect,
  // so the interval doesn't re-create on every keystroke.
  useEffect(() => {
    contentRef.current = content
    // Content change marks dirty (unless content matches the last saved value ‚Äî
    // that branch lets us no-op when the user types the same string back).
    if (content !== lastSavedRef.current) {
      dirtyRef.current = true
    }
  }, [content])

  // Open / key change: re-evaluate the restore prompt + reset dirty state.
  useEffect(() => {
    if (!isOpen) {
      // Composer closed ‚Üí clear prompt (next open will re-check).
      setDraftToRestore(null)
      return
    }
    // Open OR key change while open: snapshot the draft for this key.
    const existing = getDraft(draftKey)
    setDraftToRestore(existing)
    // Reset dirty flag and last-saved tracker for the new lifecycle.
    dirtyRef.current = false
    lastSavedRef.current = existing?.content ?? null
    lastKeyRef.current = draftKey
  }, [isOpen, draftKey])

  // Auto-save interval ‚Äî runs only while open. Reads contentRef each tick to
  // avoid re-creating the interval on every keystroke.
  useEffect(() => {
    if (!isOpen) return
    const id = window.setInterval(() => {
      if (!dirtyRef.current) return
      const current = contentRef.current
      if (current.trim() === '') return
      setDraft(draftKey, current)
      lastSavedRef.current = current
      dirtyRef.current = false
    }, COMPOSER_DRAFT_TICK_MS)
    return () => window.clearInterval(id)
  }, [isOpen, draftKey])

  const restoreDraft = (): string => {
    const restored = draftToRestore?.content ?? ''
    setDraftToRestore(null)
    // After restore, the restored content becomes the "last saved" baseline
    // so an immediate auto-save tick doesn't pointlessly re-save the same
    // content. Caller's setContent will run after restoreDraft returns.
    lastSavedRef.current = restored
    dirtyRef.current = false
    return restored
  }

  const discardDraft = (): void => {
    removeDraft(draftKey)
    setDraftToRestore(null)
    lastSavedRef.current = null
  }

  const clearDraft = (): void => {
    removeDraft(draftKey)
    setDraftToRestore(null)
    lastSavedRef.current = null
    dirtyRef.current = false
  }

  return { draftToRestore, restoreDraft, discardDraft, clearDraft }
}
```

**Guardrails (DO NOT):**
- DO NOT save on every keystroke (W1 ‚Äî debounce via dirty flag, not throttle).
- DO NOT save empty/whitespace content (advisor + acceptance criterion alignment).
- DO NOT trigger restore-prompt logic on React mount; use `isOpen` transition (R4).
- DO NOT carry dirty state across `draftKey` changes (advisor flag ‚Äî postType change in the same component instance).
- DO NOT use `setInterval` outside the open effect (would burn CPU on closed composers).
- DO NOT include `content` in the interval `useEffect` dependency array ‚Äî that would re-create the interval on every keystroke and defeat the dirty-flag design. Use `contentRef` instead.
- DO NOT log `content` or `draftKey` with content payloads anywhere (W8).

| # | Decision | Reason | Reversible? |

| 1 | Split into storage service + hook (brief says "one custom hook"). | Matches the canonical pattern from Spec 6.8 (`verse-dismissals-storage.ts` + `useVerseFindsYou`). Adds one file, but the file is small (~50 lines) and isolates IO from React. | Yes ‚Äî could be collapsed into a single file in a follow-up if Eric prefers. |

| 3 | Restore prompt is an inline card with `role="status"` + `aria-live="polite"` + focus on primary button, NOT a modal with `useFocusTrap`. | Brief Gate-5 calls for keyboard-reachable + focus-managed, not modal. A modal restore prompt would feel heavy for "want your draft back?" and would complicate dismissal. | 4 | The hook accepts `content: string` as an option (caller's controlled state passed in), NOT a `save(text)` callback. | 6 | `T11 ‚Äî postType change in same component instance` added beyond the brief's T1‚ÄìT12 list. | Advisor flag: InlineComposer doesn't unmount across postType changes (verified via recon). Without this test, the hook's `useEffect` deps on `draftKey` could regress silently. | Yes ‚Äî could be omitted, but at the cost of catching regression late. |
