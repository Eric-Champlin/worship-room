## Affected Frontend Routes

The PresenceIndicator embeds on the PrayerWall feed surface. The crisis-flagged-page suppression rule (Gate-G-CRISIS-SUPPRESSION) applies on every route that can render flagged posts.

- `/prayer-wall` — feed; PresenceIndicator embedded in feed header (right of `PrayerWallViewTabs`), suppressed if any rendered post has `crisisFlag: true`
- `/prayer-wall/:id` — PrayerDetail; PresenceIndicator suppressed if `post.crisisFlag` is true
- `/prayer-wall/user/:id` — PrayerWallProfile; suppressed if any post in the rendered list is crisis-flagged
- `/prayer-wall/dashboard` — PrayerWallDashboard (protected); suppression rule applies if any rendered post is flagged
- `/settings` — "Count me as present when I'm reading" toggle in Gentle extras section

`/prayer-wall/answered` is referenced by the brief but not in the canonical route inventory; recon confirmed no separate route exists (answered posts surface inside `/prayer-wall` via the view-tabs `?view=answered` query param, which is handled by the same `/prayer-wall` route — covered by the feed suppression rule above).

## Architecture Context

### Backend conventions (from recon)

- **Base package** is `com.worshiproom` (the rename from `com.example.worshiproom` shipped in Spec 1.1). New package: `com.worshiproom.presence`.
- **Phase 3 Execution Reality Addendum applies in full** — Spec 6.11b is the first Phase 6 backend-heavy spec that interacts with cross-controller infrastructure (interceptor + cookie + Spring Cache + scheduled job + new column on `users`).
- **Spring Cache is wired** via `CacheConfig.java`:
  - Dev/test → `ConcurrentMapCacheManager` (in-memory, profile-gated).
  - Prod → `RedisCacheManager` wrapped in `CircuitBreakingCacheManager` (3 failures → 30s bypass).
  - `@EnableCaching` already activated (Spec 6.1 wired it for `PrayerReceiptService`).
  - Per-cache TTLs read from `spring.cache.redis.time-to-live.<cacheName>` in `application-prod.properties`.
  - `RepoWideTtlEnforcementTest` verifies every `@Cacheable` has a corresponding TTL property — must add a `spring.cache.redis.time-to-live.prayer-wall-presence` entry to keep the test green.
- **Redis is available** via `RedisTemplate<String, String>` bean (Spec 5.6). Use it directly for sorted-set ops; do NOT introduce a new Redis client.
- **`@EnableScheduling` is already on** via `com.worshiproom.upload.SchedulingConfig`. The new `PresenceCleanupJob` only needs `@Component` + `@Scheduled` annotations.
- **Rate-limit pattern is established** via `VerseFindsYouRateLimitConfig` (per-user Caffeine-bounded bucket map, `@ConfigurationProperties(prefix = "worshiproom.verse-finds-you")`). Mirror this for presence — one config for authenticated (`worshiproom.presence.auth-rate-limit`), one for anonymous (`worshiproom.presence.anon-rate-limit`).
- **CORS is currently `allowCredentials(false)`** via the `ALLOW_CREDENTIALS = false` constant in `CorsConfig.java`. The presence cookie requires this to flip to `true` — see Plan-Time Divergence #1 below.
- **SecurityConfig pattern** — `/api/v1/prayer-wall/presence` must be ADDED to `OPTIONAL_AUTH_PATTERNS` in `PublicPaths.java` so anonymous visitors can call it. Anonymous calls extract no principal; authenticated calls extract one.
- **`@RestControllerAdvice` scoping** — create `com.worshiproom.presence.PresenceExceptionHandler` scoped to `basePackages = "com.worshiproom.presence"` per Phase 3 Addendum item 6. Domain-scoped, NEVER extend `PostExceptionHandler` for presence exceptions.

## Spec-Category-Specific Guidance

### This is a BACKEND-AND-FRONTEND spec (NOT dual-write, NOT cutover)

- Backend ships a new endpoint, interceptor, cleanup job, and column. Frontend ships a new component, hook, and settings toggle.
- This is NOT a dual-write migration; presence is greenfield with no localStorage primary.
- The opt-out preference DOES persist to both localStorage (`wr_settings.presence.optedOut`) AND the backend (`users.presence_opted_out`) per the brief's explicit requirement — but this is opt-out persistence, not a migration. localStorage is the fast-read client-side mirror; backend is the authority for the count-exclusion check.

```java
@Service
public class PresenceService {

    private static final Logger log = LoggerFactory.getLogger(PresenceService.class);
    private static final String USER_PREFIX = "user:";
    private static final String ANON_PREFIX = "anon:";

    private final RedisTemplate<String, String> redis;
    private final PresenceProperties props;
    private final UserRepository userRepository;

    public PresenceService(RedisTemplate<String, String> redis,
                           PresenceProperties props,
                           UserRepository userRepository) {
        this.redis = redis;
        this.props = props;
        this.userRepository = userRepository;
    }

    /** Bump the score for an authenticated user (idempotent — multiple tabs same user). */
    public void bumpUser(UUID userId) {
        String member = USER_PREFIX + userId;
        redis.opsForZSet().add(props.getSortedSetKey(), member, nowSeconds());
    }

    /** Bump the score for an anonymous visitor by their cookie session id. */
    public void bumpAnon(String sessionId) {
        String member = ANON_PREFIX + sessionId;
        redis.opsForZSet().add(props.getSortedSetKey(), member, nowSeconds());
    }

    /**
     * Returns the current count of "present" members (score newer than 60-min TTL),
     * minus any authenticated users with presence_opted_out=true.
     *
     * <p>Cached for 30 seconds (Spring @Cacheable). Single global cache entry.
     */
    @Cacheable(value = "prayer-wall-presence")
    public PresenceResponse getCount() {
        double now = nowSeconds();
        double cutoff = now - props.getTtlSeconds();
        Set<String> members = redis.opsForZSet()
            .rangeByScore(props.getSortedSetKey(), cutoff, Double.POSITIVE_INFINITY);
        if (members == null || members.isEmpty()) {
            return new PresenceResponse(0);
        }

        // Partition members by prefix.
        Set<UUID> userIds = new HashSet<>();
        int anonCount = 0;
        for (String member : members) {
            if (member.startsWith(USER_PREFIX)) {
                try {
                    userIds.add(UUID.fromString(member.substring(USER_PREFIX.length())));
                } catch (IllegalArgumentException e) {
                    // Malformed member — skip silently, will age out via cleanup job.
                    log.warn("Skipping malformed presence member: {}", member);
                }
            } else if (member.startsWith(ANON_PREFIX)) {
                anonCount++;
            }
            // Unknown prefix → skip (defensive — should never happen).
        }

        int authCount = userIds.size();
        if (!userIds.isEmpty()) {
            Set<UUID> optedOut = userRepository.findIdsByPresenceOptedOutTrue(userIds);
            authCount -= optedOut.size();
        }

        return new PresenceResponse(Math.max(0, authCount + anonCount));
    }

    /** Sorted-set cleanup — remove members older than 60-min TTL. Returns removed count. */
    public long cleanup() {
        double cutoff = nowSeconds() - props.getTtlSeconds();
        Long removed = redis.opsForZSet()
            .removeRangeByScore(props.getSortedSetKey(), 0, cutoff);
        return removed != null ? removed : 0L;
    }

    private double nowSeconds() {
        return Instant.now().getEpochSecond();
    }
}
```

**Objective:** Centralize the cookie-issue logic. Used by the interceptor (Step 7) and any future code that needs to read or refresh the cookie.

**Details:**

```java
@Service
public class PresenceCookieService {

    private final PresenceProperties props;

    public PresenceCookieService(PresenceProperties props) {
        this.props = props;
    }

    /** Extract the session id from the request cookie, or empty if absent. */
    public Optional<String> readSessionId(HttpServletRequest request) {
        Cookie[] cookies = request.getCookies();
        if (cookies == null) return Optional.empty();
        return Arrays.stream(cookies)
            .filter(c -> props.getCookieName().equals(c.getName()))
            .map(Cookie::getValue)
            .filter(this::isValidSessionId)
            .findFirst();
    }

    /** Issue a new session id and write the cookie to the response. */
    public String issue(HttpServletResponse response) {
        String sessionId = UUID.randomUUID().toString();
        ResponseCookie cookie = ResponseCookie.from(props.getCookieName(), sessionId)
            .httpOnly(true)
            .secure(props.isCookieSecure())
            .sameSite("Lax")
            .path(props.getCookiePath())
            .maxAge(Duration.ofSeconds(props.getCookieMaxAgeSeconds()))
            .build();
        response.addHeader(HttpHeaders.SET_COOKIE, cookie.toString());
        return sessionId;
    }

    /** Defensive validation — UUID format, no padding, no whitespace. */
    private boolean isValidSessionId(String value) {
        if (value == null || value.length() != 36) return false;
        try {
            UUID.fromString(value);
            return true;
        } catch (IllegalArgumentException e) {
            return false;
        }
    }
}
```

```java
@Component
public class PresenceTrackingInterceptor implements HandlerInterceptor {

    private static final Logger log = LoggerFactory.getLogger(PresenceTrackingInterceptor.class);

    private final PresenceService service;
    private final PresenceCookieService cookieService;

    public PresenceTrackingInterceptor(PresenceService service, PresenceCookieService cookieService) {
        this.service = service;
        this.cookieService = cookieService;
    }

    @Override
    public boolean preHandle(HttpServletRequest request,
                             HttpServletResponse response,
                             Object handler) {
        // Only bump on GET — write methods are not "reading the wall."
        if (!"GET".equalsIgnoreCase(request.getMethod())) return true;

        // Authenticated branch: SecurityContextHolder has the principal.
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth != null
                && auth.isAuthenticated()
                && auth.getPrincipal() instanceof AuthenticatedUser principal) {
            try {
                service.bumpUser(principal.userId());
            } catch (Exception e) {
                // Presence is a best-effort signal — never fail the request.
                log.warn("presenceBumpFailed userId={} cause={}", principal.userId(), e.getClass().getSimpleName());
            }
            return true;
        }

        // Anonymous branch: read or issue the session cookie.
        try {
            String sessionId = cookieService.readSessionId(request)
                .orElseGet(() -> cookieService.issue(response));
            service.bumpAnon(sessionId);
        } catch (Exception e) {
            log.warn("presenceBumpAnonFailed cause={}", e.getClass().getSimpleName());
        }
        return true;
    }
}
```

```java
public class PresenceAuthRateLimitService {
    private final Cache<UUID, Bucket> userBuckets;
    private final PresenceAuthRateLimitConfig config;

    public PresenceAuthRateLimitService(PresenceAuthRateLimitConfig config) {
        this.config = config;
        this.userBuckets = Caffeine.newBuilder()
            .maximumSize(config.getBucketMapMaxSize())
            .expireAfterAccess(Duration.ofMinutes(config.getBucketMapTtlMinutes()))
            .build();
    }

    public void checkAndConsume(UUID userId) {
        Bucket bucket = userBuckets.get(userId, k -> newBucket());
        ConsumptionProbe probe = bucket.tryConsumeAndReturnRemaining(1);
        if (!probe.isConsumed()) {
            long retryAfterSec = Math.max(1L,
                TimeUnit.NANOSECONDS.toSeconds(probe.getNanosToWaitForRefill()));
            throw new PresenceRateLimitException(retryAfterSec, "auth");
        }
    }

    private Bucket newBucket() {
        int max = config.getRequestsPerMinute();
        Bandwidth limit = Bandwidth.classic(max, Refill.intervally(max, Duration.ofMinutes(1)));
        return Bucket.builder().addLimit(limit).build();
    }
```

```ts
  const [count, setCount] = useState<number | null>(null)
  const [paused, setPaused] = useState(false)
  const intervalRef = useRef<ReturnType<typeof setInterval> | null>(null)
  const backoffRef = useRef<ReturnType<typeof setTimeout> | null>(null)

      setCount(null)

        const result = await apiFetch<PresenceResponse>('/api/v1/prayer-wall/presence')
        if (!cancelled) setCount(result.count)
      } catch (err) {
        if (err instanceof ApiError && err.status === 429) {
          // Backoff for 60s, then resume.
          if (intervalRef.current) {
            clearInterval(intervalRef.current)
            intervalRef.current = null
          }
          backoffRef.current = setTimeout(() => {
            backoffRef.current = null
            startPolling()
          }, BACKOFF_AFTER_429_MS)
        }
        // Other errors: silent failure (Gate-G-SILENT-FAILURE convention from 6.8).
      } finally {
        isFetchingRef.current = false
      }
    }

    function startPolling() {
      if (cancelled || suppressed) return
      void fetchCount()
      intervalRef.current = setInterval(fetchCount, POLL_INTERVAL_MS)
    }

    function handleVisibility() {
      if (document.visibilityState === 'visible') {
        setPaused(false)
        startPolling()
      } else {
        setPaused(true)
        if (intervalRef.current) {
          clearInterval(intervalRef.current)
          intervalRef.current = null
        }
      }
    }

    document.addEventListener('visibilitychange', handleVisibility)

    if (document.visibilityState === 'visible') {
      startPolling()
    } else {
      setPaused(true)
    }

    return () => {
      cancelled = true
      document.removeEventListener('visibilitychange', handleVisibility)
      if (intervalRef.current) clearInterval(intervalRef.current)
      if (backoffRef.current) clearTimeout(backoffRef.current)
    }
  }, [suppressed])

  return { count, paused }
}
```

**Guardrails (DO NOT):**

- Do NOT make a burst of catch-up requests on `visibilitychange` → visible — one immediate poll then resume cadence (brief W20).

- Do NOT fire a fetch when `suppressed` is true — Gate-G-CRISIS-SUPPRESSION is an early exit (no API call wasted).
