## Implementation Steps

### Step 1: Backend — parameterize `ContentTooLongException`

**Objective:** Make the exception accept the per-type limit and interpolate it into the message. All call sites updated atomically.

**Files to create/modify:**
- `backend/src/main/java/com/worshiproom/post/ContentTooLongException.java` — replace the 0-arg constructor with a 1-arg `(int maxLength)` constructor. Build the message via string concatenation with the parameter.

**Details:**

```java
// backend/src/main/java/com/worshiproom/post/ContentTooLongException.java
package com.worshiproom.post;

import org.springframework.http.HttpStatus;

public class ContentTooLongException extends PostException {
    public ContentTooLongException(int maxLength) {
        super(HttpStatus.BAD_REQUEST, "INVALID_INPUT",
              "Post content exceeds the " + maxLength + " character limit after HTML sanitization.");
    }
}
```

Do NOT keep a deprecated 0-arg constructor as a back-compat shim. The single file-internal call site in PostService is updated in Step 2.

**Guardrails (DO NOT):**
- Do NOT change the error code (`INVALID_INPUT` stays — API contract).
