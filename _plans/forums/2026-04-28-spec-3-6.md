## Edge Cases & Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Edit-window error status | **409 CONFLICT** with code `EDIT_WINDOW_EXPIRED` (NOT 400 per Phase 3.6 Addendum line 3806) | Within-codebase API consistency with Spec 3.5's `EditWindowExpiredException` (HttpStatus.CONFLICT). Frontend handles `EDIT_WINDOW_EXPIRED` once across both posts and comments. Spec D1. |
| Endpoint paths | POST nested (`/posts/{postId}/comments`); PATCH/DELETE non-nested (`/comments/{id}`) | REST convention + parity with Spec 3.5's `PATCH/DELETE /api/v1/posts/{id}`. The addendum's nested PATCH path (`/posts/{post_id}/comments/{comment_id}`) is redundant ‚Äî comment UUID alone identifies ownership and parent post. Spec D2. |
| Activity type | New `INTERCESSION` (13th enum entry), 10 points, source-feature `prayer-wall-comment` | Master plan calls for "intercession activity type, smaller point value than posting." 10 matches `pray=10`/`listen=10`/`devotional=10` low-intensity-participation tier. PRAYER_WALL=15 stays larger. Spec D3. |
| Crisis detection mechanism | Keyword-only (`CommentCrisisDetector`, mirror of `PostCrisisDetector`) | Inherits Spec 3.5 D1. LLM classifier upgrade tracked in followup #15. |
| Crisis alerting | Sentry + DB flag (NO email) | Inherits Spec 3.5 D2. SMTP wiring tracked in followup #16. |
| Soft-delete content treatment | Preserve content; user_id stays | Inherits Spec 3.5 D6. Account-deletion content replacement is Spec 10.11. |
| `is_anonymous` | NOT supported on comments | Schema has no column. Future spec problem if needed. Spec D7. |
| 403 vs 404 on already-soft-deleted-not-owned DELETE | **403 FORBIDDEN** | Matches Spec 3.5 resolved choice (Option A from 3.5 plan review). UUIDs aren't enumerable; UX upside (telling legitimate user "this isn't yours") wins over theoretical privacy-leak. Spec D8. |
| `comment_count` on soft-delete | **Decrements** with `> 0` guard | Read endpoint filters `is_deleted=true`. Without decrement, displayed count drifts upward. SQL-side `UPDATE posts SET comment_count = comment_count - 1 WHERE id = ? AND comment_count > 0`. Spec D9. |
| Idempotency-Key support on POST | **YES**, mirror Spec 3.5 (per-(userId, key), 24h TTL, body-hash mismatch ‚Üí 422 IDEMPOTENCY_KEY_MISMATCH) | Same network-retry duplication risk as posts. Master plan didn't specify but mirroring is safer than not. Spec D10. |

**Details:**

`CrisisAlertService.java` new alert method:
```java
public void alert(UUID contentId, UUID authorId, ContentType type) {
    log.info("crisisKeywordMatch contentId={} userId={} contentType={}", contentId, authorId, type);
    Sentry.captureMessage(
        "Crisis keyword match on prayer wall content",
        scope -> {
            scope.setLevel(SentryLevel.WARNING);
            scope.setTag("event_type", "crisis_keyword_match");
            scope.setTag("content_type", type.name().toLowerCase());  // "post" or "comment"
            scope.setTag("content_id", contentId.toString());
            scope.setTag("user_id", authorId.toString());
            // NO content tag ‚Äî content stays in Postgres.
        }
    );
}
```

`CrisisDetectedEventListener.java` updated handler:
```java
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
public void handleCrisisDetected(CrisisDetectedEvent event) {
    if (alertedContentCache.getIfPresent(event.contentId()) != null) {
        log.debug("Crisis alert dedup hit contentId={}", event.contentId());
        return;
    }
    alertedContentCache.put(event.contentId(), Boolean.TRUE);
    try {
        crisisAlertService.alert(event.contentId(), event.authorId(), event.type());
    } catch (Exception e) {
        log.warn("crisisAlertFailed contentId={} userId={}", event.contentId(), event.authorId(), e);
    }
}
```

Note dedup key is `contentId` only ‚Äî a user who hits crisis on both a post AND a comment within 1 hour will (correctly) get TWO alerts, since post and comment UUIDs are different. Spec line 48.

**Guardrails (DO NOT):**
- DO NOT add `contentType` to the dedup cache key. UUIDs are globally unique; same UUID can never appear as both post and comment.
- DO NOT log content text or any portion of the user's message in either the application log or the Sentry message string (07-logging-monitoring.md PII boundary, hard rule).
- DO NOT change `@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)` to `@Async` or `BEFORE_COMMIT`. AFTER_COMMIT means alerts only fire for actually-persisted content.
- DO NOT remove the try/catch in the listener ‚Äî Sentry network failures must not propagate.

```java
@Service
public class CommentsRateLimitService {
    private final Cache<UUID, Bucket> bucketsByUser;
    private final CommentsRateLimitConfig config;

    public CommentsRateLimitService(CommentsRateLimitConfig config) {
        this.config = config;
        this.bucketsByUser = Caffeine.newBuilder()
                .maximumSize(config.getRateLimit().getBucketCacheSize())
                .expireAfterAccess(Duration.ofHours(2))  // longer than the 1h refill window so an active bucket isn't evicted prematurely (per spec line 46)
                .build();
    }

    public void checkAndConsume(UUID userId) {
        Bucket bucket = bucketsByUser.get(userId, this::newBucket);
        ConsumptionProbe probe = bucket.tryConsumeAndReturnRemaining(1);
        if (!probe.isConsumed()) {
            long retryAfterSeconds = probe.getNanosToWaitForRefill() / 1_000_000_000L;
            throw new CommentsRateLimitException(retryAfterSeconds);
        }
    }

    private Bucket newBucket(UUID userId) {
        Bandwidth limit = Bandwidth.classic(
                config.getRateLimit().getMaxPerHour(),
                Refill.intervally(config.getRateLimit().getMaxPerHour(), Duration.ofHours(1))
        );
        return Bucket.builder().addLimit(limit).build();
    }
}
```

```java
public Optional<CreateCommentResponse> lookup(UUID userId, String idempotencyKey, int requestBodyHash) {
    if (idempotencyKey == null || idempotencyKey.isBlank()) return Optional.empty();
    CachedEntry entry = cache.getIfPresent(new CacheKey(userId, idempotencyKey));
```

```java
public CreateCommentResponse createComment(
        UUID postId,
        UUID authorId,
        CreateCommentRequest request,
        String idempotencyKey,
        String requestId
) {
    // Step 1: Idempotency lookup BEFORE rate limit (replay must not consume tokens).
    int bodyHash = request.hashCode();
    Optional<CreateCommentResponse> cached = idempotencyService.lookup(authorId, idempotencyKey, bodyHash);
    if (cached.isPresent()) return cached.get();

    rateLimitService.checkAndConsume(authorId);

    // Step 3: Parent post existence (live posts only ‚Äî soft-deleted = 404 for write).

            .orElseThrow(PostNotFoundException::new);

                .findByIdAndPostIdAndIsDeletedFalse(request.parentCommentId(), postId)
                .orElseThrow(InvalidParentCommentException::new);
        // Implicit: parent's postId matches URL's postId (the query enforces it).
    }

    // Step 5: Sanitize.
    String sanitizedContent = htmlSanitizerPolicy.sanitize(request.content()).trim();
    if (sanitizedContent.isEmpty()) throw new EmptyCommentContentException();

    comment.setId(UUID.randomUUID());
    comment.setPostId(postId);
    comment.setUserId(authorId);
    comment.setParentCommentId(request.parentCommentId());

    idempotencyService.store(authorId, idempotencyKey, bodyHash, response);

    // Step 14: Structured log INFO ‚Äî IDs only.
    log.info("commentCreated commentId={} postId={} userId={} parentCommentId={} crisisFlag={} requestId={}",
            saved.getId(), postId, authorId, request.parentCommentId(), crisisFlag, requestId);

    return response;
}
```

```java
@Transactional
public CommentDto updateComment(
        UUID commentId,
        AuthenticatedUser principal,
        UpdateCommentRequest request,
        String requestId
) {
    if (request.isEmpty()) throw new EmptyPatchBodyException();  // reuse existing post exception

    // Step 1: Lookup live comment (soft-deleted = 404).
    PostComment comment = commentRepository.findByIdAndIsDeletedFalse(commentId)
            .orElseThrow(PostCommentNotFoundException::new);

    // Step 2: Author or admin gate.
    boolean isAuthor = comment.getUserId().equals(principal.userId());
    if (!isAuthor && !principal.isAdmin()) {
        throw new CommentForbiddenException();
    }

    // Step 3: Edit window check.
    OffsetDateTime now = OffsetDateTime.now(ZoneOffset.UTC);
    boolean withinWindow = !now.isAfter(
            comment.getCreatedAt().plusMinutes(config.getEditWindowMinutes()));
    if (!withinWindow) {
        throw new CommentEditWindowExpiredException();
    }

    // Step 4: Sanitize new content.
    String sanitizedContent = htmlSanitizerPolicy.sanitize(request.content()).trim();
    if (sanitizedContent.isEmpty()) throw new EmptyCommentContentException();
    if (sanitizedContent.length() > 5000) throw new ContentTooLongException();  // reuse existing post exception

    // Step 5: Crisis re-detection. If newly detected (was clean, now crisis), set flag and publish event.
    //         If previously flagged but content edited to clean, flag stays (not clearable by author).
    boolean fireCrisisEvent = false;
    if (CommentCrisisDetector.detectsCrisis(sanitizedContent)) {
        if (!comment.isCrisisFlag()) comment.setCrisisFlag(true);
        fireCrisisEvent = true;
    }
    // crisis_flag NEVER cleared on author edit.

    // Step 6: Apply mutation.
    comment.setContent(sanitizedContent);
    comment.setUpdatedAt(now);
    PostComment saved = commentRepository.save(comment);

    // Step 7: NO last_activity_at bump on PATCH (Watch-For #15).
    // Step 8: NO activity recording on edit (mirrors PostService).

    // Step 9: AFTER_COMMIT crisis event if needed.
    if (fireCrisisEvent) {
        eventPublisher.publishEvent(new CrisisDetectedEvent(saved.getId(), saved.getUserId(), ContentType.COMMENT));
    }

    // Step 10: Resolve author + map.
    AuthorDto author = resolveAuthor(saved.getUserId());

    log.info("commentUpdated commentId={} userId={} editorId={} crisisFlag={} requestId={}",
            saved.getId(), saved.getUserId(), principal.userId(), saved.isCrisisFlag(), requestId);

    return commentMapper.toDto(saved, author);
}
```

```java
@Transactional
public void deleteComment(UUID commentId, AuthenticatedUser principal, String requestId) {
    // Step 1: Idempotency ‚Äî if already soft-deleted, ownership check + 204.
    Optional<PostComment> already = commentRepository.findByIdAndIsDeletedTrue(commentId);
    if (already.isPresent()) {
        PostComment deletedComment = already.get();
        if (!deletedComment.getUserId().equals(principal.userId()) && !principal.isAdmin()) {
            throw new CommentForbiddenException();  // Spec D8 ‚Äî 403 not 404
        }
        log.info("commentDeleteIdempotent commentId={} userId={} requestId={}",
                commentId, principal.userId(), requestId);
        return;
    }

    // Step 2: Live lookup.
    PostComment comment = commentRepository.findByIdAndIsDeletedFalse(commentId)
            .orElseThrow(PostCommentNotFoundException::new);

    // Step 3: Ownership gate.
    if (!comment.getUserId().equals(principal.userId()) && !principal.isAdmin()) {
        throw new CommentForbiddenException();
    }

    // Step 4: Apply soft-delete. Content stays. user_id stays. Replies stay (no cascade).
    OffsetDateTime now = OffsetDateTime.now(ZoneOffset.UTC);
    comment.setDeleted(true);
    comment.setDeletedAt(now);
    comment.setUpdatedAt(now);
    commentRepository.save(comment);

    // Step 5: SQL-side counter decrement with > 0 guard (Spec D9).
    postRepository.decrementCommentCount(comment.getPostId());
    // We deliberately do NOT assert rowsAffected==1 here ‚Äî under any race, the
    // > 0 guard correctly produces 0 rows affected and we treat that as
    // "counter already at floor, no-op". Any negative drift would be a bug.

    // Step 6: NO last_activity_at bump on DELETE (Watch-For #15).
    // Step 7: NO cascade to replies (Watch-For #22).
    // Step 8: NO activity reversal (no negative-activity rows).

    log.info("commentDeleted commentId={} postId={} userId={} deletedBy={} requestId={}",
            commentId, comment.getPostId(), comment.getUserId(), principal.userId(), requestId);
}
```
