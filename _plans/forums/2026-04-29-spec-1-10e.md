## Edge Cases & Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Add `/dev-storage/**` to `PATTERNS` or `OPTIONAL_AUTH_PATTERNS`? | OPTIONAL_AUTH_PATTERNS (per spec § "Phase 3 Execution Reality Addendum — Applicability"). | Both technically permit anonymous access. PATTERNS would skip JwtAuthenticationFilter entirely; OPTIONAL_AUTH_PATTERNS lets it run but treats result as optional. The spec author chose OPTIONAL_AUTH_PATTERNS; both are functionally equivalent for the dev controller use case. Documented as Plan-Time Divergence #1 below. |
| MinIO container vs LocalStack | MinIO. | S3-API-compatible, ~1s startup, well-known Testcontainers integration. LocalStack is heavier and overkill for a simple S3 contract test. |
| Per-class MinIO container vs JVM-singleton | Per-class (`@Container static`) declared on `S3StorageAdapterIntegrationTest`. | Only one test class needs it. Singleton-across-classes (the PostgreSQL pattern) is overkill — adds 1s of JVM-shared startup that doesn't amortize since only one class uses it. |
| Where does the dev-profile HMAC signing secret live? | `worshiproom.storage.dev-signing-secret` in `application-dev.properties`, with a dev-only fallback value. | Mirrors the JWT_SECRET pattern (`jwt.secret=${JWT_SECRET:dev-jwt-secret-...}` in `application-dev.properties`). Dev-only; not used in test/prod profiles since `LocalFilesystemStorageAdapter` is dev-only. |
| Sidecar `.meta.json` placement for LocalFilesystem | Same directory as the data file, named `<filename>.meta.json` (e.g., `post-images/abc.jpg` + `post-images/abc.jpg.meta.json`). | Spec body line 2515. Keeps metadata co-located; cleanup is "delete both" on `delete()`. |
| Key sanitization rejection mechanism | `IllegalArgumentException` thrown from a static `StorageKeyValidator.validate(String key)` helper called by every adapter method that takes a key. | Per spec acceptance criterion line 2581. Validation runs BEFORE any IO so unit tests don't need a live adapter (Plan Tightening Audit Lens 6). |
| Allowed key character set | `[a-z0-9._/-]` (lowercase letters, digits, period, underscore, forward slash, hyphen). Reject `..` substring, leading slash, trailing slash, double slashes. | Spec body line 2521 ("lowercase, forward-slash-separated, hierarchical"). Hyphen and underscore are common in object stores; period is allowed for file extensions. |
| Max key length | 256 chars (per spec body line 2524). | S3 allows 1024; the buffer protects against provider drift. |
| Retry strategy delay sequence | 100ms, 300ms, 1000ms (3 attempts, exponential backoff). | Spec body line 2536. Encoded as a static `List<Duration>` constant in `S3StorageAdapter`. |
| Multipart upload threshold | AWS SDK v2 `S3TransferManager` default (8 MB partition). | Sufficient for the 6 MB large-file test. Re-evaluating partition size is Spec 4.6b's concern when image upload sizes are known. |
| What does "test" profile use — MinIO or LocalFilesystem? | MinIO via S3StorageAdapter (per spec body line 2502 "Active under `test` profile" + master plan body line 2569 "Spring profile `test` selects MinIOStorageAdapter (via same S3StorageAdapter code)"). | The whole point of the test profile is to exercise the S3 code path against an S3-compatible backend so the contract test is meaningful for production. |
| `S3StorageAdapter` constructor — env var vs S3Client bean? | `@ConfigurationProperties`-bound `StorageProperties` injected; `S3StorageAdapter` constructs its own `S3Client` and `S3Presigner` from those properties. | Mirrors Phase 1 patterns (`PostsRateLimitConfig` injected into `PostsRateLimitService`). Construction-from-properties is testable (test passes its own `StorageProperties`). |
| Production fail-fast validator placement | `@PostConstruct` method on `S3StorageAdapter` itself. | Per spec acceptance criterion line 2585. The bean is constructed before `ApplicationContext` finishes refreshing; `@PostConstruct` throwing `IllegalStateException` aborts startup before traffic arrives. |
| Should the fail-fast validator run in test profile? | NO — only in `prod` profile. | Test uses MinIO with values supplied by `@DynamicPropertySource`; dev uses LocalFilesystem (S3 adapter not even constructed); only prod profile boot with missing env vars is the dangerous case. Validator checks `Environment.getActiveProfiles()` for "prod" before throwing. |

### Step 3: Create `StorageKeyValidator` static helper + tests

**Objective:** Centralize key sanitization so it runs BEFORE any IO and is unit-testable without a live adapter (Plan Tightening Audit Lens 6).

**Files to create/modify:**
- `backend/src/main/java/com/worshiproom/storage/StorageKeyValidator.java` — final class with private constructor and static `validate(String key)` method. Throws `IllegalArgumentException` with descriptive message on rejection.

**Details:**

```java
public final class StorageKeyValidator {
    private StorageKeyValidator() {}

    private static final Pattern ALLOWED = Pattern.compile("^[a-z0-9._/-]+$");
    private static final int MAX_LENGTH = 256;

    public static void validate(String key) {
        if (key == null || key.isEmpty()) {
            throw new IllegalArgumentException("Storage key must not be null or empty");
        }
        if (key.length() > MAX_LENGTH) {
            throw new IllegalArgumentException("Storage key exceeds 256 characters");
        }
        if (key.startsWith("/") || key.endsWith("/")) {
            throw new IllegalArgumentException("Storage key must not start or end with '/'");
        }
        if (key.contains("//")) {
            throw new IllegalArgumentException("Storage key must not contain '//'");
        }
        if (key.contains("..")) {
            throw new IllegalArgumentException("Storage key must not contain '..'");
        }
        if (!ALLOWED.matcher(key).matches()) {
            throw new IllegalArgumentException("Storage key contains illegal characters (allowed: a-z 0-9 . _ / -)");
        }
    }
}
```

**Guardrails (DO NOT):**
- Do NOT log the rejected key value at WARN — only the error class and rejection reason. (Defensive: an attacker probing for path traversal would otherwise have their probes immortalized in logs.)
- Do NOT use `String.contains` for the `..` check on its own — verify with the regex AND the explicit `contains("..")` check (belt-and-suspenders; the regex already rejects `..` since `.` is allowed but the explicit check makes the intent obvious).
- Do NOT make the validator `static` on the interface — keep it a separate class so adapters can call it without inheritance gymnastics.

**Verification:**
- [ ] `./mvnw compile` passes
- [ ] All 8+ rejection cases fail closed (throw IAE)
- [ ] Valid keys pass without exception

public interface ObjectStorageAdapter {

- `put(...)`:
  1. `StorageKeyValidator.validate(key)` first.
  2. Resolve target path: `Paths.get(properties.getLocalPath()).resolve(key)`. Defensive check: ensure the resolved path stays within `properties.getLocalPath()` (defensive double-check vs `..` traversal — the validator already rejects `..`, but a layered guard is cheap).
  3. `Files.createDirectories(target.getParent())`.
  4. Stream-copy `data` to a temp file via `Files.copy(data, tempPath, StandardCopyOption.REPLACE_EXISTING)`. Track byte count via a counting input stream; throw `ObjectStorageIntegrityException` if `actualBytes != contentLength`.
  5. Atomic move to final path: `Files.move(tempPath, target, StandardCopyOption.ATOMIC_MOVE)`. (Spec § "Out-of-band notes" acknowledges LocalFilesystem is not production-grade for concurrency; ATOMIC_MOVE is best-effort.)
  6. Write sidecar `.meta.json` with `{ contentType, metadata }` to `<target>.meta.json`.
  7. Compute MD5 hex as ETag (LocalFilesystem-specific; S3 will use its own ETag).
  8. Return `new StoredObject(key, sizeBytes, etag, contentType)`.

- `get(key)`:
  1. Validate key.
  2. Read sidecar `.meta.json` to recover `contentType` and `metadata`.
  3. Return `Optional.of(new StoredObjectStream(key, Files.newInputStream(target), Files.size(target), contentType, metadata))`. Empty if file does not exist.

- `exists(key)`: validate key, return `Files.exists(target)`.

- `delete(key)`:
  1. Validate key.
  2. Delete data file AND sidecar `.meta.json`. Return true if either existed.

- `list(prefix, maxResults)`:
  1. Validate prefix as a partial key (the validator may reject empty prefix; treat empty as "list all" — special-case to allow). Actually: the validator rejects empty strings, so for `list("", 100)` we skip validation. Only validate non-empty prefixes.
  2. `Files.walk(rootPath)`, filter to regular files (skip `.meta.json` sidecars), filter by `relativizedKey.startsWith(prefix)`, sort by key ascending, limit `maxResults`.
  3. Map each to `StoredObjectSummary` via `Files.size` and `Files.getLastModifiedTime` → `Instant`.

- `generatePresignedUrl(key, expiry)`:
  1. Validate key.
  2. Validate `expiry.isPositive()` else IAE.
  3. Cap at `properties.getMaxPresignHours()`: `Duration capped = expiry.compareTo(Duration.ofHours(maxPresignHours)) > 0 ? Duration.ofHours(maxPresignHours) : expiry`.
  4. Compute `expiresAt = Instant.now().plus(capped).getEpochSecond()`.
  5. Compute HMAC-SHA256 of `key + ":" + expiresAt` using `devSigningSecret` as key. Base64-URL-encode.
  6. Return `"http://localhost:8080/dev-storage/" + key + "?expires=" + expiresAt + "&signature=" + signature`.

**Guardrails (DO NOT):**
- Do NOT use `String.format("%s/%s", root, key)` — use `Paths.get(root).resolve(key)` so the OS-correct separator is applied (Windows compatibility, even though we don't target Windows).
- Do NOT log the sidecar contents on read — sidecar may contain user-provided metadata in the future (4.6b image uploads). Log only the key.
- Do NOT use `MessageDigest.getInstance("MD5")` directly without `try/catch (NoSuchAlgorithmException)` — but MD5 is universally available; wrap in `try` and rethrow as `IllegalStateException` if it ever fails.
- Do NOT include the signing secret in any log line. Failure logs include `key=` and `error class=`, never the secret.

Retry helper:
```java
private <T> T withRetry(Supplier<T> operation) {
    Duration[] delays = { Duration.ofMillis(100), Duration.ofMillis(300), Duration.ofMillis(1000) };
    int attempts = 0;
    while (true) {
        try {
            return operation.get();
        } catch (S3Exception ex) {
            int status = ex.statusCode();
            boolean retriable = status >= 500 || status == 408 || status == 429;
            attempts++;
            if (!retriable || attempts >= delays.length) {
                throw ex;
            }
            log.warn("S3 transient failure attempt={} status={} key=...", attempts, status);
            try { Thread.sleep(delays[attempts - 1].toMillis()); } catch (InterruptedException ie) { Thread.currentThread().interrupt(); throw ex; }
        }
    }
}
```

- ERROR on final failure: include key + error class + status code + requestId from MDC. Sentry captures via the Spring starter (Spec 1.10d).

### Step 9: Create `DevStorageController` (`@Profile("dev")`)

**Objective:** Serve presigned URLs from `LocalFilesystemStorageAdapter` for dev-mode browser fetches.

**Files to create/modify:**

**Details:**

```java
@RestController
@RequestMapping("/dev-storage")
@Profile("dev")
public class DevStorageController {

    private final ObjectStorageAdapter adapter; // injected — LocalFilesystem in dev
    private final StorageProperties properties;

    public DevStorageController(ObjectStorageAdapter adapter, StorageProperties properties) {
        this.adapter = adapter;
        this.properties = properties;
    }

    @GetMapping(value = "/{*key}", produces = MediaType.APPLICATION_OCTET_STREAM_VALUE)
    public ResponseEntity<StreamingResponseBody> serve(
            @PathVariable String key,
            @RequestParam("expires") long expires,
            @RequestParam("signature") String signature) {

        // Strip leading slash that Spring includes with {*key} matrix matcher.
        if (key.startsWith("/")) key = key.substring(1);

        StorageKeyValidator.validate(key);

        // Verify expiry
        if (Instant.now().getEpochSecond() > expires) {
            return ResponseEntity.status(HttpStatus.GONE).build();
        }

        // Verify HMAC
        String expected = computeHmac(key, expires, properties.getDevSigningSecret());
        if (!MessageDigest.isEqual(expected.getBytes(StandardCharsets.UTF_8),
                                    signature.getBytes(StandardCharsets.UTF_8))) {
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED).build();
        }

        var streamOpt = adapter.get(key);
        if (streamOpt.isEmpty()) {
            return ResponseEntity.notFound().build();
        }
        var stream = streamOpt.get();
        StreamingResponseBody body = out -> {
            try (var in = stream.stream()) { in.transferTo(out); }
        };
        return ResponseEntity.ok()
                .contentType(MediaType.parseMediaType(stream.contentType()))
                .contentLength(stream.sizeBytes())
                .body(body);
    }

    private static String computeHmac(String key, long expires, String secret) {
        try {
            Mac mac = Mac.getInstance("HmacSHA256");
            mac.init(new SecretKeySpec(secret.getBytes(StandardCharsets.UTF_8), "HmacSHA256"));
            byte[] sig = mac.doFinal((key + ":" + expires).getBytes(StandardCharsets.UTF_8));
            return Base64.getUrlEncoder().withoutPadding().encodeToString(sig);
        } catch (NoSuchAlgorithmException | InvalidKeyException e) {
            throw new IllegalStateException("HMAC-SHA256 unavailable", e);
        }
    }
}
```

`@PathVariable String key` with `{*key}` captures multi-segment keys (e.g., `post-images/abc/def/thumb.jpg`).

- Do NOT serve the file unless BOTH expiry AND signature pass — fail closed on either.
- Do NOT register this controller under the `test` or `prod` profile (`@Profile("dev")` on the class).
