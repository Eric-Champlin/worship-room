### Pattern reference

The existing AuthModal.tsx:89 migration is the canonical reference for how the closing-transition pattern should look after migration:

```typescript
// AuthModal.tsx (already shipped ‚Äî reference pattern)
import { ANIMATION_DURATIONS } from '@/constants/animation'
// ...
const handleClose = useCallback(() => {
  if (reducedMotion) {
    onClose()
    return
  }
  setIsClosing(true)
  closeTimeoutRef.current = setTimeout(() => {
    setIsClosing(false)
    onClose()
  }, ANIMATION_DURATIONS.fast)
}, [onClose, reducedMotion])
```

The 3 remaining setTimeout migrations (PrayerCard, InteractionBar, DeletePrayerDialog, ReportDialog) follow the same shape: top-of-file named import + token at the call site.

### `prefers-reduced-motion` interaction

Each of the four affected dialog/card components already short-circuits with `useReducedMotion()` BEFORE reaching the animation-coupled setTimeout. The token migration changes only the literal value passed to `setTimeout`; it does NOT alter the reduced-motion code path. The global safety net in `frontend/src/styles/animations.css` also clamps any leaked animations to `0.01ms`. W4 verification confirms both layers still work post-migration.
