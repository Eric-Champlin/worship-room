### Step 1: Extend Playwright config to accept `PLAYWRIGHT_BASE_URL` env var and add `test:e2e` script

**Objective:** Enable the phase01 roundtrip test to run against local dev (default) OR against a deployed Railway URL (prod smoke mode) via a single env var, without forking any test code. Also wire `pnpm test:e2e` as the canonical invocation.

**Files to create/modify:**
- `frontend/playwright.config.ts` — extend to read `process.env.PLAYWRIGHT_BASE_URL`. Outside `frontend/src/**`, so this does NOT violate spec § 10 guardrail.
- `frontend/package.json` — add `"test:e2e": "playwright test"` script. Outside `src/**`.

**Details:**

`frontend/playwright.config.ts` — replace the 9-line file with:

```ts
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testMatch: ['e2e/**/*.spec.ts', 'tests/**/*.spec.ts'],
  use: {
    headless: true,
    baseURL: process.env.PLAYWRIGHT_BASE_URL ?? 'http://localhost:5173',
  },
});
```

Two behavioral changes: (1) `baseURL` falls back to localhost when `PLAYWRIGHT_BASE_URL` is unset; (2) `testMatch` scopes Playwright's scan to `e2e/**/*.spec.ts` and `tests/**/*.spec.ts`. The `testMatch` bound is load-bearing — without it, Playwright's default scan picks up Vitest test files under `frontend/src/**/__tests__/` (which share the `.spec.ts` suffix with Playwright tests) and collides on `$$jest-matchers-object`. Using `testMatch` instead of `testDir: './e2e'` preserves discoverability of the pre-existing BB-36 performance tests at `frontend/tests/performance/` and BB-41 integration tests at `frontend/tests/integration/` — those directories are documented in `_protocol/99-project-specific-overrides.md` and `frontend/docs/performance-measurement.md` and must continue to respond to `npx playwright test tests/performance/` and `npx playwright test tests/integration/`.

`frontend/package.json` — add one script inside the existing `"scripts"` block, alphabetical placement between `test` and `test:watch`:

```json
"test:e2e": "playwright test",
```

**Guardrails (DO NOT):**
- DO NOT modify any file under `frontend/src/**` or `backend/src/**` (spec § 10 Guardrail 2).
- DO NOT install any new npm deps (`@playwright/test` is already present).
- DO NOT change the default `headless: true`. Eric runs interactively via `--headed` flag when needed.
- DO NOT add webServer config. Playwright launching the dev server from config would introduce a new startup mode that conflicts with the spec's "Prerequisites before running" (§ 6.3).

### Step 2: Author `phase01-auth-roundtrip.spec.ts` with 6 scenarios + axe-core sweep

**Objective:** Produce the canonical Phase 1 end-to-end test. Six scenarios per spec § 6.1, exercising the real backend (no mocks). Plus two axe-core sweeps on `/` and `/prayer-wall` logged-out per Universal Rule 17.

**Files to create/modify:**

**Details:**

File structure (template anchors on `spec-1-9-auth-flow.spec.ts` layout, minus all `page.route` mocks):

```ts
/**
 * Spec 1.10 — Phase 1 Cutover and End-to-End Test
 *
 * Full auth roundtrip against a REAL backend. Two modes:
 *   - Local dev (default):  backend on :8080, frontend on :5173, dev-seed users.
 *   - Production smoke:     set PLAYWRIGHT_BASE_URL to the deployed frontend.
 *                           Backend URL is NOT inferred; health probe is skipped
 *                           in prod mode (scenario 1 fails loudly if backend is down).
 *
 * Dev-seed password 'WorshipRoomDev2026!' is public — it's stored in plaintext in
 * backend/src/main/resources/db/changelog/contexts/dev-seed.xml (context='dev',
 * never loaded in prod). Hardcoding it here is intentional.
 *
 * NO page.route mocks. Every HTTP call goes through to the backend.
 */
import { test, expect, type Page, request as pwRequest } from '@playwright/test'
import AxeBuilder from '@axe-core/playwright'

const IS_PROD_MODE = Boolean(process.env.PLAYWRIGHT_BASE_URL)
const LOCAL_BACKEND_HEALTH_URL = 'http://localhost:8080/actuator/health'
const DEV_SEED_EMAIL = 'sarah@worshiproom.dev'
const DEV_SEED_PASSWORD = 'WorshipRoomDev2026!'  // Public — see file header
const TEST_PASSWORD = 'PlaywrightSmoke2026!'

function freshRegisterEmail(): string {
  return `playwright-test+${Date.now()}@worshiproom.dev`
}

test.describe('Spec 1.10 — Phase 1 auth roundtrip', () => {
  test.beforeAll(async () => {
    if (IS_PROD_MODE) {
      // Prod smoke: trust the operator; scenario 1 will fail loudly if backend is down.
      return
    }
    const ctx = await pwRequest.newContext()
    try {
      const res = await ctx.get(LOCAL_BACKEND_HEALTH_URL, { timeout: 5000 })
      if (!res.ok()) {
        test.skip(true, `Backend health probe returned ${res.status()} at ${LOCAL_BACKEND_HEALTH_URL}. Start the backend: 'docker compose up -d' or './mvnw spring-boot:run'.`)
      }
    } catch (err) {
      test.skip(true, `Backend unreachable at ${LOCAL_BACKEND_HEALTH_URL}: ${(err as Error).message}. Start the backend before running this test.`)
    } finally {
      await ctx.dispose()
    }
  })

  // Scenarios 1-6 + axe-core below.
})
```
